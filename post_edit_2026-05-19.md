## Update — F6.1 partial close (later 2026-05-19)

Per user request, added the per-block direct-sum Killing form as a **toggle**
rather than flipping the default. The ambient `'killing'` path is unchanged;
a new `'killing_per_block'` value selects the direct-sum metric.

**New code:**
- `transformer/core/gauge_preconditioner.py` — `build_killing_form_preconditioner_per_block(generators, irrep_dims, center_reg=None, center_only=True, return_both=False, block_mass_threshold=0.9)`.
  - Identifies each generator's owning block by Frobenius-mass dominance.
  - Builds `metric_h = 2·d_h·gram_h − 2·tr_h⊗tr_h` per block; assembles block-diagonally in the generator index.
  - Per-block center regularisation (each `gl(d_h)`'s center lifted independently).
  - Raises `ValueError` if any generator straddles blocks (direct-sum Killing form is undefined for those).
- `transformer/core/phi_evolution.py::precondition_phi_gradient` — `mode='killing_per_block'` routes through the same `apply_killing_form_natural_gradient` kernel; the semantic difference lives entirely in how the preconditioner was built upstream.

**Wiring:**
- `transformer/vfe/config.py` — `phi_preconditioner: Literal['clip', 'cartan', 'killing', 'killing_per_block']`. Validator updated.
- `transformer/core/block_config.py` — `phi_natural_gradient` docstring lists the new option.
- `transformer/vfe/e_step.py` — `VFEEStep.__init__` builds the per-block preconditioner when `cfg.phi_preconditioner == 'killing_per_block'`.
- `transformer/core/variational_ffn.py` — legacy publication path (`VariationalFFNDynamic.__init__`) supports the new mode; raises if `irrep_dims` is not supplied.

**Tests added** (`tests/transformer/test_gauge_preconditioner.py::TestKillingFormPreconditionerPerBlock`, 10 tests):
- Shape, symmetry, block-diagonality in generator index, per-block scale `2·d_h` (vs ambient `2·K_full`), positive-definiteness, natural-gradient finiteness, natural-gradient does-not-couple-blocks, straddling-generator raises, irrep_dims-sum mismatch raises, per-block ≠ ambient on the same generator bank.

**Validation:**
- `pytest -x` → 950 passed, 3 skipped (+10 vs the v4 baseline, all from the new test class). No regressions.
- End-to-end smoke test on a 2-layer 30-dim model: both `'killing'` and `'killing_per_block'` train, produce finite gradients, and produce numerically distinct losses (5.1657 vs 5.0207) confirming the toggle is responsive.

**Defaults:** unchanged. `phi_preconditioner='killing'` remains the active default; users opt into `'killing_per_block'` explicitly.
