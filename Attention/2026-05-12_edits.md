# 2026-05-12 Edits: Attention-Entropy Term Added to VFE

## Motivation

Audit (3 parallel investigators + verifier) confirmed that both the `transformer/vfe/`
package and the legacy `transformer/core/` + `transformer/train.py` paths implement
the **entropy-suppressed surrogate** `Σ β·KL(q‖Ω q)` rather than the full manuscript
free energy. The manuscript (`Attention/Participatory_it_from_bit.tex`,
`\label{eq:free_energy_functional_final}` near line 1234–1242, sec
`sec:complete_free_energy`) defines the canonical functional with explicit attention
entropy:

```
F = ... + Σ_ij [ β_ij·KL(q_i‖Ω q_j) + κ · β_ij · log(β_ij / π_ij) ] + ...
```

Manuscript line 1261 explicitly warns: the entropy-suppressed surrogate "does not share
stationary points" with the canonical F; its gradient differs by
`-τ⁻¹ Cov_{β*}(KL, ∇KL)`. Practical impact in code:

- Reported scalar F biased by `+κ·H(β)` (skews early stopping, ELBO logging).
- `learnable_kappa` gradient missing `-H(β/π)` term.
- (μ, Σ, φ) gradients envelope-correct at fixed β; `λ_soft·(β·KL.detach()).sum()`
  already incidentally provides the manuscript's covariance correction for these.

Slow channel (`s, r, γ, λ_h`) out of scope per user.

## Changes

### `transformer/vfe/config.py`

Added `include_attention_entropy: bool = True` flag after `learnable_kappa`. Default
ON; disable to recover entropy-suppressed surrogate.

### `transformer/vfe/e_step.py`

Two insertions:

1. Inside `_update_phi` after the existing `alignment_loss` (product-rule decomposition
   of `β·KL`), added:
   ```python
   if self.cfg.include_attention_entropy:
       beta_safe = beta_phi.detach().clamp(min=1e-30)
       entropy_value = (beta_safe * beta_safe.log()).sum()
       alignment_loss = alignment_loss + _kappa * entropy_value
   ```
   `_kappa = self.effective_kappa` is the live tensor (Parameter with gradient when
   `learnable_kappa=True`). `β.detach()` makes envelope theorem give zero
   (μ, Σ, φ) gradient at softmax fixed point; the live κ multiplier provides the
   correct `-H(β)` κ-gradient.

2. Diagnostic metric `attention_entropy_loss` added alongside `attention_entropy` at
   the metrics-collection site (was line 424). Reports `κ · Σ β·log β` (the
   actual F contribution, with sign).

### `transformer/vfe/trainer.py`

Logging line picks up the new metric:
```python
if 'attention_entropy_loss' in metrics:
    msg += f" | F_H {metrics['attention_entropy_loss']:.3f}"
```

### `transformer/train.py`

- Added `include_attention_entropy: bool = True` parameter to
  `compute_free_energy_loss`.
- After `belief_align_loss` assembly (was around line 418), added
  `belief_entropy_loss = M_beta * κ_scalar · Σ β·log β / dim_scale` where `κ_scalar`
  is read from `model.blocks[0].ffn.kappa` with safe fallback to 1.0. Scalar κ here
  (not live tensor) — this term is for F-value correctness only; κ-gradient flow
  lives in the E-step path (`variational_ffn.py`).
- `total_loss` assembly now includes `belief_entropy_loss`.
- Metrics dict reports `loss/belief_entropy`.

### `transformer/core/variational_ffn.py`

- Added `include_attention_entropy: bool = True` constructor parameter; stored as
  `self.include_attention_entropy`.
- Docstring math spec at lines 27–30 extended with the κ·Σβ·log(β/π) term.
- Three alignment-loss assembly sites (the `(β_h, kl_h)`, `(β_phi_h, kl_h)`, and
  `(β_phi, kl_matrix)` variants near lines 1035, 1225, 1257) each appended a
  detached-β entropy term `self.kappa * (β_safe · log β_safe).sum()` gated by the

## Theoretical correctness

For β = softmax(-KL/κ) at the iteration fixed point:

- **κ gradient** (when `learnable_kappa=True`): manuscript envelope-theorem result is
  `∂F/∂κ = Σ β·log(β/π) = -H(β/π)`. The new code's contribution
  `_kappa * Σ β.detach()·log β.detach()` differentiates as
  `∂[_kappa · S]/∂_kappa = S = (β·log β).sum()` — matches manuscript.
- **(μ, Σ, φ) gradient**: detached β gives zero gradient from the new term — matches
  manuscript envelope-theorem result; the existing `λ_soft · (β · KL.detach()).sum()`
  term continues to provide the manuscript's off-equilibrium covariance correction
  `-τ⁻¹ Cov_{β*}(KL, ∇KL)`.
- **Reported F value**: equals manuscript F up to the dropped additive constant
  `κ · log N_valid` (uniform-prior assumption, π = 1/N_valid). Presence-absorbed
  prior `π̃_ij = χ_ij π_ij / Σ_k χ_ik π_ik` is a follow-up refinement.

## Follow-up correction (same day): τ_eff = κ · √K, per-head live κ in legacy path

The first pass used `κ` as the entropy weight, but the softmax in code uses
`β = softmax(-KL / (κ · √K))`, so the manuscript-consistent effective
temperature is `τ_eff = κ · √K`. Without the √K, β is not the stationary point
of F — envelope theorem fails. Corrected:

- `transformer/vfe/e_step.py`: entropy weight now `_kappa · √embed_dim` (single-
  effective-block treatment; multi-irrep configs assume uniform K per block).
- `transformer/core/variational_ffn.py` per-head sites (1041, 1235): entropy weight
  now `kappa_h · √d_h` where `kappa_h = self._get_kappa_h(h, d_h)` (live tensor
  when `learnable_head_kappa=True`).
- `transformer/core/variational_ffn.py` consolidated multi-head site (1271): when
  `learnable_head_kappa=True`, builds a per-head live `kappas` tensor via
  `_get_kappa_h` and weights `(κ_h · √d_h) · entropy_h` per head; else uses
  scalar `self.kappa · √d_h_canon`.
- `transformer/train.py`: removed spurious `/dim_scale` from entropy term; the
  per-component-normalized form is `M_beta · κ · (β·log β).sum().mean()` since
  `τ_eff · entropy / √K = κ · entropy`.

## Second-pass correction: attached β at all 4 E-step entropy sites

After the first pass committed, a unit test (`tests/test_entropy_envelope.py`)
swept 6 (λ_soft, β-attach) configurations and found that the detached-β entropy
combined with the existing `λ_soft = 1` default gave the **wrong gradient** —
mismatched the manuscript envelope-theorem prediction by ~10× the signal in
||∂F/∂μ||, ||∂F/∂Σ||, and |∂F/∂κ|.

**Root cause.** The `λ_soft · (β · KL.detach()).sum()` term contributes
`−τ⁻¹ · Cov_β(KL, ∇KL)` to the gradient — exactly the manuscript line 1261
"entropy-suppressed surrogate offset". With β detached in the entropy term,
nothing cancels it. Two consistent configurations exist:

- `λ_soft = 1` AND attached β in entropy: autograd-through-softmax produces the
  canceling `+τ⁻¹ · Cov`. Net gradient matches manuscript ∇F_red at softmax β.
- `λ_soft = 0` AND detached β in entropy: pure envelope-clean form; no off-
  equilibrium correction.

Chose option 1 (attached β) because it is robust to any user `λ_soft` setting
and is the literal autograd implementation of the manuscript F functional.

**Files changed (second pass).**

- `transformer/vfe/e_step.py`: `beta_phi.detach().clamp(...)` → `beta_phi.clamp(...)`.
- `transformer/core/variational_ffn.py` three entropy sites: same `.detach()` removal.
- `tests/test_entropy_envelope.py` (new): asserts production config matches
  envelope-theorem prediction to tolerance 1e-10.

**Verification.** `python tests/test_entropy_envelope.py` reports
`λ_soft=1, β.attached (PROD)` MATCH at 5.86e-19 / 2.31e-18 / 8.67e-19, hard
assertion PASS.
