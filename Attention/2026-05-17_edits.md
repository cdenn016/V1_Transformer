### Fix 1 — `variational_ffn.py::_finalize_e_step`

The audit trimmed `beta_current` and `beta_heads` from the signature
on the theory that they had become dead. They had not: the body's
`_last_beta` write block immediately after the alpha/omega writes
still consults both — preferring `torch.stack(beta_heads, dim=1)`
when the per-iteration head list is non-empty, otherwise falling
back to the scalar `beta_current`.

Surgery:

- Restored both parameters on the signature
  (`beta_current: Optional[torch.Tensor]`,
  `beta_heads: Optional[list]`).
- Updated the docstring's `Writes:` line to include `_last_beta`
  alongside `_last_alpha_i` and `_last_omega`.
- Updated the single call site at line 2668 to pass
  `beta_current` and `beta_heads`, which are local variables in
  the calling scope (`_forward_vfe_body`) initialised at lines
  2467–2468 and refreshed inside the iteration loop.

Runtime trace before the fix:

```

  at variational_ffn.py:1450 in _finalize_e_step
```

### Fix 2 — `model.py::forward_with_attention`

The implicit_em deletion removed three contiguous lines that lived
together as a unit immediately before `final_norm`:

```python
last_block = self.transformer.blocks[-1]
implicit_mu_scale = getattr(last_block.ffn, '_last_implicit_mu_scale', None)
implicit_sigma_scale = getattr(last_block.ffn, '_last_implicit_sigma_scale', None)
```

The first line set up `last_block` for the implicit_em rescaling
block. Several lines further down, the adaptive-`alpha_i` retrieval
block also read `last_block`:

```python
alpha_i = getattr(last_block.ffn, '_last_alpha_i', None)
```

That second read was not part of the implicit_em cluster but was
satisfied by the same local variable. Removing the cluster
orphaned the read.

Surgery:

- Added `last_block = self.transformer.blocks[-1]` immediately
  before the `alpha_i = getattr(...)` line. Single statement,
  scoped to the same block of code, so no semantic drift.

Runtime trace before the fix:

```
NameError: name 'last_block' is not defined
  at model.py:1305 in forward_with_attention
```

### Lesson

The audit's verification pipeline (`py_compile` + import smoke)
catches missing imports and syntax errors but does not exercise
function bodies. For deletion passes that remove local variables,
the appropriate check is either:

1. Search-then-trim — grep the variable name in the surviving file
   before deleting its definition; trim only if zero references
   remain.
2. Function-call smoke — actually invoke the affected forward
   path with a tiny config before declaring the audit verified.

Both regressions would have been caught by (1).

## Manuscript fix — "message" terminology in `GL(K)_attention.tex`

The user flagged that `GL(K)_attention.tex` calls "the message" two
different things. Three parallel investigation agents
(general-purpose) plus an independent verifier confirmed the flag and
extended it to **four** distinct mathematical objects sharing the
words "message" or the symbol `$m_i$`:

1. Aggregation / posterior mean
   `$m_i = \sum_j \beta_{ij}\,\Omega_{ij}\mu_j$` —
   lines 999, 1223, 1243, 1253, 1579, 1790.
2. Euclidean gradient term
   `$\sum_j (\beta_{ij}/\sigma^2)(\Omega\Omega^\top)^{-1}(\mu_i-\Omega\mu_j)$` —
   lines 1353-1367, called "attention-weighted message aggregation"
   at line 1369.
3. Per-edge residual `$e_{ij} \equiv \mu_i - \Omega_{ij}\mu_j$` —
   line 1813, called "The message $e_{ij}$".
4. Model latent in `$\mathcal{E}_p$`,
   `$m_i \in \mathbb{R}^{K_p}$` — lines 583, 593, 604, 881.
   Unrelated to attention.

The verifier additionally found that the natural-gradient reduction
at line 1399 was overclaimed: Fisher preconditioning with
`$\Sigma_i = \sigma^2 I$` cancels only the scalar `$\sigma^{-2}$` —
the boxed update at 1379-1397 still contains
`$(\Omega\Omega^\top)^{-1}(\mu_i - \Omega\mu_j)$`. Full reduction to
the value aggregation `$\sum_j \alpha_{ij} V_j$` requires Limit~2
(absorption of `$(\Omega\Omega^\top)^{-1}$` into
`$W_Q W_K^\top$`) and Limit~3 (constant gauge `$\Omega$` absorbed
into `$W_V$`).

### Fixes applied

1. **Aggregation symbol rename** `$m_i \to \hat\mu_i$` at lines 999,
   1223, 1243, 1253, 1579, 1790. Mnemonic preserved: `$\hat\mu_i$`
   is the posterior-mean estimator of `$\mu_i$` under mixture
   responsibilities, matching the manuscript's own derivation at
   line 1220. The model-latent `$m_i$` (lines 583/593/604/881)
   is unchanged, preserving the q↔k, p↔m duality.

2. **Premature intro rewrite (line 996-1002)**. Replaced
   "The message (update) received by agent $i$ is then" with a
   forward-pointing summary that defers to §Value Aggregation:
   "The value-aggregation output received by agent $i$ is the
   posterior mean of the mixture variational model under
   responsibilities $\beta_{ij}$ (derived formally in
   Section~\ref{sec:value_aggregation})". The misleading "(update)"
   gloss is dropped — `$\hat\mu_i$` is an output, not a parameter
   update.

3. **Bridge sentence at line 1369**. Recast the Term-A clause to
   make it explicit that the gradient term is the gradient of the
   alignment energy whose Fisher-preconditioned step direction
   targets `$\hat\mu_i$` of `eq:glk_mixture_aggregation`. The
   gradient and the aggregation are now connected in the text
   rather than left as a name collision.

4. **Tightened limit chain at line 1399**. Replaced
   "alignment term $\sum_j \beta_{ij}\,\Omega\mu_j$ maps directly to
   the value aggregation $\sum_j \alpha_{ij} V_j$ without spurious
   $\sigma^{-2}$ factors" with an explicit three-step reduction:
   (i) Fisher preconditioning cancels the scalar `$\sigma^{-2}$`;
   (ii) absorbing `$(\Omega\Omega^\top)^{-1}$` into `$W_Q W_K^\top$`
   (Limit~2 of Section~\ref{sec:transformer_limit}); (iii)
   absorbing the constant gauge `$\Omega$` into `$W_V$` (Limit~3).
   Added a clarifying tail: the E-step algorithm applies Fisher
   preconditioning but does not impose the projection-absorption
   limits.

5. **Per-edge residual rename (line 1805, 1813, 1820)**.
   - 1805: "the message contribution of neighbor $j$" → "the
     per-edge contribution of neighbor $j$".
   - 1813: "The message $e_{ij}$" → "The residual $e_{ij}$";
     "the message magnitude" → "the residual magnitude".
   - 1820: "Each message's contribution" → "Each per-edge
     contribution".

6. **Added two missing labels**.
   - `\subsection{Connection to Standard Transformers}` at line
     1004 now carries `\label{sec:transformer_limit}` (referenced
     in Edit 4).
   - `\subsubsection{Value Aggregation}` at line 1218 now carries
     `\label{sec:value_aggregation}` (referenced in Edit 2).
