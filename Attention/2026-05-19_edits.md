# Cross-head coupling ported to `/vfe`

User asked: "investigate whether cross_coupling feature is available in /vfe
implementation ŌĆ” build it out plug any metrics and visualizations". Investigation
confirmed the feature was legacy-only (`transformer/core/model.py` +
`transformer/core/block_equivariant_mixer.py`); shared math kernels lived in
`math_utils/generators.py` (`generate_glK_cross_head_generators`,
`merge_coupled_heads`, `reorder_cross_head_generators`, `_dedup_cross_couplings`).
This pass ports it into the `/vfe` package end-to-end, covering the full +
non-flat + full-cov scope per the user's selection.

## What `cross_couplings` is

Each `(a, b)` entry adds `d_head┬▓` off-diagonal generators spanning head-a's
subspace into head-b's. The resulting `╬® = exp(Žå┬ĘG)` becomes non-block-diagonal
across the coupled heads, so gauge transport itself moves information between
them ŌĆö a different mechanism from the value-level `VFEHeadMixer`. Coupled heads
are merged via union-find into super-blocks; the K-axis is permuted so super-
blocks are contiguous.

## Files changed

`transformer/vfe/config.py`

- `cross_couplings: List[Tuple[int,int]]`, `auto_close_cross_head_basis: bool`,
  `validate_cross_head_closure: bool` fields added.
- `__post_init__` validates: GLK-only, single-type multihead, `n_heads >= 2`,
  rejects self-coupling, out-of-range head indices, malformed pairs; silently
  dedups exact duplicates (preserves directional `(a,b)` vs `(b,a)`).
- `n_gen` updated to add `|cross_couplings| ┬Ę d_head┬▓` to the diagonal count.

  `super_block_head_groups`, `effective_block_dims` (returns
  `super_block_dims` when cross-coupled, else `irrep_dims`). Lazily computed
  via `merge_coupled_heads`; results cached on the instance.

- `_build_generators` dispatches to `generate_glK_cross_head_generators` ŌåÆ
  `merge_coupled_heads` ŌåÆ `reorder_cross_head_generators` when
