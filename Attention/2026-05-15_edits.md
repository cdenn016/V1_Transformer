# Edits — 2026-05-15

## Summary

Deep audit of `em_mode='ift_phi'` with `skip_attention=True`, verification of
`learnable_head_kappa` wiring into the VFE FFN, and removal of two now-redundant
EM modes (`'straight_through'` and `'implicit_ift'`). Backward-compat
`infer_em_mode()` shim removed. New default `em_mode` is `'ift_phi'`.

## Audit findings

### 1. `ift_phi + skip_attention=True` is fully wired and mathematically pure

Verified by three parallel investigation agents and a verifier agent.

- The FFN's E-step computes β at every iteration: `compute_attention_weights`
  call at `transformer/core/variational_ffn.py:1035`. The full per-head
  attention-entropy term `κ * √d_h * (β·log β).sum()` is added to
  `alignment_loss` at line 1067 (default `include_attention_entropy=True`
  branch). Under `em_mode='ift_phi'` the `exact_phi_grad` flag is True, so the
  `_detach_beliefs` check at line 1271 (`= not (amortized_inference and
  exact_phi_grad)`) evaluates to False, keeping `mu`, `sigma`, and `phi` in
  the autograd graph through the entire E-step iteration sequence.
- `transformer/core/block_config.py:__post_init__` validates that
  `skip_attention=True` is compatible with `ift_phi`; it warns on the
  detaching modes (`em_phi_p`, `em_phi_q`) which would silently freeze
  `sigma_embed` and `phi_embed` at initialization because the attention
  sublayer (the only remaining autograd path back to those embeddings under
  detached modes) is disabled.
- End-to-end smoke test: single-block GaugeTransformerLM with
  `em_mode='ift_phi'`, `skip_attention=True`, two E-step iterations,
  diagonal covariance, forward + backward on random inputs. Gradient norms:
  `mu_embed.weight` ≈ 1.82, `log_sigma_diag` ≈ 0.089, `phi_embed.weight`
  ≈ 0.354. All three embedding parameters receive non-zero gradients,
  confirming the wiring claim.

### 2. `learnable_head_kappa` is wired into the VFE FFN

The user asked about `learnable_kappa_beta`. No literal of that name exists.
The relevant mechanism is `learnable_head_kappa`, a per-head
`log_kappa_per_head` `nn.Parameter` owned by `transformer/core/attention.py`
(constructed at lines 1502–1517). The FFN accesses it through a shared
reference set at `transformer/core/blocks.py:638`
(`self.ffn.__dict__['_kappa_attn_ref'] = self.attention`) and reads it via
`_get_kappa_h()` at `transformer/core/variational_ffn.py:693–730`. The reader
returns the live Parameter tensor, so autograd flows back to the owning
Parameter even when `skip_attention=True` (the attention module is still
constructed in the block; only its forward call is skipped).

Sites in the FFN that consume `kappa_h`:
- Line 1035: alignment loss β computation
- Lines 1059–1067: per-head F evaluation with entropy term
- Lines 1232, 1309: closed-form / Picard κ scaling
- Lines 145, 152–165, 198, 263, 374 in `vfe_closed_form.py`: closed-form
  E-step β construction

Verdict: wired into the VFE FFN, not just the attention sublayer. The
gradient path survives `skip_attention=True`.

### 3. Correction to one investigator claim

At `variational_ffn.py:1266` the form `lambda_belief * (beta_phi_h.detach() *
kl_h).sum()` lives in the `include_attention_entropy=False` (entropy-suppressed
surrogate) branch, not the default path. The default branch at line 1265
keeps β attached for the F evaluation, then uses the canonical (β·KL + entropy)
sum. Recorded here so the surrogate path is not mistaken for the production
contract.

## Code changes

### `transformer/core/em_modes.py` — rewrite

- Deleted rows `'straight_through'` and `'implicit_ift'` from
  `EM_MODE_TABLE`.
- Deleted `infer_em_mode()` (backward-compat shim that mapped legacy flag
  tuples to mode strings).
- Updated module docstring. Table now contains exactly three modes:
  `'ift_phi'`, `'em_phi_q'`, `'em_phi_p'`.

### `transformer/core/block_config.py`

- Default `em_mode` changed from `'straight_through'` to `'ift_phi'` (line
  130, with updated comment).
- Removed `infer_em_mode` import and `_infer_em_mode` alias.

### `transformer/core/variational_ffn.py`

### Note: `detach_beta_m_step` is NOT dead

The original verifier flagged `detach_beta_m_step` as "deprecated test
file, legacy only". Closer reading shows it is an active flag used by
`transformer/train.py:273, 413` (controls whether β is detached when
computing the M-step belief-alignment loss; True is the correct EM
behavior). I only removed the one occurrence in
`scripts/verify_active_inference.py:88` which was a redundant config-dict
override (default is True everywhere else). The flag is otherwise
preserved across `transformer/training/config.py`,
`transformer/training/experiment_runner.py`,
`transformer/training/train_fast.py`, and `transformer/resume_training.py`.
