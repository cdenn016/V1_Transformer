# 2026-04-24 Edits

## scripts/run_ablation_suite.py — add `e_step_sigma_floor` sweep

Added a new single-parameter sweep to `SWEEPS` with explicit values
`[0.01, 0.05, 0.1, 0.25]` and baseline `0.01`. Motivation: 0.01 caps
E-step precision at 100, which can drive overconfidence in the q
posterior; the right value is coupled to K, data statistics, and the
adaptive α posterior, so it needs an empirical sweep.

The key `e_step_sigma_floor` is already wired through `BlockConfig` and
read by `transformer/core/blocks.py` (lines 512, 560, 644, 1097) and the
norm constructors, so dict-level injection via `BASELINE_CONFIG` reaches
runtime through `run_single_experiment` without any further plumbing.

Appended `'e_step_sigma_floor'` to `SWEEP_ORDER` immediately after
`'M_alpha'` so it runs in the currently-active block.
