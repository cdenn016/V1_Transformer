# Edits — 2026-04-14

## Fix: Implicit-EM alpha + Full-Covariance SPD bugs

Three bugs where adaptive (per-dimension, learnable) alpha interacts incorrectly with implicit-EM gradient scaling and the full-covariance closed-form E-step.

### Bug 1: `_finalize_e_step` uses `self.alpha` instead of `alpha_effective`

**File:** `transformer/core/variational_ffn.py`, line 1341

`_finalize_e_step` computed implicit-EM scales using the fixed `self.alpha` scalar instead of the passed-in `alpha_effective` (which is `(B, N, K)` when `learnable_alpha=True`). The IFT backward correction was therefore inconsistent with the forward E-step fixed point whenever alpha was token- or dimension-dependent.

**Fix:** `_alpha_for_scale = self.alpha` → `_alpha_for_scale = alpha_effective`

### Bug 2: Full-covariance closed-form breaks SPD with per-dimension alpha

**File:** `transformer/core/vfe_closed_form.py`, lines 294-299, 321-325

When `alpha_h` has shape `(B, N, d_h)`, the code formed `A_prior_h = alpha_h.unsqueeze(-1) * Sigma_p_inv_h`, which is a row-wise broadcast producing `diag(α) @ Σ_p⁻¹` — a non-symmetric matrix. The subsequent Cholesky factorization of `A_h` is invalid on non-symmetric input. Same issue for `Sigma_star_h = entropy_scale * A_inv_h`.

**Fix:** Replaced row-scaling with symmetric sandwich products:
- `A_prior_h = diag(√α) @ Σ_p⁻¹ @ diag(√α)` — preserves SPD structure
- `Σ*_h = diag(√(α+λ)) @ A⁻¹ @ diag(√(α+λ))` — preserves symmetry

### Bug 3: `ImplicitEMGradientSigma` row-scales full-cov gradients

**File:** `transformer/core/vfe_implicit_em.py`, lines 107-112

For full-covariance gradients `(B, N, K, K)`, the backward pass did `scale.unsqueeze(-1) * grad_output` which row-scales the gradient matrix, breaking symmetry. This is not the correct IFT correction for full covariance.

**Fix:** Replaced with symmetric sandwich: `diag(√s) @ grad @ diag(√s)^T`. Diagonal path unchanged (element-wise scaling is already symmetric for vectors).
