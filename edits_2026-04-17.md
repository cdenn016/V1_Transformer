# Edits — 2026-04-17

## Foundation for gauge-covariant numerical conditioning

Pure additions — no behavior change until callers opt in.

### New file: `transformer/core/gauge_ridge.py`

Two helpers for replacing `eps * torch.eye(K)` at covariance-regularization sites with a gauge-covariant ridge:

- `gauge_covariant_eye(exp_phi, eps, normalize=False)` — returns `eps * (g g^T)` built from the local frame `g = exp(phi)`. Under a frame change `g → h g` this transforms exactly as `h (g g^T) h^T`, matching how covariances transform under the sandwich product. `normalize=True` rescales so `tr(g g^T) == K`, but this breaks exact covariance under non-orthogonal `h` (trace is not gauge-invariant); kept as an opt-in with the tradeoff documented.
- `make_ridge(K, eps, *, exp_phi, use_covariant, ...)` — unified entry point. Collapses to `eps * torch.eye(K)` when `use_covariant=False` or when the caller has no frame, so flag-off numerics are bitwise identical to the pre-change code path.

### Config flag: `transformer/core/block_config.py`

Added `BlockConfig.gauge_covariant_ridge: bool = False` (field + `from_config()` wiring). Documented inline. Default `False` preserves current numerics; call sites that opt in will gate on this flag.

### Tests: `tests/transformer/test_gauge_ridge.py`

15 new tests covering:
- Exact sandwich covariance of the ridge under random `GL(K)` frame changes (K = 3, 8, 16).
- `Σ + ridge` transforms as `Σ`.
- Documents the bug: `ε I` ridge fails the same invariance by `||(h h^T − I) ε I||_F`.
- Identity frame reproduces `ε I` exactly.
- Batch-shape broadcasting for `(B, N, K, K)` tensors.
- `make_ridge` fallback paths (flag off, frame missing).
- KL divergence invariance: with the covariant ridge, `KL(q||p)` is gauge-invariant to `1e-8`; with `ε I` it drifts by `> 1e-7`. This is the proof-of-value test.

### Bucket-A wiring (follow-up pass)

Threaded the covariant ridge through the Bucket-A sites. All changes are backward-compatible: `exp_phi=None` preserves original `ε · I` numerics bit-for-bit.

**Core math insight corrected during implementation**: initial plan proposed trace-normalizing `M_loc = g g^T * K / tr(g g^T)`. Flipped default to `normalize=False` after catching that `tr(h M h^T) ≠ tr(M)` under non-orthogonal `h` — normalization breaks exact gauge covariance. The un-normalized `ε · (g g^T)` is exactly covariant; magnitude stability has to come from bounding `||φ||` via existing `phi_max_norm` / `phi_project_slk` instead.

**Simplified `make_ridge` API**: dropped the `use_covariant` bool. `exp_phi=None` alone controls the fallback. Call-site pattern becomes `make_ridge(K, eps, exp_phi=frame_or_None, ...)`.
