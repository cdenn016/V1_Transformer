# 2026-04-19 Edits — Surgical stride + random_offset_per_epoch rollout

## Context

Re-introduces `stride`, `random_offset_per_epoch`, `eval_stride`, and
`stride_base_seed` to the dataset/dataloader stack after the 2026-04-18 rollout
was reverted. The prior rollout produced a train-PPL drift at step 100 (781.7
vs 710.3 baseline) even with the feature disabled at its defaults. Post-mortem
traced it to a `num_sequences` formula rewrite — legacy `max(1, N - T)` to
strided `(N - T) // stride + 1` — that changed the length of the permutation
produced by `torch.randperm` inside PyTorch's `RandomSampler`. Same model
weights, different batch order, diverging loss trajectory.

The non-negotiable invariant this time: when `stride=None` (default), every
line of code executed is byte-identical to the pre-feature legacy path. No
equivalent-math rewrites, no "identical up to" claims — the literal original
expression on the default path. The user confirmed the `710.3 @ step=100`
baseline is reproducible on the current machine before we started.

All nine commits land with the full test suite green (795 passed: 31 new
stride tests + 764 pre-existing). No existing tests required modification.

## Scope of changes

### Config layer (`transformer/training/config.py`)

Added four optional fields to the `TrainingConfig` dataclass after
`max_seq_len`:

- `stride: Optional[int] = None` — window stride. `None` preserves legacy
  formula. `int` produces windows at offset + k·stride.
- `random_offset_per_epoch: bool = False` — when `stride > 1`, draw a
  per-epoch offset in `[0, stride)` from an isolated `torch.Generator`
  seeded by `stride_base_seed + epoch`. Forced `False` automatically when
  `stride is None` or `stride == 1`.
- `eval_stride: Optional[int] = None` — separate stride for val/test
  loaders. Default `None` preserves dense eval-time coverage.
- `stride_base_seed: int = 0` — seed for the isolated offset generator.

All four threaded through `TrainingConfig` primarily for introspection and
forward-compatibility. The trainer does not branch on them; all actual
windowing logic lives in the dataset classes.

### Dataset layer (`transformer/data/datasets.py`)

Four dataset classes — `WikiText2Dataset` (transformers/BPE),
`WikiText2TiktokenDataset` (primary tiktoken path), `WikiText2CharDataset`
(char-level), `WikiText2ByteDataset` (byte-level) — each received:

1. **New `__init__` kwargs**: `stride`, `random_offset_per_epoch`,
   `base_epoch_seed`, all with legacy-preserving defaults.
2. **Storage**: `self.stride`, `self.random_offset_per_epoch` (auto-gated off
   when stride is None or ≤1), `self.base_epoch_seed`, `self._epoch_offset=0`.
3. **`num_sequences` branch**:
   ```python
   if self.stride is None:
       # LEGACY PATH — DO NOT MODIFY.
       self.num_sequences = max(1, len(self.tokens) - self.max_seq_len)
   else:
       N, T, S = len(self.tokens), self.max_seq_len, int(self.stride)
       k_max = max(0, (N - T - S) // S)
       self.num_sequences = max(1, k_max + 1)
   ```
   The `(N - T - S) // S` reserves `(S - 1)` tokens at the tail so every
   offset in `[0, S)` fits a full unpadded window. Consequence: `stride=1`
   gives `N - T` sequences (matches legacy count); `stride=T` gives
   `(N - 2T)//T + 1` full non-overlapping windows with room to shift.
4. **`__getitem__` branch**:
   ```python
   if self.stride is None:
       start_idx = idx  # LEGACY PATH
   else:
       start_idx = self._epoch_offset + idx * int(self.stride)
   ```
5. **`set_epoch(epoch)` method** — identical body on each class. Early-returns
   when `random_offset_per_epoch` is False. Uses an isolated
   `torch.Generator()` (verified to not mutate the global RNG state).

### DataLoader factories (`transformer/data/datasets.py`)

Both `create_dataloaders` and `create_char_dataloaders` gained the same four
kwargs. Train datasets receive `stride=stride, random_offset_per_epoch=...,
base_epoch_seed=...`; val/test datasets receive `stride=eval_stride,
random_offset_per_epoch=False, base_epoch_seed=0`.

Added a hard safety assertion at the top of each factory: when any stride is
active, `num_workers > 0` raises `ValueError`. PyTorch pickles datasets to
workers once under `persistent_workers=True`, so per-epoch offset updates on
the main-process instance would not reach worker copies. This was the top
verifier-flagged bug. As defense in depth, `persistent_workers` is also
forced to `False` when stride is active.

### Call-site threading (`experiment_runner.py`, `resume_training.py`)

Threaded `config.get('stride', None)` and the three siblings through:

- `_create_dataloaders` (both char and BPE branches) at
  `transformer/training/experiment_runner.py:1707`.
- `TrainingConfig(...)` construction at
  `transformer/training/experiment_runner.py:1929` (primary FastTrainer build).
- `TrainingConfig(...)` construction at
  `transformer/resume_training.py:449` (resume path).
- `create_dataloaders` / `create_char_dataloaders` calls at
  `transformer/resume_training.py:352, 360` so resumed runs can opt into
  stride via the config dict.

The three preset helpers in `config.py` (`get_standard_config`,
`get_vfe_dynamic_config`, `get_pure_fep_config`) did not need edits — they
use the dataclass default (all `None`/`False`/`0`).

### Training-loop `set_epoch` hooks

Module-level `_maybe_set_epoch(dataset, epoch)` helper in
`experiment_runner.py` (which `getattr`-guards dataset.set_epoch for
datasets that don't implement the method). Hooked at five sites:

1. `FastTrainer.train` main loop — init at 1208, StopIteration at 1281.
2. `experiment_runner.py` pure_fep loop — init at 2687, StopIteration at 2721
   (accumulated micro-batch path) AND 2748 (single-step path).
3. `transformer/training/train_fast.py` — init at 374, StopIteration at 393.
4. `transformer/vfe/trainer.py` — init at 569, StopIteration at 584.

`train_fast.py` and `vfe/trainer.py` use inline `getattr/callable` guards
rather than importing the helper (avoids cross-package coupling for a
two-line check).

At each epoch boundary we increment `_epoch_idx` and call
`_maybe_set_epoch`. Under `stride=None` the method is a no-op; under
`stride>1` and `random_offset_per_epoch=True`, it updates the window offset
via the isolated generator.

### Tests (`tests/transformer/test_dataset_stride.py`, new, 31 tests)

- `TestNumSequencesTable` (6): locks the `num_sequences` formula at known
  values across `(N, T, stride)` triples including degenerate `N==T`.
- `TestSetEpochContract` (16, parametrized over 4 dataset classes): proves
  each class's `set_epoch` is a no-op under the default flag, consumes ZERO
  global torch RNG when the flag is on, produces deterministic offsets given
  fixed `base_epoch_seed`, and varies across epochs.
- `TestStrideNoneIdentity` (2): synthetic-dataset end-to-end check that
  `stride=None` indexing matches the legacy formula.
- `TestStrideTReproducibility` (2): same base_seed reproduces the full offset
  stream; `stride=T, offset=0` produces expected contiguous windows.
- `TestWorstCaseOffsetBounds` (1): last idx × last offset stays inside the
  token tensor (guarantees no silent padding).
- `TestEpochZeroUnderOptionB` (1): different `base_epoch_seed` at epoch 0
  produces different offsets — confirms Option B (no epoch-0 special case).
- `TestFactoryAssertion` (3): both factories raise `ValueError` on
  `stride + num_workers>0` before constructing any dataset.

## Design decisions locked in (from the planning pass)

- **Option B for epoch-0**: always draw the offset from the generator, even
  at epoch 0. `stride_base_seed` controls the first-epoch trajectory; cross-
  seed runs differ immediately. Rejected Option A ("force offset=0 at epoch
  0") because it would artificially correlate first-epoch data across seeds
  and render `stride_base_seed` inert at epoch 0.
- **Tail reservation `(S-1)`**: guarantees every offset in `[0, S)` maps to a
  full unpadded window. Wastes up to `S-1` tokens at the corpus end, but
  eliminates silent-padding data entering batches.
- **Factory assertion over silent fallback**: `stride + num_workers>0` hard-
  raises rather than coercing `num_workers=0`. Explicit failure is better
  than silent behavior change for a user who expected workers.
- **`stride=1` is NOT bit-identical to `stride=None`**: `stride=1` routes
  through the strided branch (different code path, same `__getitem__` output
  at offset 0 but different `if` evaluation). Use `stride=None` for bit-
  identical reproduction of pre-feature trajectories. Documented in the
  `TrainingConfig` field docstring.

## Usage after landing

Default TrainingConfig is unchanged — `stride=None` preserves the pre-feature
trajectory bit-identically. Users opt in via the config dict:

```python
# Non-overlapping windows with per-epoch shift (MPT/Llama-style).
config['stride'] = config['max_seq_len']          # e.g. 64
config['random_offset_per_epoch'] = True
config['stride_base_seed'] = 6                    # reproducibility
# Keep eval dense (full corpus coverage at test PPL).
config['eval_stride'] = None
```

`stride + num_workers>0` is rejected by the factory. Use `num_workers=0`
when stride is active (already the repo default on Windows).

## Second change set — `phi_project_slk=True` restored under `use_prior_bank=True`

### Context

User report: `phi_project_slk=True` produced no visible effect when `use_prior_bank=True`. Three Explore agents and one independent verifier confirmed that `VFEPriorBank._apply_phi_det_control` runs as documented (`transformer/vfe/prior_bank.py:239-271`, called at encode `:287` and decode `:333`), but the positional encoder immediately undoes the projection. `VFEPositionalEncoding.pos_phi` was an unconstrained `nn.Parameter` of shape `(max_seq_len, n_gen)`. Since `tr([X,Y])=0`, every term in the BCH series past first order is traceless, so `tr(BCH(X,Y)) = tr(X)+tr(Y)` exactly. A traceless token phi composed with a non-traceless `pos_phi` yielded a composed phi with per-block trace equal to `tr(pos_phi) ≠ 0`. The same defect existed in the legacy `GaugePositionalEncoding` (`transformer/core/embeddings.py`).

### Fix

Re-parameterize the positional Lie-algebra parameter onto the traceless subalgebra `sl(K_1) ⊕ ... ⊕ sl(K_H)` whenever `phi_project_slk=True`. Chosen over output-side projection because it eliminates gradient flow along the per-block trace directions structurally rather than zeroing it each forward.

New shared helper `build_slk_basis(generators, irrep_dims) -> (V_blocks, P)` in `transformer/core/vfe_utils.py`: returns per-block trace vectors and an orthonormal basis `P` of the traceless complement (via `eigh` of `I − Σ_h V̂_h V̂_h^T`). Both positional encoders store `pos_phi_free` of shape `(max_seq_len, n_gen − H)` and expand as `pos_phi_free @ P.T` at forward time. `V_blocks @ P = 0` by construction, so the expansion lies in sl(K) for every representable value of `pos_phi_free`.

|---|---|

| `transformer/vfe/positional.py` | `VFEPositionalEncoding.__init__` accepts `irrep_dims`; replaced `pos_phi` with `pos_phi_free + pos_phi_basis` when `cfg.phi_project_slk`. `forward()` expands through the basis. |

### Verification

- New test file: 9/9 passed.
- Regression sweep: `test_vfe_package.py` (31) + `test_embeddings.py` (9) + `test_prior_bank.py` (17) — 57/57 passed.
- `transformer/baselines/hybrid_gauge_transformer.py:376` was left unchanged; the new `GaugePositionalEncoding` kwargs are optional and default to no-op, so baseline behavior is preserved bit-identically when `phi_project_slk=False`.

### Backward compatibility

Old checkpoints written before this change store `pos_phi` (full `n_gen` width). When reloading with `phi_project_slk=True` the parameter name and shape both change to `pos_phi_free` (width `n_gen − H`), so state-dict loading of pre-existing checkpoints under the new flag will fail at the shape mismatch. Load with `phi_project_slk=False` (unchanged behavior) or retrain.

### Root-cause trace (for future readers)

The projection path at `prior_bank.py:287` and `:333` was never the bug. The non-traceless drift entered at `positional.py:50-52` / `embeddings.py:996` via an unconstrained `nn.Parameter`. Because BCH preserves additive traces exactly, projecting either side alone is insufficient — both sides of the composition must be traceless.

## Third change set — Real Yang-Mills curvature + skip-subplot when zero

### Context

End-of-publication-run emitted a matplotlib `UserWarning` from
`gauge_geometry_plots.py:158` (`Data has no positive values, and therefore
cannot be log-scaled`). Three Explore agents plus an independent verifier
traced it to `publication_metrics.py:1701` hardcoding
`energies=[0.0] * len(steps)` with comment "YM=0 for flat transport", which
was being fed unconditionally to a log-scaled axis. The comment was also
misleading: the framework's bilinear transport `Omega_ij = exp(phi_i) exp(-phi_j)`
is cocycle-flat by construction and yields YM = 0 exactly up to float noise
regardless of training state — not "for flat transport" but always.

User requested: implement real YM curvature (zero-dimensional base manifold,
so holonomy-based, not continuum `F_{μν}`) and skip the subplot when
energies are zero. Initial wiring used the existing `logm` path and ran 5+
minutes per diagnostic at publication scale (8000 complex eigendecompositions
of 128×128 matrices). Refactored to a Wilson/lattice action default which
drops cost to ~20 ms per call on realistic VFE_dynamic structure.

### Math

Base manifold is discrete (tokens are points), so curvature is loop
holonomy on the attention graph, not a continuous 2-form. For a triangle
`(i,j,k)` the holonomy is `C_{ijk} = Omega_{ij} · Omega_{jk} · Omega_{ki}`
and the Yang-Mills action admits two equivalent discrete forms:

- **Wilson** (default): `E_YM = (1/2) Σ_t ||C_t − I||_F^2`. No
  eigendecomposition required; `O(T K^2)` per call. This is the standard
  lattice gauge-theory action.
- **logm** (opt-in): `E_YM = (1/2) Σ_t ||log(C_t)||_F^2`. Requires complex
  `torch.linalg.eig` on every sampled triple; `O(T K^3)` and dominates
  runtime at publication K.

For bilinear (phi-path) transport the cocycle identity gives
`Omega_ij · Omega_jk · Omega_ki = omega_i · omega_j^{-1} · omega_j · omega_k^{-1} · omega_k · omega_i^{-1} = I`
exactly. Reported YM energy is therefore zero up to float noise. A
non-trivial value requires a genuinely non-flat edge correction
(e.g. `connection.py` MLP mode with learned `delta_ij`).

### Design decisions

- **Wilson action as default**: on a zero-dimensional base manifold it is
  the standard lattice gauge-theory action, and avoids the `O(K^3)` complex
  eig that was causing the 5-minute diagnostic stall. `logm` remains
  available as an opt-in for the continuum correspondence.
- **`batch_limit=1` default**: matches the existing
  `compute_gauge_orbit_dimension` convention (first batch element only).
  Pass `batch_limit=None` to process the full batch.
- **Analytical inverse via `stable_matrix_exp_pair`**: solve-based
  inversion accumulates `O(K)` roundoff that surfaced above the physical
  noise floor at K=128, pushing the computed YM above the skip threshold
  on purely flat transport. Using `exp(-phi·G)` directly drops the noise
  floor by ~4 orders of magnitude at publication scale.
- **`ym_tol = 1e-4` skip threshold in the plotter**: above the observed
  float-noise floor for bilinear transport (~2e-8 realistic,
  ~3e-4 pathological test), below any physically meaningful non-flat YM
  which scales like `phi_scale^2 ≈ 2.5e-3`.

## Fourth change set — Close the EM_CONFIG leak: legacy PriorBank + gauge_fixed_priors=False

### Context

User reported that after the second change set, `det(Ω)` under their EM_CONFIG still matched the pre-fix distribution bit-identically (`mean=1.2366, median=1.0000, std=1.0300, min=0.0335, max=29.8822`). Three investigative agents plus an independent verifier traced the gap to two layers of `gauge_fixed_priors`-gated dead code that kept the projection silent in the user's configuration (`use_prior_bank=True`, `gauge_fixed_priors=False`, `phi_project_slk=True`, `irrep_spec=[('fund',2,10)]`, `em_mode='em_phi_p'`).

The second change set had fixed `VFEPositionalEncoding` and `GaugePositionalEncoding` to hold `pos_phi` on the traceless submanifold. What remained uncovered:

1. Legacy `transformer/core/prior_bank.py` had no `_apply_phi_det_control` at all — `phi_project_slk` was not a kwarg, no `_phi_trace_vec` buffer was registered, and the token phi flowed raw into downstream transport.
2. `transformer/core/model.py:949` passed `generators` to the bank only when `gauge_fixed_priors=True`. With `False` the bank could not build its trace-vector buffer even if the projection code existed.
3. `transformer/core/model.py:306` set `_embed_irrep_dims=None` unless `gauge_fixed_priors=True`, so the per-head block structure never reached the bank. Empirically this reduced `max|s_h|` from 0.152 only to 0.088 — the projection was collapsing two GL(10) blocks into one GL(20) and killing only the full-matrix trace, leaving per-head traces alive.

### Root-cause trace (for future readers)

All three change sets are variations on the same class of bug: a correctness feature (`phi_project_slk`) was plumbed through the top config and into one code path (`VFEPriorBank`, the retraction path, then the positional encoders in change set 2), but guarded elsewhere by `gauge_fixed_priors` — a separate concern about whether token priors share a base distribution or learn independently. The two flags are orthogonal in intent but were entangled at three gating sites. Fix: decouple them at every site where `gauge_fixed_priors` was used as a proxy for "bother computing block structure / expose generators / run det control".
