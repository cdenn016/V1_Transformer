## Fix: training_history.csv train_ppl using normalized CE

**Bug**: When `normalize_ce_by_dim=True`, the CE loss is divided by `sqrt(K)` for VFE-vs-CE weight balancing. The `training_history.csv` (generated by `PublicationMetrics`) was computing `train_ppl = exp(normalized_ce)` and `train_bpc = normalized_ce / ln(2)` — producing artificially low PPL (e.g., 4.5 instead of ~270). The experiment_runner's console log and TrainingHistoryTracker correctly used `loss/ce_raw` (un-normalized) for PPL, but this raw value was not passed through to PublicationMetrics.

**Root cause**: `experiment_runner.py:1610` passed `'ce_loss': metrics['train_loss_ce']` (which is `loss/ce`, the normalized value) to `pub_metrics.record_training_step()`. `publication_metrics.py:210-212` then computed `train_ppl = exp(ce_loss)` using this normalized value.

**Fix**:
- `experiment_runner.py:1137`: Added `'train_loss_ce_raw'` field carrying `loss/ce_raw` from `compute_free_energy_loss`
- `experiment_runner.py:1611`: Pass `'ce_loss_raw': metrics.get('train_loss_ce_raw', ...)` to pub_metrics
- `publication_metrics.py:212-213`: Use `ce_loss_raw` (falling back to `ce_loss`) for PPL and BPC computation

**Files modified**:
- `transformer/training/experiment_runner.py` — plumb `loss/ce_raw` through metrics dict and into pub_metrics call
- `transformer/analysis/publication_metrics.py` — prefer `ce_loss_raw` over `ce_loss` for PPL/BPC

## NaturalGradientOptimizer: 4 Implementation Fixes

Four fixes to `NaturalGradientOptimizer` addressing a correctness bug, numerical stability, and diagnostic gaps.

### Concern 1: Embedding heuristic gated on group name (correctness)

The shape heuristic `grad.dim() == 2 and grad.shape[-1] >= 4` caught any 2D parameter (attention weights, output projection, FFN params), not just embeddings. Non-embedding params got spurious Fisher blocks allocated and wrong natural-gradient updates.

**Fix**: Added class constant `_EMBEDDING_GROUPS = frozenset({'mu_embed', 'sigma_embed', 'phi_embed', 'omega_embed', 'sign_embed'})`. The dispatch now checks `gname in self._EMBEDDING_GROUPS` as the primary gate, with the shape check retained as a secondary guard. Memory estimation in `create_optimizer()` also updated to only count embedding groups. Unnamed groups default to SGD fallback.

### Concern 2: Fisher state always fp32 (numerical stability)

## Fix: Three Alpha-Handling Bugs in Implicit-EM and Full-Covariance Paths

Three mathematical bugs where per-dimension alpha broke symmetry/SPD constraints or used the wrong value.

### Bug 1: Implicit-EM uses fixed `self.alpha` instead of learnable `alpha_effective`

**File**: `transformer/core/variational_ffn.py`, line 1333

**Problem**: `_finalize_e_step()` receives `alpha_effective` as a parameter (which is the adaptive per-dimension alpha from `get_bayesian_alpha()` when `learnable_alpha=True`), but line 1333 ignores it and uses `self.alpha` (fixed scalar). The IFT scale factors are then inconsistent with the forward fixed point.

**Fix**: `_alpha_for_scale = alpha_effective if alpha_effective is not None else self.alpha`

### Bug 2: Full-covariance closed-form breaks SPD when alpha is per-dimension

**File**: `transformer/core/vfe_closed_form.py`, lines 294-295 and 321-323

**Problem**: `alpha_h.unsqueeze(-1) * Sigma_p_inv_h` where `alpha_h` is `(B,N,d_h)` and `Sigma_p_inv_h` is `(B,N,d_h,d_h)` performs row-scaling: element `[i,j] = α_i * Σ^{-1}_{ij}`. This is NOT symmetric (requires `α_i = α_j` for all i,j). The result is passed to Cholesky, which requires SPD input.

Same issue with `entropy_scale = (alpha_h + lambda_belief).unsqueeze(-1); Sigma_star = entropy_scale * A_inv` — row-scales a symmetric matrix, breaking symmetry.

**Fix**: Sandwich product `diag(√α) Σ^{-1}_p diag(√α)`, implemented as:
```python
sqrt_alpha = alpha_h.sqrt().unsqueeze(-1)  # (B, N, d_h, 1)
A_prior = sqrt_alpha * Sigma_p_inv * sqrt_alpha.transpose(-1, -2)
```
This preserves symmetry and SPD structure. Reduces to `α * Σ^{-1}_p` when α is scalar.

### Bug 3: ImplicitEMGradientSigma backward invalid for full covariance

**File**: `transformer/core/vfe_implicit_em.py`, lines 105-112

**Problem**: Scale is `(B,N,K)`. For full-cov gradients `(B,N,K,K)`, `.unsqueeze(-1)` gives `(B,N,K,1)`, which row-scales the gradient matrix, breaking symmetry. Element `[i,j]` becomes `s_i * grad_{ij}` instead of the symmetric `√s_i * grad_{ij} * √s_j`.

**Fix**: Sandwich product in backward: `√s @ grad @ √s^T`, implemented as:
```python
sqrt_s = scale.sqrt().unsqueeze(-1)  # (B, N, K, 1)
scaled_grad = sqrt_s * grad_output * sqrt_s.transpose(-1, -2)
```

## Full-Covariance Exact KL Decode Path for PriorBank

Added `full_cov_decode: bool = False` option to PriorBank. When enabled and sigma_q is (B,N,K,K), decode uses the exact Gaussian KL with full covariance matrices instead of extracting diagonals.

### Implementation

**File**: `transformer/core/prior_bank.py`

New method `_decode_full_cov()` uses the second-moment identity:

`tr(Sigma_p^{-1} Sigma_q) + mu_q^T Sigma_p^{-1} mu_q = tr(Sigma_p^{-1} M)`

where `M = Sigma_q + mu_q mu_q^T`. This fuses the trace and quadratic-in-mu_q terms into a single `(B*N, K^2) x (K^2, V)` matmul via reshape.

Cost: O(B*N*V*K^2) vs O(B*N*V*K) for the diagonal path. The K-fold slowdown is the price of off-diagonal fidelity.

When Sigma_p is diagonal (gauge_fixed_priors=False), the full-cov and diagonal paths produce identical logits (verified: max_diff < 2e-6). When Sigma_p is full (gauge_fixed_priors=True, diagonal_covariance=False), the paths disagree because `diag(Sigma_p^{-1}) != 1/diag(Sigma_p)` — the full path captures the correct inverse.

### Changes

- Added `full_cov_decode` parameter to `__init__` (default False, backward compatible)
- Added dispatch in `decode()`: when `full_cov_decode=True` and `sigma_q.dim() == 4`, calls `_decode_full_cov()`
- `_decode_full_cov()`: 90-line method handling Cholesky inversion of full Sigma_p, second-moment matmul, cross/bias terms, sigma_ce_scale, learnable_temperature

### Verification

- 581 passed, 24 skipped, 0 failures
- Smoke test: diagonal sigma_p agreement < 2e-6; full sigma_p shows expected divergence; gradients flow through off-diagonal sigma_q entries

## Fix: Direct-Omega Transport — Remove Unconditional Ridge, Preserve Exact Cocycle

The direct-Omega path unconditionally added a ridge `ε·I` (ε=1e-6) to omega before inverting, which breaks the cocycle identity `Ω_ij @ Ω_jk = Ω_ik` by O(ε) on every forward pass regardless of conditioning.

### Problem

```python
omega_reg = omega + 1e-6 * eye_K          # Always applied
omega_j_inv = torch.linalg.inv(omega_reg)  # Inverts REGULARIZED matrix
```

This means `Ω_ij = Ω_i @ (Ω_j + εI)^{-1}` instead of `Ω_i @ Ω_j^{-1}`. The cocycle identity requires `(Ω_j + εI)^{-1} @ (Ω_j + εI) = I`, but `(Ω_j + εI) ≠ Ω_j`, so composition through j introduces O(ε) error even when Ω_j is well-conditioned.

### Fix

**File**: `transformer/core/transport_ops.py`

Replaced unconditional ridge+inv with a three-tier inversion strategy:

1. **Primary**: `torch.linalg.solve(omega, I)` — LU with pivoting, ridge-free, cocycle exact to float precision
2. **Fallback 1**: Ridge ε=1e-6 + solve — only when omega is near-singular. Logs `omega_inv_ridge_fallback` via numerical monitor
3. **Fallback 2**: `torch.linalg.pinv(omega)` — last resort. Logs `omega_inv_pinv_fallback`

Same pattern applied to `omega_to_block_exp_pairs()` per-block inversion (`omega_blk_inv_ridge_fallback`, `omega_blk_inv_pinv_fallback`).

Updated docstring: cocycle is "exact to float precision" (not "automatic"), with explicit note that fallbacks break it.

- 43 transport + omega gradient tests pass (including cocycle identity tests)

## Add amortize_sigma and exact_phi_grad Flags

Two opt-in flags to expand the amortized inference gradient scope beyond the current mean-only default.

### Background

`amortized_inference=True` only flows gradients through prior means (mu_p), not covariances (sigma_p). The post-loop phi update (`_compute_phi_grad`) detaches beliefs, producing a semi-gradient. Both are intentional design choices but limit the amortization scope.

### New Flags

**`amortize_sigma: bool = False`**: When True (and `amortized_inference=True`), sigma_p stays attached in the E-step. Gradients flow back through prior covariances, enabling learned prior uncertainty through E-step dynamics. Default off preserves the sigma cascade prevention.

**`exact_phi_grad: bool = False`**: When True (and `amortized_inference=True`), `_compute_phi_grad` keeps mu_current and sigma_current attached. This gives the full total derivative dF/dphi through the E-step iteration graph (IFT-correct) instead of the semi-gradient with frozen beliefs.

## Principled EM Mode: `em_phi_mode` Flag

Added `em_phi_mode: str = 'amortized'` to `BlockConfig` with two opt-in EM modes that enforce a clean E-step/M-step boundary:

- **`'E_phi_q'`** (φ ∈ q): E-step optimizes (μ_q, Σ_q, φ_q). All three are detached at the EM boundary before returning to the transformer stack. M-step sees q* as frozen constants.
- **`'M_phi_p'`** (φ ∈ θ): E-step optimizes only (μ_q, Σ_q) with φ detached/frozen. M-step optimizes φ alongside priors/readout via backprop through the attention coupling KL term.

The default `'amortized'` preserves existing behavior (straight-through estimator).

### What each mode enforces

| Detachment site | `'amortized'` | `'E_phi_q'` | `'M_phi_p'` |
|----------------|---------------|-------------|-------------|
| mu_p (prior mean) | Attached | **Detached** | **Detached** |
| phi at E-step start | Attached | Attached | **Detached** |
| E-step output (mu, sigma) | Attached | **Detached** | **Detached** |
| E-step output (phi) | Attached | **Detached** | Already detached |
| Hierarchical cascade | Attached | **Detached** | **Detached** |
| Phi E-step evolution | User controls | User controls | **Forced off** |

### M-step gradient paths in EM modes

With E-step outputs detached, M-step gradients still reach model parameters:
- `mu_embed`: through ∂KL(q*||p)/∂mu_p = α(mu_p - mu_q*)/σ_p²
- `sigma_embed`: through ∂KL(q*||p)/∂sigma_p
- `W_out`: through ∂CE(W_out @ mu_q*, targets)/∂W_out
- `phi_embed` (M_phi_p): through attention sublayer transport Ω(φ) in β·KL
