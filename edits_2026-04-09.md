# Edits — 2026-04-09

## Added `min_lr_ratio` to ablation sweep

**Files modified:** `scripts/run_ablation_suite.py`

### Changes

1. **BASELINE_CONFIG** — Added explicit `'min_lr_ratio': 0.1` entry (previously fell through to `TrainingConfig` default via `getattr`). Makes the baseline value visible and overridable in the sweep infrastructure.

2. **SWEEPS dict** — Added `min_lr_ratio` sweep definition under new Tier 11 (LR schedule):
   - Range: `[0.0, 0.5, 0.05]` → 11 values (0.0, 0.05, 0.10, …, 0.50)
   - Baseline: 0.1
   - Description: "Minimum LR as fraction of peak (scheduler floor): effective_min = base_lr × ratio"

3. **SWEEP_ORDER** — Appended `'min_lr_ratio'` after `M_sigma_p_lr` in the active sweep list.

### Note

`min_lr` (absolute floor, default 3e-5) was **not** added — it is not read by the scheduler (`FastTrainer._create_scheduler` and `PureVFETransformer._lr_scale` only use `min_lr_ratio`). Sweeping it would have no effect on training.
