## Single/multi-run trajectory figure

### Why

The headline sample-efficiency claim for the gauge-VFE model is best
shown as a `val PPL vs tokens_seen` curve, optionally overlaid with a
matched-architecture standard-transformer baseline. The existing
`fig_trajectory_by_K` shows multiple K values from the scaling sweep but
not a single arbitrary run, and not in the inductive-bias / capacity
comparison framing. This script produces that figure from any one or
more run directories.

### Approach

`scripts/trajectory_plot.py` (NEW) — click-to-run, no CLI args. Top-of-
file `CONFIG` dict lists one or more runs to overlay; each entry pins
`run_dir`, `label`, `color` (or None for next from `PUB_CYCLE`), and
`linestyle`. Output: a 2-panel PNG/PDF (training PPL on the left,
validation PPL on the right), both on log-log axes, saved to
`publication_outputs/trajectory/`.

Reuses without modification:

- `transformer/analysis/scaling_ingest.py::load_trajectory` and
  `_safe_load_json` to read each run's `metrics.csv` and config and
  compute `tokens_seen = step * batch_size * seq_len` per row.
  Importantly, this reads `batch_size` from each run's own
  `experiment_config.json`, so overlaying runs with different batch
  sizes (e.g. K=40 at bs=32 vs K=100 at bs=16) collapses cleanly onto
  the tokens axis.
- `transformer/visualization/pub_style.py::set_pub_style`,
  `PUB_COLORS`, `PUB_CYCLE` for figure styling.

Per-run defaults: rolling-mean smoothing window=25 (configurable via
`smoothing_window`; set 0 to disable), log-log on both axes
(configurable via `x_log` / `y_log`).

## Methods.md gains a discussion paragraph

`transformer/analysis/scaling_report.py::write_methods_paragraph`
gained an optional `discussion: Optional[str] = None` parameter.
When supplied, the discussion text is appended after the methods
paragraph with a blank line separator; the banned-word linter applies
to the combined text, so Claude-isms stay out of either section.
The soft word-cap is bumped to 500 when discussion is supplied
(was 250 for methods alone).

`scripts/scaling_analysis_K.py::CONFIG` now carries a `discussion`
key with a wiki-ja contextualization paragraph (≈220 words):

- Notes that the K-sweep floor c ≈ 61 is an undertrained-convergence
  ceiling, not an architectural one — based on the user's
  observation that wiki-ja with ~1B tokens reaches test PPL ≈ 15-30
  at comparable K.
- States the cross-dataset / cross-tokenizer caveats (gpt2 BPE
  50,257 vocab vs cl100k_base 100,277 vocab; English vs Japanese).
- Contrasts the b ≈ −1 exponent with the Chinchilla-style
  cross-entropy exponent of ≈ 0.34 against total parameter count,
  noting that K is embed dim not param count and the gauge model
  has no learned QKV (so the K→N map is linear here, quadratic in
  a standard transformer).
