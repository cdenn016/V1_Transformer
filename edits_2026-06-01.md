# Edits — 2026-06-01

## Summary

Two pieces of work: (1) a deep multi-agent audit of the `ift_phi` +
`skip_attention=True` production path, and (2) a surgical refactor that hardcodes
`skip_attention=True` and removes the `IrrepMultiHeadAttention` sublayer from
`GaugeTransformerBlock`. The block is now always the pure-VFE form (the FFN E-step
computes its own β and handles all cross-position communication). The
`IrrepMultiHeadAttention` class and the `attention.py` module-level functions are
kept (used by the VFE loss, the hybrid baseline, and standalone tests).

The refactor is behavior-preserving for every `skip_attention=True` config: it is
gated by two equivalence harnesses (loss + per-param grad norms, pinned weights)
under both `gauge_mode='learned'` and `gauge_mode='constant'`. Both pass; the full
transformer test suite passes (807 passed, 3 skipped).

## Audit deliverables (no code change)

- `docs/audit_skip_attention_ift_phi_2026-06-01.md` — audit report. Verdict: the
  path is correct; `mu_embed`, `log_sigma_diag`, `phi_embed` all receive gradient
  under `ift_phi`+`skip_attention` (no silent-freeze gap); numerics clean; gauge

- `docs/refactor_plan_remove_attention_2026-06-01.md` — the refactor plan executed

- `docs/audit_workspace/` — 7 investigator memos, 3 verifier verdicts, the two

## Code changes

### `transformer/core/variational_ffn.py`
- The FFN is now the registered owner of `constant_omega`. When
  `gauge_mode='constant'` and no external ParameterList is passed, it builds its
  own per-head `nn.ParameterList` of identity GL(d_head) matrices (mirrors the old
  attention-side init). Legacy borrow path retained for callers that still pass
  one. This is the BR-1 fix: previously `constant_omega` lived on the attention
  module and the FFN borrowed it via `__dict__`; naive removal would have made it
  vanish from the optimizer (Ω frozen at identity, silent wrong math).

### `transformer/core/blocks.py`
- Removed the `IrrepMultiHeadAttention` instantiation and import.
- Removed the entire `if not self.skip_attention:` branch in `forward`; the block
  is now norm1 → FFN E-step → optional block mixer → σ update → optional residual.
- `self.skip_attention` is pinned `True` (vestigial, read by diagnostics/harnesses).
- Added block-level `self.irrep_dims` (mirrors `ffn.irrep_dims`; falls back to
  expanding `irrep_spec`). Re-pointed the non-flat `PerHeadGaugeConnection`
  construction to it (was `self.attention.irrep_dims`).
- Removed the κ-sharing redirect block; the FFN's own `log_kappa_per_head` is the
  sole owner. Removed the attention-side `rope_full_gauge_mode` assignment (kept
  the FFN side). Removed `self.norm2` (dead under skip; the single VFE sublayer
  uses `norm1`).
- DS-4 fix: under `non_flat_transport=True` the block now computes `delta_ij` on
  the pre-FFN path and passes `connection_delta` to the FFN. Previously this was
  computed only inside the attention branch, so non-flat transport silently
  no-op'd whenever attention was skipped.

### `transformer/core/model.py`
- Deleted the cross-layer transport-cache machinery in `_embed_and_prepare`
  (`cached_head_transports` was consumed only by the removed attention sublayer;
  the block forward never forwarded it to the FFN). `cached_head_transports` is now
  unconditionally `None`. `gauge_param='omega'` still works (omega threads to the
  FFN directly). Removed the `blocks[0].attention.*` dereferences.

### `transformer/core/block_config.py`
- `skip_attention` default is now `True` (vestigial); a config requesting `False`
  is forced to `True` with a warning. `from_config` default is `True`.
- The detaching-mode (`em_phi_p`/`em_phi_q`) and `use_equivariant_head_mixer`
  warnings now fire unconditionally (there is no attention autograd path / mixer
  host any more), pointing to `ift_phi` and `use_block_equivariant_mixer`
  respectively.

### `transformer/training/optimizer.py`
- Added an explicit `constant_omega` routing rule in `create_param_groups` so the
  re-homed `ffn.constant_omega.*` keeps `M_attention_lr` (it would otherwise fall
  through to the `ffn` group at `M_vfe_hyperparam_lr` — the BR-1a ~7.3× silent LR
  move).

### `transformer/training/experiment_runner.py`
- `_set_kappa_frozen` iterates only `block.ffn` (the sole owner of
  `log_kappa_per_head`); it previously also dereferenced `block.attention`.

### `transformer/train.py`
- Belief-coupling guard now triggers when `beta is None OR kl is None` (the pure-VFE
  block surfaces its own β but no KL matrix) and also initializes
  `belief_entropy_loss=0` there (a latent UnboundLocalError that only surfaced once
  the default became `skip_attention=True`).

### Entry-point configs
- `scripts/run_optuna_hpo.py`, `scripts/verify_active_inference.py`:
  `skip_attention` changed `False → True` (the `False` path no longer exists).

### Tests updated (attention sublayer removed)
- `test_cross_head_coupling.py` (3): `block.attention.irrep_dims` → `block.irrep_dims`.
- `test_block_equivariant_mixer.py`: `skip_attention=False` now forced to True; no
  attention submodule.
- `test_equivariant_head_mixer.py`: warn test updated to the new "no effect" message.
- `test_per_head_gauge_connection.py` (6): dropped `block.attention.rope_full_gauge_mode`
  assertions; rewrote the non-flat cache test as a DS-4 check (delta reaches the
  FFN); `'both'` + diagonal now runs via the FFN rope-full path instead of raising
  the old attention-side error (BR-5 behavior change).

## Behavior changes worth noting (beyond the no-op refactor)

- `rope_full_gauge='both'`: the attention-side σ-rotation is gone (BR-5). `'both'`
  and `'vfe_only'` now both reduce to the FFN-side rope-full path (diagonal σ, via
  lift). `'both'` + diagonal no longer raises; it runs.
- `non_flat_transport=True` now actually runs under the pure-VFE block (DS-4 fix),
  instead of silently no-op'ing. Holonomy-penalty tracking (`_last_exp_delta`) under
  pure-VFE non-flat is not wired (the block no longer materializes the full Ω); it
  reads `None`. Follow-up if holonomy_penalty > 0 is ever used with non-flat.

## Dead-code removal: five deprecated VFE module clusters

Removed five deprecated alternative-inference modules together with all of their
imports, delegators, config flags, gated branches, tests, and helper scripts. Each
module implemented an alternative to the pure-VFE iterative natural-gradient E-step
and was already dead under the live production config
(`train_publication.py`: `closed_form_e_step=False`, `use_deq` default `False`,
`active_inference=False`, `em_mode='ift_phi'`). The behavior of every removed branch
was therefore already not-taken on the live path, so the removal is behavior-preserving
by construction; the only live coupling was the import/call wiring, which is what the
edits sever.

Files deleted outright:

- `transformer/core/vfe_closed_form.py` — closed-form precision-weighted E-step
  (gated by `closed_form_e_step`).
- `transformer/core/vfe_deq.py` — DEQ implicit-differentiation fixed point and its
  two autograd `Function`s (gated by `use_deq`).
- `transformer/core/active_inference.py` — expected-free-energy pragmatic/epistemic
  E-step extension and the readout-wiring/FFN-configuration helpers (gated by
  `active_inference`).
- `transformer/core/expected_free_energy.py` — EFE term implementations and
  `generate_active_inference` (coupled to `active_inference`).
- `transformer/core/hebbian.py` — backprop-free P-flow / delta-rule learning
  (reached via lazy imports from the `'hebbian'` training mode).
- `tests/transformer/test_vfe_closed_form.py`, `scripts/verify_active_inference.py`,
  `scripts/closed_form_analysis.py` — tests and analysis scripts that existed only
  for the removed features.

The consumer edits were consolidated per file so each multi-cluster file was touched
once. `variational_ffn.py` lost the three module-level imports, the `use_deq` /
`deq_*` / `closed_form_e_step` / `n_picard_steps` / `picard_trust_region` constructor
parameters and their `self.` attributes, the `_make_deq_step_fn(_with_phi)` and
`_closed_form_e_step` wrapper methods, the active-inference / EFE gradient fold, and
the closed-form, DEQ, and Picard branches inside the E-step; the descent loop now reads
`_n_iters = self.n_iterations` unconditionally. `block_config.py` dropped the
corresponding dataclass fields in lockstep with their `from_config` constructor
kwargs (removing a field without its kwarg would have raised `TypeError` on every run,
including the live pure-VFE path), and the `__post_init__` Picard validation was
removed with its now-dead fields. `blocks.py` and `model.py` lost the
`configure_ffn_active_inference` and `wire_readout_references` imports and their
unconditional construction-time calls; the live decode path reads `self.prior_bank`
(a registered submodule) rather than the deleted `_prior_bank_ref`, so dropping the
wiring is safe. The lazy hebbian delegators in `model.py`, `prior_bank.py`,
`embeddings.py`, and `experiment_runner.py` were removed.

The headline correctness risk was that `experiment_runner.train_step` called
`_apply_p_flow_and_delta_rule` unconditionally on every step, and that delegator
lazy-imported `hebbian` — so deleting the module without removing the call would have
crashed live training at runtime (not at import). The call and delegator were removed
together with the file.

At the user's request the delta-rule plumbing was removed by the maximal sweep rather
than left inert: `use_delta_rule` and `_tied_weights` in `train_step`, the
`out_proj.weight` gradient-zeroing and the `requires_grad` toggles in
`_run_forward_and_backward`, and the four `use_p_flow` / `p_flow_ema_decay` /
`use_delta_rule_w_out` / `delta_rule_lr` fields in `TrainingConfig` (with their
`from_config` kwargs in `resume_training.py` and the direct attribute reads in
`experiment_runner.py`). The cascade-dead `p_flow/*` metric writes in
`train.py::compute_free_energy_loss` were removed since their only consumer was the
deleted hebbian update. The `'hebbian'` training mode was removed from
`train_publication.py` (the `HEBBIAN_CONFIG` preset, the dispatch branch, the argparse
choice, and the help/docstring/error mentions), and the orphaned active-inference /
closed-form config-dict keys were cleaned from `train_publication.py`,
`run_ablation_suite.py`, and `run_optuna_hpo.py`. Three test classes that exercised
the deleted delegators (`TestDEQNeumannBackward`, `TestPFlowAndDeltaRule`,
`TestPriorBankUpdateFromBeliefs`) were removed surgically, leaving the rest of each
file intact. A small set of docstrings/comments that named the now-deleted modules
(`em_modes.py`, `variational_ffn.py`) were corrected to drop the stale references
while preserving the accurate caveats.

The shared live state that sits adjacent to the deleted code was explicitly preserved:
`learnable_head_kappa`, `detach_phi`, `sigma_ce_scale`, `e_step_early_exit_tol`,
`_mh_cached_bep`, `_use_omega`, the `beta_current`/`beta_heads` initializers,
`n_iterations`, and `update_phi`.
