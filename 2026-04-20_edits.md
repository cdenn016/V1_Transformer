# 2026-04-20 Edits вЂ” Attention-KL sigma_floor plumbing for W_O=False full-cov stability

## Context

With the Exp-74 baseline (`scripts/run_ablation_suite.py:BASELINE_CONFIG`) the
user reported NaN training failure when `use_output_projection` was flipped
from `True` to `False`, all other flags held: `em_mode='em_phi_p'`,
`diagonal_covariance=False`, `use_prior_bank=True`, `gauge_fixed_priors=False`,
`hierarchical_priors=True`, `gauge_group='GLK'`, 2 heads of dim 10,
`e_step_sigma_floor=0.01`, `phi_trace_clamp=0.75`.

Observed trace:

```
Step 400 | Loss 1.3664 | PPL 450.7   [NUM] gauge_kl_full_chol_fallback_diag: 2
Step 500 | Loss 1.3693 | PPL 456.6   [NUM] gauge_kl_full_chol_fallback_diag: 9
Step 552 | loss is nan, skipping backward
```

Loss was low and monotone right up to the NaN вЂ” this was not divergence; it
was Cholesky failure inside the attention-KL path
(`transformer/core/gauge_utils.py`) plus the Cholesky-fallback branch feeding
`1/Пѓ_diag` gradients back into the M-step. The fallback counter climbing
2 в†’ 9 over 100 steps signalled progressive Пѓ collapse; accumulated fallback
gradients drove one eigenvalue past the hard floor and the next forward
produced NaN.

## Root cause

`fused_block_diagonal_kl_full` (`gauge_utils.py:515`) Cholesky-decomposes
`sigma_i_reg = sigma_q + eps В· I_d` and
`sigma_t_reg = О©_ij @ sigma_q @ О©_ij^T + eps В· I_d` at `gauge_utils.py:620-627`.
The jitter is `eps = 1e-6`. With `use_output_projection=True`, learnable W_O
(applied to Ој only) provides cross-head mixing pressure, and the block-diagonal
ОЈ leaving attention retains enough per-block conditioning for `+ 1e-6В·I_d` to
suffice. With `use_output_projection=False`, the M-step has no cross-head
learnable parameter in attention, all mixing pressure falls on П†, and
per-block Пѓ_q eigenvalues drift below 1e-6 after sandwich transport by
О©-matrices with `phi_trace_clamp=0.75`-permitted spectral spread. Cholesky
fails в†’ diagonal-fallback branch runs в†’ `log(Пѓ_diag)` and `1/Пѓ_diag`
gradients pull Пѓ further toward zero в†’ NaN.

The E-step already floors Пѓ_q eigenvalues at `e_step_sigma_floor=0.01` via
`spd_eigfloor` (`variational_ffn.py:1534`, `vfe_utils.py:345-427`). The
attention-KL path did not honor that floor вЂ” it used its own un-configured
`eps=1e-6` jitter.

## Fix (B1 + B3): plumb sigma_floor into the attention-KL Cholesky guard

### `transformer/core/gauge_utils.py`

- Added `spd_eigfloor` to the `from transformer.core.vfe_utils import вЂ¦`
  line (line 16).
- `fused_block_diagonal_kl_full` now accepts `sigma_floor: Optional[float] = None`
  (line 521). When provided, replaces `sigma_i + eps * I_d` /
  `sigma_transported + eps * I_d` with `spd_eigfloor(В·, floor=sigma_floor)`
  calls at the pre-Cholesky site (lines 620-633). Bounds condition number
  to `Пѓ_maxВІ / sigma_floor`, guaranteeing Cholesky success.
- Diagonal-fallback branch (lines 688-695) now clamps `sigma_diag_t` and
  `sigma_diag_i` at `sigma_floor` when provided (else `eps` вЂ” legacy
  behavior preserved). Keeps `в€‚log(Пѓ)/в€‚Пѓ = 1/Пѓ` gradients bounded in the
  fallback path as well.

### `transformer/core/kl_computation.py`

# 2026-04-20 Edits (afternoon) вЂ” Default-off W_O and equivariant head mixer

## Context (second change, same day)

Follow-on theoretical question from the user: "should we even have an
output-projection path with PriorBank?" Three parallel investigators plus a
verifier converged on a clear answer: **no**. `VFE_Transformer_Idea.md` В§18.2
explicitly states *"No separate `nn.Linear` output projection вЂ” Law 3 is
enforced by using the same PriorBank for both ends."* W_O applies a learnable
KГ—K linear to Ој only, never to ОЈ, so under any gauge action h в€€ GL(K) the
returned pair `(W_OВ·hВ·Ој, hВ·ОЈВ·hбµЂ)` is not a coherent belief unless
`[W_O, h] = 0 в€Ђ h в€€ G` (Schur). A generic 400-param W_O does not satisfy
that; by Schur-Weyl the commutant of `[('fund', 2, 10)]` under a tied GL(10)
is Mв‚‚(в„ќ) (4 scalar parameters).

Goal of this edit: align default behavior with the thesis, expose a
principled opt-in replacement, and break nothing.

  rationale + reference to В§18.2.

- Updated the `config.get('use_output_projection', True)` fallback to `False`
  (line 441) so dict-based constructors respect the new default.

Forward integration at `attention.py:1996-2001`: W_O takes precedence when
both are enabled (legacy behavior); otherwise the mixer runs if enabled; else
Ој, ОЈ pass through unchanged.

'use_equivariant_head_mixer', False)` into `IrrepMultiHeadAttention`.

## Context (third change, same day)

Investigation across four parallel agents (cartography, numerical-stability
audit, refactor design, independent verifier) identified an unhardened
Cholesky at `transformer/core/prior_bank.py:925` inside `_decode_full_cov`

The audit traced the following causal chain: `em_mode='em_phi_p'` resolves
to `amortize_sigma=False` (`em_modes.py:16`), which at
`variational_ffn.py:1500-1502` detaches `sigma_p` at the self-coupling
slogdet site. Under that detach the only gradient path to `sigma_p` is the

## Fix

### `transformer/core/prior_bank.py` вЂ” `_decode_full_cov`

- Line 910: `variance_floor = max(self.eps, 1e-4)` в†’ `max(self.eps, 0.01)`.

- Lines 921-927: Replaced bare `torch.linalg.cholesky(sigma_p_safe)` +

# 2026-04-20 Edits (evening, Phase 3) вЂ” LU-solve for gauge-covariant spd_eigfloor

## Context (fifth change, same day)

Phase 3 replaces the explicit matrix inverse in `spd_eigfloor`'s
gauge-covariant branch at `transformer/core/vfe_utils.py:411`. The prior
implementation materialized `A_inv = torch.linalg.inv(A)` where
`A = exp_phi.to(float32)` is a general GL(K) matrix, then used it in the
whitening sandwich `W = A^{-1} ОЈ A^{-T}`. The original comment at lines
404-405 flagged this as the "critical numerical step" that "breaks under
fp16."

Under the user's `scripts/run_ablation_suite.py` config, `A` is built by
`matrix_exp(phiВ·G)` with `phi` Frobenius-clamped at 20 (`gauge_utils.py:75`),
so the worst-case operator norm per factor is `exp(20)`. With the cocycle
structure `О© = exp(phi_i) exp(-phi_j)` and `rope_full_gauge='both'` layered
on top, effective `cond(A)` can compound well past fp32's precision limit.
`spd_floor_mode='eigclamp'` (the user's default) routes every E-step
iteration through this branch, so the path is hot on every forward pass.

The Phase 1 verifier (claim R4, CONFIRMED) established that
`cholesky(A A^T)` does not give `A^{-1}` for a general GL(K) matrix. The
correct inverse-free substitute is an LU or QR factor followed by
triangular solves.

## Fix

### `transformer/core/vfe_utils.py`

Lines 406-426 (`spd_eigfloor` gauge-covariant branch) вЂ” the three-line
`linalg.inv` + matmul block replaced by:

```python
LU, pivots = torch.linalg.lu_factor(A)
left = torch.linalg.lu_solve(LU, pivots, Sigma_f)              # A^{-1} ОЈ
W = torch.linalg.lu_solve(LU, pivots, left.transpose(-1, -2))  # A^{-1} (A^{-1} ОЈ)^T
W = W.transpose(-1, -2)                                        # = A^{-1} ОЈ A^{-T}
W = 0.5 * (W + W.transpose(-1, -2))
```

Mathematical derivation: `W = A^{-1} ОЈ A^{-T}` factors as two successive
`A^{-1}` actions on appropriate rearrangements of ОЈ. Step 1 solves
`A В· left = ОЈ` for `left = A^{-1} ОЈ`. Step 2 solves `A X = left^T` for
`X = A^{-1} left^T`; transposing gives `W = X^T = left В· A^{-T} =
A^{-1} ОЈ A^{-T}`.

Design notes:

1. `lu_factor` computed once, reused for both solves. FLOP-neutral versus
   `linalg.inv` + two matmuls, strictly better on numerical stability
   because the inverse is never materialized.
2. The forward-action line at 425 (`out = A @ W_clamped @ A^T`) is
   unchanged вЂ” it is a forward application, not an inverse.
3. Symmetrization, `_safe_eigh` jitter, `sigma_max` clamp, and fp32
   autocast context are all preserved verbatim.
4. Autograd flows through `lu_solve` via the standard backward kernel
   (two triangular solves internally), supported since PyTorch 1.13.
