# Blue Opening — wilson-observable-scope

## Steelman (opposing position)

The manuscript at PIFB:868 and PIFB:874 makes two direct existence claims about the implementation (`holonomy_penalty` regularizer and `cocycle_relaxation` configuration parameter), the `/vfe` module exposes the edge-relaxed Omega construction `Omega_ij = U_i exp(delta_ij·G) U_j^{-1}` under `use_non_flat_transport=True`, and the named identifiers `holonomy_penalty`, `cocycle_relaxation`, `wilson` do not appear anywhere in `transformer/vfe/`; therefore the language-modeling pure path the user has nominated as canonical materially fails to realize the construction the manuscript says it realizes.

## Position

The absence of `cocycle_relaxation`, `holonomy_penalty`, and Wilson observable code from `transformer/vfe/` is not a falsifying gap in the language-model-only restriction of the PIFB framework. The Wilson observable is Regime II content (PIFB:824, PIFB:826, PIFB:880); the autoregressive attention DAG that `/vfe` is trained under has no closed cycles, which makes the Wilson observable structurally trivial as a measurement (PIFB:876); the named regularizers do exist in the wider repository (`transformer/core/connection.py`, `transformer/analysis/holonomy_metrics.py`, `transformer/pure_vfe/gauge.py`); and `non_flat.py` in `/vfe` is at initialization a Regime I path because every gate is zero. The disagreement between the implementation sentences at PIFB:868 and PIFB:874 and the `/vfe` subtree is a manuscript scope-of-reference ambiguity, not a falsification of the Regime I language-modeling implementation.

On a directed acyclic graph there is no closed directed cycle, hence no nontrivial Wilson loop in the canonical lattice-gauge sense. The structural triviality in autoregressive attention is a canonical-source consequence, not a PIFB peculiarity.

## Falsification conditions

This defense is wrong if any of the following hold.

1. PIFB elsewhere labels Regime II as part of the language-modeling core (not the gravitational extension). If a passage outside PIFB:824–PIFB:880 promotes Regime II to a language-model-essential component, the scope argument collapses.
2. The `/vfe` training entry point exercises `use_non_flat_transport=True` with sufficient training to drive `raw_strength` away from zero, *and* the user's CLAUDE.md or claim binds `/vfe` to implement the Wilson observable diagnostic when in that state. The init-is-Regime-I argument covers the default configuration; it does not cover a trained Regime II run with no observable.
3. PIFB:868 (`holonomy_penalty`) and PIFB:874 (`cocycle_relaxation`) are interpreted by the manuscript itself as commitments to the `/vfe` module specifically (not the wider codebase). If PIFB contains a sentence binding "the implementation" to `/vfe` exclusively, the wider-tree defense collapses.
4. The user states that the canonical "pure path" in CLAUDE.md is required to contain every implementation referent in PIFB regardless of regime classification. The CLAUDE.md text reviewed at dispatch does not say this — it requires a pure path to exist, not that the pure path contain every construction PIFB names.
5. PIFB:876's structural-triviality argument is wrong: a closed undirected cycle in attention data flow exists under causal masking. If a counterexample is produced (a closed directed cycle in a lower-triangular DAG), the Wilson-triviality defense fails.

None of these are demonstrated in `01_evidence.md`. The defense survives the evidence pack as presented.
