## Steelman (opposing position)

## Position

`stack.py:145` is not a falsifying deviation from PIFB ¬ß1502-1551. The "named" slot `priors.phi` is never read anywhere in `transformer/vfe/e_step.py`; it is a dead field. The posterior gauge frame *does* cascade across blocks via `beliefs.phi`, which the next block's E-step re-reads at `e_step.py:779` (`phi = beliefs.phi`) and evolves through `_update_phi` during its inner iteration. Because the operative gauge state of layer ‚Ñì+1 IS the converged posterior œÜ of layer ‚Ñì, there is no gauge-frame information loss at the layer boundary, and PIFB ¬ß1502-1551's prescription ‚Äî that œÜ evolve on its slow timescale ‚Äî is honored by the `beliefs.phi` cascade, not the `priors.phi` slot. The manuscript's prescription is a dynamical statement about the œÜ-flow, not an attribution statement about which BeliefState field carries it.

## Falsification conditions

This defense fails if any of the following holds.

1. **A code path inside `transformer/vfe/` that reads `priors.phi` and uses the value to build Œ©, the KL functional, the attention logits, or any other operational quantity.** Grep confirms no such read exists today; if one is found, the "dead slot" argument collapses and the cascade question becomes operationally live.

2. **A passage of PIFB ¬ß1502-1551 that explicitly prescribes a per-layer posterior-œÜ-to-prior-œÜ cascade for the L-block transformer stack.** PIFB:1537 uses "may evolve" and PIFB:1545 writes the flow as a temporal natural-gradient equation, not a layer-discretized cascade. If red can produce a manuscript line that writes the layer-cascade form `œÜ_p^{(‚Ñì+1)} = œÜ^{(‚Ñì)*}` analogous to `Œº_p^{(‚Ñì+1)} = Œº^{(‚Ñì)*}`, the modal-permissiveness defense fails. The closest candidates ‚Äî PIFB:1571-1574 (transformer-limit setup) and PIFB:1812-1839 (value-aggregation, complete attention formula) ‚Äî describe the per-position zero-dimensional limit and do not prescribe an L-block œÜ-cascade.

3. **A demonstration that `beliefs.phi` is NOT actually the posterior œÜ of the previous layer at the entry point of the next block's E-step** ‚Äî e.g., if `VFEBlock.forward` overwrote `beliefs.phi` post-E-step with the embedding value, or if a normalization layer in `_resolve_vfe_norm` mutated phi in flight. `block.py:172-192` shows the head_mixer touches (Œº, Œ£) only and the norm touches Œº only; phi survives unmodified from the E-step output through to the next block's input.

4. **Empirical evidence that the user actually intends `priors.phi` to be live** ‚Äî e.g., a config flag like `prior_handoff_phi` exists in `VFEConfig`. `transformer/vfe/config.py` has `prior_handoff_rho` (Œº) and `prior_handoff_sigma` (œÉ) but no `prior_handoff_phi`. The absence of the toggle is itself evidence that the slot was always intended to be inert; the cascade is architecturally routed through `beliefs.phi`, by design.

5. **A canonical-VI argument that the named-slot identity matters even when the operational gauge state cascades correctly through the belief channel.** Standard hierarchical VI prescribes that "the posterior of level ‚Ñì informs level ‚Ñì+1," and `beliefs.phi` is the operative vehicle of that information. If the canon requires the prior-named slot specifically to carry the posterior value as a matter of *naming* rather than *function*, then the canon is being read as nominalism rather than mathematics, and the defense reverts to "no operational difference, hence no falsification."

The defense rests on a single empirical fact and a single textual fact: `priors.phi` has zero downstream readers (empirical), and PIFB:1537 uses "may" with PIFB:1545 writing a temporal flow not a depth cascade (textual). Falsify either and red wins.
