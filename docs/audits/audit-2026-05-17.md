  local `train_vfe.py` config, so production runs are using a

  reminder, the user may already be in this regime.

  lacks `trusted=`; the canonical `checkpoint.py:53` already exposes

- **2.1** lowered critical → high — bug is real, but with
  `e_sigma_lr=0.001` the clamp would not bind in practice; effect is
  "trust region effectively absent" not "σ frozen".
- **2.5**, **2.10** lowered to low — not bugs; gradient still flows
  to kappa via outer backward, and the in-place clamp targets a
  freshly allocated tensor.
- **3.6** lowered to low — duplicate KL is a refactor opportunity,
  not a correctness issue.
- **4.3**, **4.5** lowered — `torch.eye` cost is small at K=64;
  `set_to_none=True` is the PyTorch ≥2.0 default.

## Confirmed Punch List (critical → high)

Ranked, executable items the user can authorise. None are auto-fixed
by this audit.

1. **[high]** `_LayerNormSigmaAdapter` is the active default and uses
   `nn.LayerNorm` (learnable affine) — `transformer/vfe/block.py:46` +
   `transformer/vfe/train_vfe.py:103`. Either remove the adapter and
   default to `mahalnorm`, or extend the documented exception list in
   CLAUDE.md to cover learnable-affine LayerNorm.
2. **[high]** `VFEConfig` is missing the `e_sigma_q_trust` field that
   CLAUDE.md documents as default 5.0 —
   `transformer/vfe/config.py:38-50`. As a result,
   `transformer/vfe/e_step.py:493` passes `sigma_trust=self.e_sigma_lr`,
   collapsing the trust region onto the step LR.
3. **[high]** Closed-form diagonal transport silently uses the
   approximation regardless of `exact_diagonal_transport=True` —
   `transformer/core/vfe_closed_form.py:181-183`. Closed-form users
   asking for the exact path get the diagonal one with no warning.
4. **[high]** `f_history` monotone monitor in the VFE E-step omits
   the `tau·β·log(β/π)` attention-entropy term when
   `include_attention_entropy=True` —
   `transformer/vfe/e_step.py:377-380`. Monitor measures a different
   objective than the one being descended.
5. **[high]** Closed-form path materialises the full `(B,N,N,d_h,d_h)`
   Omega tensor twice per head per E-step —
   `transformer/core/vfe_closed_form.py:180-187` and 413-418.
   Dominant memory term at large `N`.
6. **[high]** `_update_phi` does a second full `compute_kl_attention`
   pass per E-step iteration without forwarding the cache —
   `transformer/vfe/e_step.py:499, 527-545`.
7. **[high]** Final-iteration diagnostics block fires ~15 `.item()`
   syncs on every forward — `transformer/vfe/e_step.py:413-480`. Same
   pattern in `variational_ffn.py:2235`. Gate behind a flag.
8. **[high]** Two stale `.tmp` editor crash artifacts in source dir,
   not gitignored — `transformer/core/model.py.tmp.22376.*`,
   `transformer/core/variational_ffn.py.tmp.22376.*`.
9. **[high]** `em_mode: str` rather than `Literal[...]` —
   `transformer/core/block_config.py:129`. CLAUDE.md memory rule
   "no lenient config aliases" indicates the user expects strict
   typing here.
10. **[high]** `create_dataloaders` uses Python 3.10+ `Tuple|Tuple`
    union in annotation position without `from __future__ import
    annotations` — `transformer/data/datasets.py:2202`. SyntaxError on
    3.9 at import time.
