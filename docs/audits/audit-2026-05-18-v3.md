## Confirmed punch list (critical → low)

### Critical
*(none — no findings are immediate-data-loss or RCE-now level under the current config; 1.4 was a hypothetical for future loaders, scored INCONCLUSIVE)*

### High

1. **Stats collector early-return discards all layers but the first** —
   `trainer.py:384-394`. Hoist `return` outside the `for` block, or
   rename the function to document layer-0-only semantics.
2. **Head-mixer "tied-gauge" docstring is contradicted by multi-head
   GL(K) generator construction** — `head_mixer.py:33-41, 163-190`;
   `math_utils/generators.py:870-953`. `generate_glK_multihead_generators`
   gives each head an independent `gl(d_head)` block, so `kron(A, I_d)`
   does not commute with the actual gauge action. Restrict
   `use_equivariant_head_mixer=True` to `gauge_group='SON'` and emit a
   `UserWarning` for GL(K), or rewrite the manuscript-style equivariance
   argument.
3. **`lambda_softmax` ablation sweep is a hard no-op under
   `BASELINE_CONFIG`** — `vfe_ablation_suite.py:256-261` +
   `e_step.py:669,728,783,1107`. `include_attention_entropy=True`
   in the baseline forces `lambda_softmax=0.0` in every gradient path,
   so all 8 sweep rows produce bitwise-identical loss. Add
   `'requires': {'include_attention_entropy': False}` to the sweep entry.
4. **`gauge_fixed_priors` is absent from `BASELINE_CONFIG`** —
   `vfe_ablation_suite.py:83-192`. `train_vfe.py` runs with `False`
   (per-token μ/σ lookup); BASELINE_CONFIG omits the key, so
   `VFEConfig`'s default `True` (gauge-orbit shared base) is used. Every
   ablation row trains a structurally different model than the live
   entry point.
5. **`phi_preconditioner='pullback'` raises `TypeError` at runtime** —
   `e_step.py:1131-1136, 1437-1442` + `core/gauge_preconditioner.py:371`.
   Required `structure_constants` is not threaded through; option
   appears in the `Literal` enum and config docstring. Either wire
   `structure_constants` at `VFEEStep.__init__` or remove `'pullback'`
   from the enum until the call chain is complete.
6. **Unconditional `compute_gauge_transport` rebuild for the diagnostic
   snapshot** — `e_step.py:1012-1023`. Fires on every final E-step
   iteration of every training step; result consumed only at
   `eval_interval` boundaries (~1 in 2000). Gate behind an
   eval-pending flag or reuse the iteration-start `block_exp_pairs`.
7. **Bare `Optional[Callable]` on four public forwards** —
   `stack.py:73, block.py:127, e_step.py:520, 1461`. Narrow to
   `Optional[Callable[[torch.Tensor, torch.Tensor], Tuple[Optional[torch.Tensor], Optional[torch.Tensor]]]]`.
8. **`preference_mode: str` should be `Literal[...]`** — `efe.py:56`.
   Invalid strings silently reach `compute_risk` with no validation.
9. **Outer M-step doesn't minimize the manuscript F functional** —
   `model.py:234-251`; aux loss reads detached (μ, σ, φ, μ_p, σ_p) at
   `e_step.py:437-441`. Defensible as amortised inference, but the
   manuscript framing presents this as canonical variational EM.
   Document as amortised in the manuscript or add the converged-q F
   value to the outer loss.

### Medium

10. **Module-level mutable cache keyed by `id(generators)`** —
    `attention.py:26-27, 53-62`. Replace `id()` key with a content
    fingerprint (`data_ptr + shape + dtype`) or `WeakKeyDictionary`.
11. **Three sites swallow all Exception** — `trainer.py:1109, 1123,
    1367`. Narrow to `(OSError, ImportError, ValueError)` or use
    `logger.exception` so genuine `AttributeError`/`TypeError` bugs
    aren't downgraded to WARN.
12. **Non-flat covariance transport is diagonal-of-sandwich only** —
    `non_flat.py:488-494`. No `exact_diagonal_transport` toggle on the
    non-flat track. Either expose the toggle or document the limit at
    the call site.
13. **Decode σ floor (1e-4) is 100× below encode floor (0.01)** —
    `prior_bank.py:455-456, 94`. Decode receives σ values never
    reachable at encode time; set `decode_eps = self.eps` or document
    the intentional gap.
14. **`_diag_kl` duplicates `_kl_kernel_diagonal` and lacks Rényi α
    support** — `e_step.py:104-130` vs `core/kl_computation.py:370-460`.
    The F-monotone monitor uses the local copy → ignores
    `alpha_divergence != 1.0`. Replace with a thin wrapper over
    `_kl_kernel_diagonal`.
15. **`alpha_divergence` silently ignored on autograd / non-flat /
    omega-direct paths** — `config.py:361-380`, `e_step.py:1245-1253,
    1366, 1429, 1605`. Warning fires after `__post_init__` already
    forced `use_autograd_mu_sigma=True`. Replace with a runtime
    `RuntimeError` at the top of each affected forward.
16. **`_last_diagnostics` written twice when `track_layer_diagnostics=True`** —
    `e_step.py:882-897` then `901-941`. First dict allocated and
    overwritten before any read. Invert the gate.
17. **Unconditional CPU sync per training step in cheap-diagnostics** —
    `e_step.py:881`. `.cpu().tolist()` fires every step; consumed only
    every `log_interval` (default 200). Gate behind an `is_log_step`
    signal.
18. **`_forward_omega_direct` diagnostics not batched** —
    `e_step.py:1664-1682`. 10+ individual `.item()` syncs instead of
    the batched-stack-then-tolist pattern used on the phi path.
19. **Unannotated `_active_inference_fn`** — `model.py:116, 120`. Add a
    class-level `Optional[VFEActiveInference]` annotation.
20. **`train_step(batch: Dict)` accepts `list|tuple` at runtime** —
    `trainer.py:621-633`. Widen to `Union[Dict[str, Tensor], Tuple[Tensor, Tensor]]`.
21. **`cfg.decode_tau` not wired into `prior_bank.decode` in training
    forward** — `model.py:202`. Always defaults to `tau=1.0`. Pass
    `tau=self.cfg.decode_tau`.
22. **Decode rescales canonical `−KL/τ` by learnable scalar `c`** —
    `prior_bank.py:469-470, 176`. Either document `decode_log_scale` as

    generalisation in the manuscript or collapse to scalar `KL_i =

24. **Bayesian-α product-rule correction is per-dim** —
    `core/vfe_gradients.py:1071-1076`; `e_step.py:1215-1218`. Tied to

## Cross-cutting themes

Four patterns recur across multiple findings:

1. **Hot-loop CPU syncs in diagnostic paths** (#16, #17, #18, #25, #31,
   #32). Cheap-diagnostics block runs every step but is read every
   `log_interval` (default 200); same pattern in `_collect_kappa_stats`,
   `_collect_e_step_diagnostics`, `_forward_omega_direct`. Single-PR
   refactor opportunity: gate every diagnostic emission behind a
   shared `is_log_step` signal.
2. **Per-dim Bayesian-α generalisation vs. scalar manuscript** (#23,
   #24). Per-dim `(c0, b0, kl_k)` is a stronger generalisation than
   `Attention/GL(K)_attention.tex` Eq. 935 derives. Either extend the
   manuscript with a per-dim Gamma-Normal conjugacy derivation or
   collapse to scalar.
3. **Live entry-point config ≠ ablation BASELINE_CONFIG** (#4, #15,
   #38). `gauge_fixed_priors`, `alpha_divergence`, `norm_type='layernorm'`
   each diverge in policy or sweep coverage. Sweeps from
   `vfe_ablation_suite.py` should not be cited as "this is what
   `train_vfe.py` does" without an explicit baseline check.
4. **Dead branches in `Literal` enums** (#5, #28). `'pullback'` raises
   TypeError, `non_flat_tile_size=N` silently no-ops — both are
   public-API knobs the user can set. Add `__post_init__` guards (or
   delete) so misconfigurations fail loudly at construction.
