# Deep Audit — 2026-05-25

**Trigger:** "perform a deep audit on the codebase and search for any deprecated code."

**Method:** Five parallel expert investigators (code-reviewer, debugger, refactoring-specialist, performance-engineer, python-pro) swept the repo from distinct angles; one independent `general-purpose` verifier then re-read every cited source location and ran its own greps. No finding ships unverified. Comments/docstrings count as evidence of *intent*, not *behavior*.

**Headline:** The VFE/AIF/pure_vfe surgical removal in commit `866305a8` was **functionally clean** — zero orphaned runtime imports, zero dead config fields that break execution, and the `core/` package is fully self-contained. The deprecated-code residue is entirely **stale docstrings/comments pointing at deleted files** plus a handful of **dead-but-harmless config literals/aliases**. Two genuine functional bugs surfaced (a broken script import, and a config field that silently does nothing). Two of the highest-severity investigator claims were **REFUTED** by the verifier.

## Scope

`transformer/`, `math_utils/`, `scripts/`, `tests/`, top-level entry points (`generate.py`, `inference.py`). Excluded: `.claude/`, `docs/`, `Attention/`.

## Confirmed Punch List (ranked)

### Functional bugs (fix these)

1. **[high]** Broken script import — `scripts/verify_active_inference.py:275` imports `_compute_active_inference_gradient` from `variational_ffn` (moved to `active_inference.py:94`). Script `ImportError`s on first use. → change the import module.
2. **[DOWNGRADED → docs]** `TrainingConfig.sigma_ce_scale` — on deeper tracing this is NOT a functional bug. The model reads `sigma_ce_scale` from the entry-point flat CONFIG dict (`model.py:1010`); the `TrainingConfig` field is a checkpoint round-trip slot (`vars()` → resume), not a model-config source, and no caller sets it directly. Defaults left unchanged; clarifying comment added at `config.py:247`. (Audit's original "bug" framing assumed `TrainingConfig` feeds the model — it doesn't.)
3. **[medium]** `TrajectoryRecorder` fails `RecorderProtocol` — missing `record_step` (`analysis/trajectory.py`), so `isinstance(recorder, RecorderProtocol)` is `False` at runtime (`vfe_utils.py:1031`). → add a `record_step` (no-op or delegating) or relax the Protocol.
4. **[medium]** `shap_vfe_component_attribution` is dead and broken — never called, and reads `output.vfe_components` which no surviving forward produces (`interactive_belief_viz.py:722`). → delete it.

### Deprecated dead code (safe to remove)

5. **[high]** `'pure_fep'` training mode + `get_pure_fep_config` — never dispatched (`config.py:42,338`). → remove the Literal arm + preset, or raise `NotImplementedError`.
6. **[medium]** `BeliefState.omega` field — never set by surviving code (`types.py:32`). → remove.
7. **[low]** `lambda_beta` / `alpha_phi` alias kwargs (`train.py:283-284`); `ffn_lr` config fallback (`metrics_tracking.py`); `training_mode` pass-through (`resume_training.py:465`) — all vestigial. → prune at next cleanup.

### Stale docstrings/comments referencing deleted `vfe/` (documentation rot)

8. **[low]** `core/types.py:4,25-28`, `core/vfe_gradients.py:1701`, `core/active_inference.py:231`, `math_utils/transport.py:389` — all reference deleted `transformer/vfe/` files or the deleted `VFEConfig`. → repoint to `BlockConfig` / `gauge_param='omega'` and drop dead paths.
9. **[low]** `train_publication.py:716` `--mode` help lists nonexistent "amortized"; `vfe_dynamics_plots.py:19-20` docstring shows a CLI that doesn't exist. → fix the strings.
10. **[low]** Orphaned scratch artifacts in `scripts/` (`probe_output.txt`, `rope_attention_comparison*.png`). → delete or `.gitignore`.

### Performance (opt-in, not deprecation)

11. **[high]** `vfe_gradients.py:323-327` dead `C=N` chunk loop pays per-call slice/contiguous overhead; **`:455`** materializes full `Σ_j⁻¹Σ_i` for a trace (collapsible to one einsum). **[medium]** uncached `torch.eye` per block (`variational_ffn.py:1427`), duplicate `fused_block_matrix_exp_pairs` (`:2723`), per-iter `.item()` GPU sync (`:2391`). These are real but live behind the hot E-step path; treat as a separate perf pass.

### Type-hint gaps (CLAUDE.md mandates hints on all signatures)

12. **[high/medium]** `e_step_early_exit_tol: float = None` (should be `Optional`), `_resolve_gauge_mode -> Tuple` (bare), and unannotated public methods: `_forward_vfe_body`, `PriorBank.forward`, `_compute_multihead_vfe_gradients`, 4 `model.py` delegators.

### Verifier-rejected (do NOT act on)

- **E** — no `sigma_p` gradient leak; boundary enforced at `variational_ffn.py:1564-1566`.
- **Y** — `grad.clone()` is correct for odd K; `empty_like` would corrupt the trailing column.
- **AC** — `sigma_j_inv` is always needed for the μ gradient.
