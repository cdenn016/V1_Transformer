# Deep Audit — 2026-05-21

## Scope

`transformer/vfe/` — every `.py` file directly under the canonical VFE module
(config.py, model.py, e_step.py, block.py, attention.py, trainer.py,
prior_bank.py, stack.py, positional.py, efe.py, non_flat.py, omega_direct.py,
head_mixer.py, _numerics.py, cross_coupling_*.py). Excluded: `*.tmp.*`,
`__pycache__`, `vfe_ablation_results/` (data), and runtime artefacts under
`vfe_runs/`.

Active config assumed throughout: `gauge_group='GLK'`,
`irrep_spec=[('fund',2,10)]`, `diagonal_covariance=True`, `use_rope=True`,
`rope_full_gauge='off'`, `e_phi_lr=0.0`, `use_prior_bank=False`,
`n_e_steps=1`, `n_layers=1`, `em_mode=vfe_default profile`,
`eval_interval=20000`.

User-flagged motivation: lowest validation PPL observed at
`mu_init_std≈0`, `phi_scale≈0`. The audit was asked to surface any code path
that silently degenerates the gauge mechanism at small init.

## Method

Five investigator agents (code-reviewer, debugger, refactoring-specialist,
performance-engineer, python-pro) were dispatched in parallel with
self-contained briefs that forwarded CLAUDE.md hard constraints. After they
returned, the verifier agent (general-purpose) was dispatched to re-read every
cited source line; that agent hit a socket error mid-run, so the audit author
re-read each cited location directly and produced the verdicts below. Every
**CONFIRMED** verdict is backed by a fresh read of the cited file at the cited
line. Every claim whose only evidence was a comment or docstring is marked
**REFUTED** or downgraded to **INCONCLUSIVE**.

## Did anything degenerate the gauge mechanism at small init?

The user's empirical observation — lowest PPL at `mu_init≈0`, `phi_scale≈0` —
has two structural explanations in the code:

1. **Decode bypass (verdict #7).** `use_prior_bank=False` routes decode
   through `nn.Linear(K, V)` on `mu_final` (model.py:99, 195). `sigma` and
   `phi` never reach the cross-entropy gradient. CE can therefore be
   minimised by the linear head alone, regardless of whether the gauge
   structure carries useful information. Smaller init = cleaner canvas for
   the linear head; gauge geometry is decode-decoupled.

2. **Positional gauge contamination (verdict #6).** Setting `phi_scale=0` in
   the config does NOT zero the initial gauge frame, because
   `positional.py:93-100` initialises `pos_phi_free` from a hardcoded
   `* 0.01`. The user's "phi_scale near zero" sweep therefore never reaches
   the true `Ω_ij = I` regime — there is always a small positional φ.

No code-side path silently degenerates the gauge mechanism *as a function of
mu_init or phi_scale*. The opposite is true: the system carries a positional
gauge frame that ignores the user's `phi_scale` knob, and the decode path
discards gauge information unconditionally under the active config.

## Confirmed Punch List (critical → low)

### Critical
1. **Orphan-LR warning emits nothing.** `config.py:764-778`. Add
   `warnings.warn(f"VFEConfig: dead E-step LR(s): {', '.join(_dead)}.{_direct_mode_note}", UserWarning, stacklevel=2)` at the end of the `if _sigma_orphan or _phi_orphan:` block. Without this, users sweeping `e_phi_lr` or `e_sigma_lr` under `n_e_steps=1, n_layers=1` see no warning that the swept parameter is structurally dead.
2. **Decode bypasses gauge structure under active config.** `model.py:91-101, 185-201`. Either flip the active config to `use_prior_bank=True` (and `gauge_fixed_priors=True`) so the KL-to-prior decode reads sigma+phi, or document explicitly that the active configuration is a linear-decode ablation in which the gauge mechanism plays no role at the output.

### High
3. **`_update_phi` runs full autograd.grad + matrix-exp rebuild when `e_phi_lr=0`.** `e_step.py:1387, 1490-1508`. Add `if self.e_phi_lr == 0.0: return phi` as the first line of `_update_phi`. Saves one `matrix_exp` pass + one `compute_kl_attention` pass + one `autograd.grad` solve per forward.
4. **`phi_spec_max` never reaches the flat-φ path.** `e_step.py:412` reads it; only `_iter_nonflat` / `_update_phi_nonflat` / `_forward_omega_direct` consume it (lines 1836, 1876, 1934, 2083, 2089, 2141). A user setting `phi_spec_max=0.5` under the default flat path gets silent no-op. Either thread it into `compute_gauge_transport` / `fused_block_matrix_exp_pairs`, or update the field docstring to scope it explicitly to non-flat / omega-direct.
5. **`assert sbd is not None` in hot-path property.** `config.py:871`. Replace with explicit `if sbd is None: raise RuntimeError(...)`. Asserts are stripped under `python -O`.
6. **`n_e_steps < 1` guard missing from `__post_init__`.** `config.py:457-473`. Add the check alongside the LR validators; today a typo'd zero passes config construction and only errors deep in model init.
7. **`W()` cache returns tensors with freed grad_fn across consecutive forwards within one optimizer step.** `non_flat.py:269-285`. The fix is to drop the cache (the antisymmetrization is two ops; the savings versus the matrix-exp that follows are negligible) or to invalidate on any backward.
8. **NaN-sentinel `'revert'` returns a detached BeliefState.** `e_step.py:826-830, 1400-1404`. Dormant under `e_nan_check='off'`, but the only failure mode that triggers revert is exactly the one a user would want to recover gracefully from, and detached recovery silently zeros the M-step gradient through that minibatch.

### Medium
9. **`pos_phi_free` hardcoded `* 0.01` ignores `cfg.phi_scale`.** `positional.py:93-100`. Replace both sites with `* cfg.phi_scale`. This is the only place where setting `phi_scale=0` in config does not actually zero the initial gauge frame.
10. **`phi_preconditioner='killing'` uses ambient `K` instead of per-block `d_h`.** `gauge_preconditioner.py:265-281`. Documented design choice; the strict direct-sum form is available via `phi_preconditioner='killing_per_block'`. Either switch the active config or document the ambient-K convention.
11. **`decode_log_scale` placed in `m_sigma_lr` group.** `trainer.py:240-252`. Move to `m_hyper_lr` (where `log_kappa` lives) or add `m_tau_lr`.
12. **`isnan().any()` host sync inside tiled KL loop.** `gauge_utils.py:490-492, 547-549`. The `nan_to_num` already runs unconditionally; the `.any()` exists only to gate a telemetry counter. Either drop the metric or move it to a `register_buffer` accumulator updated by a fused kernel.
13. **`isotropic_covariance` invariant maintained only by retraction order.** `e_step.py:358` + KL kernel signatures. A future reorder of σ retraction relative to the KL call would silently break isotropy. Add a guard in `VFEPriorBank.encode` that projects `base_log_sigma` to its mean when `isotropic_covariance=True`.
14. **`_forward_omega_direct` docstring placed after `raise`.** `e_step.py:1981-1996, 1997`. Move the `r"""..."""` to the top of the function body.
15. **`Literal` type widened to `str` in `VFEEStep.__init__` for `e_nan_check`.** `e_step.py:411`. Use `NanCheckMode` from `_numerics` and validate the value in `VFEConfig.__post_init__`.

### Low
16. **`_SKEW_CACHE` module-global mutable state.** `attention.py:53-54, 64-79`. Move the cache onto the owning module instance; skew-detect once at `__init__`.
17. **`mass_phi` penalty does not block-respect for multi-head `gl(d_h)^H`.** `model.py:281-292`. Reshape to `(B, N, H, n_gen_per_block)` and sum per-block. Dormant under `mass_phi=0`.

## Notes on the User-Flagged Empirical Observation

Lowest PPL at `mu_init≈0`, `phi_scale≈0` is consistent with:
1. The decode-bypass finding (verdict #7) — gauge structure is structurally
   decoupled from the cross-entropy signal under the active config, so CE
   reduces purely through the `output_proj` linear head, which trains best
   from a small-norm μ canvas.
2. The positional contamination finding (verdict #6) — `phi_scale=0` does NOT
   actually zero the initial gauge frame because `pos_phi_free` carries a
   hardcoded `* 0.01`. The "phi=0" empirical regime is therefore really
   "phi=0 token, phi=0.01 positional".
3. No code path was found that degrades performance *as a function of* small
   `mu_init` or `phi_scale`. The gauge mechanism is structurally inert at
   small init (since `e_phi_lr=0.0` means φ only evolves via M-step
   backprop), and any defects identified above bite the same way regardless
   of init scale.

The empirical observation is therefore best read as: under the active
configuration, the gauge mechanism is a parameter sink rather than an
inductive bias, and the model trains best when the sink is initialised
closest to neutral. Whether the gauge structure earns its parameter cost
under a *different* configuration (e.g., `use_prior_bank=True`,
`gauge_fixed_priors=True`, `e_phi_lr>0`) is the experiment to run.
