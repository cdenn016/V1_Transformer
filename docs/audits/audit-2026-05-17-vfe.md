## Verifier-only observations

1. **`aggregate_beliefs` is dead AND has the `exact_diagonal_transport`
   gap (2.7).** Either delete it or move to a `_legacy` namespace; the
   correctness gap is currently a foot-gun for any future caller.
2. **Investigator line numbers drift by 7-10** across several findings,
   suggesting work from a stale snapshot. Numbers in the punch list
   below are verifier-corrected.
3. **Finding 2.1 (`sigma_p` autograd leak) is structurally correct but
   practically harmless** — none of the three call sites flow gradient
   from `alpha_eff` into the backward pass. The fix (detach at
   `e_step.py:262`) costs nothing and removes the foot-gun.
4. **`_last_attention` not-reset (2.6) also applies to the phi-path** at
   `e_step.py:531-533`, not only the omega-direct path the investigator
   flagged.
5. **`_decode_cache` invalidation mismatch (verifier-only):**
   `prior_bank.py:131-133` invalidate is only triggered by `model.py:128`
   when `use_prior_bank=True`, but `decode` populates the cache regardless
   of that flag. Mismatch worth a future investigation; not on the
   current punch list.

## Confirmed punch list (critical → high)

### Critical (1)

1. **[CRITICAL] F-monotone monitor fires `.item()` CUDA syncs every
   E-step iter unconditionally** — `e_step.py:468-511`. Block runs under
   `no_grad` every iteration for every layer, independent of any
   diagnostics flag. With `n_e_steps=3, n_layers=4` that's ≥12 CUDA
   stream stalls per training step.
   **Fix:** Gate the entire `f_history`/`.item()` block behind a new
   config flag `monitor_monotonicity: bool = False`.

### High (10)

2. **[HIGH] `_update_phi` recomputes full `compute_kl_attention`** —
   `e_step.py:660-669`. Doubles the dominant O(N²·K) cost per φ-update.
   **Fix:** Factor KL kernel so `(mu, sigma)`-dependent numerics are
   separable from the `phi`-dependent Omega path; rebuild only the latter
   from `phi_for_grad`.

3. **[HIGH] `_iter_nonflat` runs the full non-flat pipeline twice
   per iter** — `e_step.py:882-953` (detached for monitor + in-graph for
   autograd). Doubles the cost of pairwise Omega, two `matrix_exp`s, and
   `compute_kl_attention_pairwise`.
   **Fix:** Drop the detached forward; use `beta_g.detach()` /
   `kl_g.detach()` from the autograd pass for the monitor.

4. **[HIGH] `torch.ones_like` per-iter (B,N,N) alloc in
   `VFENonFlatConnection.forward`** — `non_flat.py:285`. Fires
   3× per E-step iter × `n_layers`.
   **Fix:** `scale = (per_edge_delta_max / fro_bound_sq.clamp(min=per_edge_delta_max**2).sqrt())`
   — pure arithmetic, no branch, no intermediate alloc.

5. **[HIGH] `sigma_p` autograd leak via `get_bayesian_alpha`** —
   `e_step.py:262`. `sigma_p = priors.sigma` (no detach). CLAUDE.md
   forbids E-step gradient writes to `sigma_p`. Currently harmless
   because every consumer of `alpha_eff` detaches first, but is a
   foot-gun.
   **Fix:** `sigma_p = priors.sigma.detach()` at line 262.

6. **[HIGH] `VFEConfig.alpha_divergence` silently ignored on
   autograd paths** — `e_step.py:111, 781-788, 911-914`. Setting α ≠ 1.0
   has no effect when `use_autograd_mu_sigma=True`,
   `use_non_flat_transport=True`, or `gauge_parameterization='omega_direct'`.
   **Fix:** Either implement Rényi-α in autograd paths or raise a
   `__post_init__` warning when `alpha_divergence != 1.0` combined with
   any autograd path.

7. **[HIGH] Dead public function `aggregate_beliefs`** —
   `attention.py:143` (110 lines). 0 production callers in
   `transformer/vfe/`; only tests reference it.
   **Fix:** Delete, or move to `transformer/vfe/_legacy.py`.

8. **[HIGH] Dead public function `aggregate_beliefs_pairwise`** —
   `non_flat.py:545`. Same pattern: 0 production callers.
   **Fix:** Delete.

9. **[HIGH] Dead config field `VFEConfig.sigma_aggregation`** —
   `config.py:72`. Inline comment admits "not used in gradient-based
   E-step." Set in `train_vfe.py` and the ablation suite, never read.
   **Fix:** Remove from `VFEConfig`, `train_vfe.py:72`,
   `vfe_ablation_suite.py:103`.

10. **[HIGH] Missing return-type on `effective_kappa`** —
    `e_step.py:222`. Returns `Union[torch.Tensor, float]`.
    **Fix:** Annotate `-> torch.Tensor | float`.

11. **[HIGH] Stringly-typed config fields without `Literal`** —
    `config.py:72,75,76,122,138` (`sigma_aggregation`, `gauge_group`,
    `phi_preconditioner`, `norm_type`, `gauge_parameterization`).
    **Fix:** Replace `str` annotation with `Literal[...]`; validate in
    `__post_init__`.
