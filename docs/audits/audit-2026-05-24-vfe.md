# Deep Audit — 2026-05-24 — `transformer/vfe/`

Five parallel expert investigators (code-reviewer, debugger, refactoring-specialist, performance-engineer, python-pro) swept `transformer/vfe/` (13,293 LOC, 28 files). A single independent verifier (general-purpose, fresh context) re-read every cited source location and confirmed/refuted each finding against executable code, not comments.

## Scope
`transformer/vfe/` — core math pipeline (`config.py`, `e_step.py`, `trainer.py`, `prior_bank.py`, `non_flat.py`, `model.py`, `block.py`, `stack.py`, `attention.py`, `efe.py`, `omega_direct.py`, `head_mixer.py`, `positional.py`, `_numerics.py`) plus secondary modules (`semantic_clustering/*.py`, `cross_coupling_*.py`, `vfe_ablation_suite.py`, `train_vfe.py`, `run_semantic_clustering.py`).

## Headline
- 37 findings raised; **30 CONFIRMED**, **2 REFUTED**, **1 INCONCLUSIVE**, **4 PARTIAL**.
- **Both originally-"critical" theory/bug claims were REFUTED** by the verifier and independently re-checked by hand:
  - The `omega_direct` pairwise-inverse "axis-swap" bug is **not a bug** — the broadcasting yields the exact matrix inverse `Ω_j exp(−δ) Ω_i⁻¹`.
  - The autograd full-cov `NotImplementedError` is **not dead code** — it is a reachable, intentional runtime rejection.
- The real action items are config-validation gaps (silent misconfiguration footguns), one security item (`weights_only=False`), and several genuine performance wins on the E-step hot path.

## Confirmed Punch List (ranked, default-config-relevant first)

**Tier 1 — silent misconfiguration footguns (fix first; cheap, high value)**
1. **[critical] `gauge_group` not validated; silent SON fallback** — `config.py:149,758-768`. A typo (`'glk'`) silently returns the wrong generator count via the `else: K*(K-1)//2` branch. Fix: validate the literal in `__post_init__`; replace the `else` with `raise ValueError`.
2. **[critical] `n_layers` not validated** — `config.py:65`. `n_layers=0` builds a silent empty stack. Fix: add `if self.n_layers < 1: raise ValueError(...)` next to the existing `n_e_steps` guard.
3. **[high] `epistemic_samples`, `kappa`, `phi_trace_clamp` unvalidated** — `config.py:256`, `e_step.py:342-349`, `prior_bank.py:153-154`. Add positive-value guards in `__post_init__` (kappa only matters when `learnable_kappa=True`).

**Tier 2 — security**
4. **[critical→practical-medium] `weights_only=False` on checkpoint load** — `run_semantic_clustering.py:89`, paired writer `trainer.py:1506`. RCE surface; in practice the user loads their own checkpoints. Fix: `weights_only=True` (checkpoint is a tensor+config dict; pass config separately).

**Tier 3 — theory/gradient consistency**
5. **[high] aux hyperparam loss uses stale `phi` under `omega_direct`** — `e_step.py:2325-2328`. Converged transport is in `omega`, but the aux loss rebuilds it from encode-time `phi`, so `log_kappa`/`raw_c0`/`raw_b0` get a gradient computed against an inconsistent transport. Only active under `gauge_parameterization='omega_direct'`. Fix: build the aux attention from the converged `omega` pairs.
6. **[medium] NO-NN constraint vs `head_mixer`** — `head_mixer.py:149`. A learned `(n,n)` projection that is not on CLAUDE.md's documented-exception list. It is opt-in (`use_equivariant_head_mixer=False` default) and identity-initialized, so it is dormant under the default config. Decision needed: either add it to the exceptions list with justification, or keep it gated and documented.

**Tier 4 — performance (E-step hot path)**
7. **[high] `_skew_cache_key` host syncs every call** — `attention.py:106,64-78`. 4× `.item()` GPU→CPU syncs to fingerprint a *static* registered buffer, on every `compute_gauge_transport` call. Fix: key on `(data_ptr, shape, dtype, device)` instead of tensor content.
8. **[high] `_iter_nonflat` double pairwise-KL pass** — `e_step.py:1839-1853`. The detached `no_grad` pass only feeds the F-monotone monitor (off by default) yet runs every non-flat E-step iteration. Fix: gate it behind `if self.monitor_monotonicity or self.track_layer_diagnostics:`.
9. **[medium] `_capture_attention_state` defaults True** — `e_step.py:392`. Rebuilds the full gauge transport (matrix_exp + inverse) on every forward's final iteration purely for plotting. Fix: default it False; the trainer already opts in on eval/plot steps.
10. **[medium] redundant `torch.stack(beta_heads)` / `.contiguous()` / post-loop KL rebuild** — `e_step.py:1039-1042`, `:980-983`, `:1476-1478`. Each a modest per-iteration waste; batch into single ops.

**Tier 5 — dead code / hygiene (lowest risk)**
11. **[high] `_original_irrep_dims` dead field** — `e_step.py:370`, `prior_bank.py:111`. Delete both assignments.
12. **[medium] `_diag_kl` duplicated** — move to `_numerics.py`, call from `non_flat.py:528-532` (also unifies eps handling).
13. **[medium] `lambda_soft` inert under default** — `e_step.py:975-976`. Add a `__post_init__` warning when `lambda_soft != 0.0 and include_attention_entropy=True`.
14. **[low] missing type hints** — `semantic_clustering/extract.py:43,121`, `pipeline.py:80,120`. Annotate `model: "VFEModel"` and `dataset`.

## REFUTED (do not action)
- **omega_direct pairwise inverse** (`omega_direct.py:215-216`) — the broadcasting yields the exact inverse `Ω_j exp(−δ) Ω_i⁻¹`; verified by hand. The investigator confused same-tensor broadcasting axes with semantic i/j indexing.
- **full-cov autograd `NotImplementedError`** (`e_step.py:1720-1728`) — reachable via standalone `use_autograd_mu_sigma=True`; an intentional runtime rejection, not dead code.
- **`_project_and_cluster` missing hints** (`pipeline.py:109-111`) — already fully annotated.
