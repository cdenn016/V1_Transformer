## Findings

### 1. Three files in `tests/` named `test_*.py` collect zero pytest items

**Location:** `tests/test_entropy_envelope.py`, `tests/test_sigma_lr_decoupling.py`, `tests/test_sigma_lr_learnable.py`
**Severity:** high
**Evidence:**
```
python -m pytest tests/test_entropy_envelope.py \
  tests/test_sigma_lr_decoupling.py \
  tests/test_sigma_lr_learnable.py --collect-only -q
# collected 0 items
```
Each file defines helpers + `main()` + an `if __name__ == "__main__":` block, but no `def test_*` and no `class Test*`. Pytest's discovery looks for those patterns (per `pytest.ini`); none match.
**Fix:** Either rename out of the `test_*.py` namespace (e.g. move to `scripts/` as `verify_sigma_lr_*.py`) or wrap `main()` in `def test_main()` so pytest actually runs them. Currently they masquerade as tests and silently never fire in CI.

### 2. `tests/transformer/test_evaluate_test_set.py` tests stdlib instead of the target module

**Location:** `tests/transformer/test_evaluate_test_set.py:46-84, 142-170`
**Severity:** high
**Evidence:**
```python
def test_dict_access(self):
    from transformer.utils.evaluation import evaluate_checkpoint
    cfg = {'vocab_size': 100, 'embed_dim': 64}
    assert cfg.get('vocab_size', None) == 100
    assert cfg.get('missing_key', 42) == 42
```
The `evaluate_checkpoint` import is unused; the test only verifies `dict.get`. `TestPerplexityComputation` (60-84) asserts `math.exp(3.0) == math.exp(3.0)`. `TestCrossEntropyCorrectness` (139-170) reimplements `F.cross_entropy` manually and asserts the reimplementation matches itself, never invoking `evaluate_checkpoint`'s PPL/BPC code path.
**Fix:** Delete the tautological tests or rewrite them to drive `evaluate_checkpoint(...)` end-to-end and assert on its returned PPL/BPC numbers.

### 3. Importing `scripts/gauge_frame_spectral_analysis.py` creates a directory as an import side-effect

**Location:** `scripts/gauge_frame_spectral_analysis.py:68-69`
**Severity:** medium
**Evidence:**
```python
SAVE_DIR = Path("./fig_spectral_analysis")
SAVE_DIR.mkdir(parents=True, exist_ok=True)
```
This runs at module-import time. `tests/test_spectral_analysis.py:11` imports symbols from this module, which causes `mkdir` in CWD on every pytest run. The repo root already contains a stray `fig_spectral_analysis/` directory from prior runs.
**Fix:** Move the `mkdir` inside `run_synthetic_validation()` / `main()` so the directory is only created when the script is executed, not when its functions are imported.

### 4. Undocumented `argparse` CLIs beyond the documented exception

**Location:** `transformer/visualization/interactive_belief_viz.py:1114`, `transformer/visualization/belief_space_viz.py:234`, `transformer/visualization/vfe_dynamics_plots.py:640`, `transformer/utils/test_query_variation.py:180`
**Severity:** medium
**Evidence:** Each file has an `argparse.ArgumentParser(...)` block at module bottom. `vfe_dynamics_plots.py:640-648` adds five `--*` flags. CLAUDE.md hard constraint: "NO CLI ARGUMENTS (with one documented exception)", the documented exception being `train_publication.py`.
**Fix:** Either remove the argparse blocks and replace with edit-the-config-dict click-to-run patterns, or explicitly enumerate these in CLAUDE.md as additional exceptions. Do not let the exception list silently grow.

### 5. Production module under `transformer/utils/` named `test_query_variation.py`

**Location:** `transformer/utils/test_query_variation.py`
**Severity:** medium
**Evidence:** A diagnostic utility module is named `test_*.py` inside `transformer/utils/`. `tests/transformer/test_query_variation.py:23` imports it as `from transformer.utils.test_query_variation import analyze_query_variation`. Under the current `testpaths = tests` setting in `pytest.ini`, it is only excluded by accident; if anyone runs `pytest` without the config (IDE auto-discovery, recursive collection), the module loads as a test module and its argparse `__main__` setup runs.
**Fix:** Rename `transformer/utils/test_query_variation.py` → `transformer/utils/query_variation.py` and update the one import site in `tests/transformer/test_query_variation.py:23`.

### 6. `tests/transformer/test_train_fast.py` leaks tempdirs per test

**Location:** `tests/transformer/test_train_fast.py:41`
**Severity:** low
**Evidence:**
```python
config.checkpoint_dir = Path(tempfile.mkdtemp())
```
`_make_trainer` is called by every test in the file; each call allocates a tempdir that is never removed. Other tests (`test_checkpoint.py`) use `tempfile.NamedTemporaryFile(delete=False)` correctly inside `try/finally`.
**Fix:** Use the pytest `tmp_path` fixture (auto-cleaned) instead of `mkdtemp()`, or wrap each call site in `with tempfile.TemporaryDirectory() as d:`.

### 7. `tests/transformer/test_integration.py` causal-mask test silently skips when attention info is missing

**Location:** `tests/transformer/test_integration.py:152-171`
**Severity:** low
**Evidence:**
```python
if 'beta' in attn_info:
    beta = attn_info['beta']
else:
    pytest.skip("No attention weights in output")
```
The only test that asserts causal masking is conditional on `'beta'` being present. If a refactor of `forward_with_attention` drops the `'beta'` key, the test silently skips and the causality regression is not caught.
**Fix:** Either `pytest.fail("attn_info missing 'beta' key")` when absent, or add a separate test that asserts `'beta' in attn_info` so removal of the key is caught.

### 8. `transformer/baselines/__init__.py` does not export `HybridGaugeTransformerLM`

**Location:** `transformer/baselines/__init__.py:9-25`
**Severity:** low
**Evidence:** `__init__.py` exports `StandardTransformerLM` and the FLOPs counters but omits `HybridGaugeTransformerLM`. `tests/transformer/test_imports.py:154` only tests `StandardTransformerLM`, so the omission is invisible to the import-coverage gate. `tests/transformer/test_hybrid_gauge_transformer.py:13` reaches around the package surface with a direct path.
**Fix:** Add `HybridGaugeTransformerLM` to `transformer/baselines/__init__.py` exports and to `tests/transformer/test_imports.py:TestBaselinesImports`, or document deliberate non-export.

## Negative findings (categories with nothing to fix)

- **Skipped tests masking broken tests:** No findings. `@pytest.mark.skipif` in `test_semantics.py:28` and `test_semantics_figures.py:33` gate optional `tiktoken`/`matplotlib` deps correctly. The brittle `pytest.skip` in `test_integration.py:155` is flagged separately as finding #7.
- **Hard-coded `device='cuda'` without `gpu` marker:** No findings. The only `cuda` reference in `tests/` is `conftest.py:38` and it guards via `torch.cuda.is_available()`.
- **Tautological assertions:** No findings beyond the `test_evaluate_test_set.py` cluster in finding #2.
- **Module/session-scope fixtures with mutation hazards:** No findings. Every `@pytest.fixture` is function-scoped.
- **Orphan imports of wave-1-deleted symbols:** No findings. `compute_transport_differential`, `_compute_dexp_generators`, `_validate_orthogonal`, `np_transport_operator` have zero references anywhere outside the wave-1 audit doc.
- **`transformer/baselines/hybrid_gauge_transformer.py` NN-constraint violation:** Acceptable as documented — the file's docstring explicitly states it is a baseline that uses neural FFN + LayerNorm + residuals deliberately, to isolate the gauge-attention contribution. CLAUDE.md treats `baselines/` as an exception class.
- **`pytest.ini` / `pyproject.toml` linter/formatter conflicts:** No findings. No Black/Ruff/pre-commit config in this repo, so no tool can rewrite manuscript text or strip LaTeX from docstrings.
- **CI tooling:** No `Makefile`, no `.github/workflows/`, no pre-commit hooks — nothing else to audit.
