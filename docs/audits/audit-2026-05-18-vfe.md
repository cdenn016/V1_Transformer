## Confirmed Punch List (critical → high → medium)

### high

1. **[high]** Missing `sigma_p.detach()` in omega-direct E-step — `e_step.py:1134`. Phi-mode at `:373` detaches; omega-direct doesn't. No actual M-step gradient leak materializes today (autograd path uses re-detached `sigma_p_d` at `:1205`), but the defensive convention is broken, and a future edit that wires `alpha_eff` into the loss directly would leak. **One-line fix: `sigma_p = priors.sigma.detach()`.**

2. **[high]** `monitor_monotonicity=False` config has no effect — `_f_monotone_step` fires every E-step iteration unconditionally for `diagonal_covariance=True` models. **3 `.item()` CUDA syncs per E-iter per layer in production.** `e_step.py:223` stores the field; `e_step.py:569` and `e_step.py:1263` call `_f_monotone_step` without reading it. Both user configs (`train_vfe.py:134`, `vfe_ablation_suite.py:191`) set `monitor_monotonicity=False` expecting this to be off. **Fix: wrap both call sites with `if self.monitor_monotonicity or self.track_layer_diagnostics:`.**

3. **[high]** `_attention_summary()` runs every training step — `trainer.py:948`. Each call is one `.tolist()` GPU→CPU sync (`trainer.py:242`). The result is consumed in two places: the tqdm postfix (which is fine — already handled by `mininterval`) and the log-interval message. **Fix: cache the last summary in a local; refresh only at `log_interval` boundaries.**

4. **[high]** Duplicate dict-literal keys in two click-to-run configs — `train_vfe.py:39-41` and `:124-126` both set `use_autograd_mu_sigma`, `use_equivariant_head_mixer`, `gauge_covariant_ridge`. Same pattern in `vfe_ablation_suite.py:96-98` and `:181-183`. Python silently keeps the later value. An ablation that toggles one of these at the first location while the second still sets `False` will look like a no-op. **Fix: delete the duplicate triple at the second location in each file.**

### medium

5. **[medium]** Two dead methods in `trainer.py` — `_get_learning_rates` (`:321-331`) and `_log_vfe_dynamics_to_csv` (`:803-852`). Zero call sites. `_log_vfe_dynamics_to_csv` writes VFE-dynamics CSV columns that the metrics dict already collects but the trainer silently drops. **Fix: either invoke `_log_vfe_dynamics_to_csv` from `train()` alongside `_log_to_csv`, or delete both methods.**

6. **[medium]** `_compute_gradient_norms()` runs every training step — `trainer.py:511`. Allocates per-group concat tensors and runs `.tolist()` every step, but the result is only emitted at `log_interval` boundaries. With `phi_embed` at V=50257, K=20, the phi-group concat is ~80 MB per step. **Fix: gate on `if (self.global_step + 1) % cfg.log_interval == 0:` (clipping has its own internal norm — independent path).**

7. **[medium]** `mu_init_std=0` silently rescales — `prior_bank.py:109`. Sweep `vfe_ablation_suite.py:294` includes `0` in the values list. The "0" data point becomes `sqrt(log(V)/K)` ≈ 0.74 — between the `1e-1` and `1` ablation points. The plotted ablation is mislabeled. **Fix: drop the `else` branch and let zero produce a degenerate prior (legitimate ablation), or remove `0` from the sweep values.**

8. **[medium]** Three type annotations contract `kappa: float` (or `Optional[Tensor]`) where callers actually pass `Tensor | float` — `attention.py:78`, `non_flat.py:415` (no annotation at all), `e_step.py:708`. Runtime works because `_kappa` is float in the common case (`learnable_kappa=False`); types are wrong. **Fix: standardize on `kappa: Union[float, torch.Tensor]` at all three signatures.**

9. **[medium]** `block_exp_pairs` recomputed inside `_update_phi` — `e_step.py:735`. Required by autograd-graph isolation (the phi leaf is fresh-detached for the local backward). **Documented limitation; not a fix in this audit's scope.** Flagged for the optimization backlog.

10. **[medium]** Non-flat path computes `compute_pairwise_omega_with_delta` twice per E-iter — `e_step.py:950, :989`. Only fires when `use_non_flat_transport=True` (non-default). **Fix: reuse `omega_pairs_det` for the monitor when `monitor_monotonicity=False` (ties back to finding #2).**

11. **[medium]** `compute_kl_attention_pairwise` allocates `torch.zeros(B,N,N,...)` every call — `non_flat.py:462`. **Fix: accept an optional `out=` buffer, or preallocate.**
