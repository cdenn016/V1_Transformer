## Theory-correctness notes

The hard CLAUDE.md invariants ‚Äî sandwich-product covariance transport (`Œ© Œ£ Œ©·µÄ`), E-step blind to targets, œÉ-LR / œÉ-trust decoupling in the active `vfe/` path ‚Äî are intact in the active codebase. The legacy `transformer/core/variational_ffn.py` path has two drifts: the LR/trust fallback (#2.2) and the RoPE wiring stub (#2.5/2.10). Neither is on the active default training path.

The diagonal-covariance approximation in `core/gauge_utils.py` keeps the diagonal of `Œ© Œ£ Œ©·µÄ` exact but discards the off-diagonal couplings the sandwich would produce. This is the documented "approximation under `exact_diagonal_transport=False`"; flagging here so the user is aware the default is the approximate path.
