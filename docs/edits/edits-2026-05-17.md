# /vfe additions ŌĆö 2026-05-17

Four new features added to `transformer/vfe/` per user request. Each is opt-in,
defaults preserve existing behavior. All gates that activate non-flat or
non-default behavior start at zero so a freshly-instantiated model with any of
the flags on is mathematically equivalent to the flag-off path at step 0
(modulo small floating-point drift from alternate compute orderings ŌĆö see
"Numerical caveats" below).

Default-flip on 2026-05-17 (separate user request): `VFEConfig.mask_self_attention`
default changed from `True` to `False`. Rationale: under causal masking, row-0
has only the diagonal as a valid target; the old `True` default required an
"escape hatch" to avoid a fully-masked row. `False` matches the active
`train_vfe.py` configs and removes the edge case from the hot path.

## Q3 ŌĆö `use_prior_bank` toggle

- `VFEConfig.use_prior_bank: bool = True` (default).
- When `False`, the decoder is `nn.Linear(K, V)` on `mu_final` (sigma is
  discarded). Encode still uses `VFEPriorBank` ŌĆö only the decode boundary
  changes. This is the one documented "neural" exception in CLAUDE.md.
- Implementation: `transformer/vfe/config.py` (new field); `transformer/vfe/model.py`
  (constructs `self.output_proj = nn.Linear(K, V, bias=False)` when off, branches
  in `forward`). Cache-invalidation call is skipped when off.

## Q4 ŌĆö Equivariant head mixer

- `VFEConfig.use_equivariant_head_mixer: bool = False` (default off).
- New module: `transformer/vfe/head_mixer.py` defining `VFEHeadMixer`.
- Per-irrep-type `A_t = I + ╬ö_t Ōłł R^{n_t ├Ś n_t}` Schur-commutant mixer applied
  as `╬╝ Ōå” kron(A_t, I_{d_t}) ┬Ę ╬╝` and `╬Ż Ōå” M ╬Ż M^T` (sandwich product). The
  `╬ö_t` parameter is stored separately from `I` so init is bitwise identity.
- Applied inside `VFEBlock.forward` after the E-step, before normalization.
- For `diagonal_covariance=True` (the default), uses the closed-form
  `Žā'[m, c] = sum_n A[m, n]┬▓ ┬Ę Žā[n, c]` ŌĆö diagonal-preserving because
  `kron(A, I_d)` only mixes the head index. Full-cov uses the exact
  per-type sandwich.
- Gauge equivariance holds under tied per-token gauges, which `/vfe` always
  satisfies (one `Žå_i` per token shared across heads via block-diagonal
  generators). `maybe_warn_independent_gauges` is a single grep-point that
  would fire if a future change ever introduces per-head gauges.

## Q7 ŌĆö Non-flat parallel transport (built from scratch)

Per user instruction: NOT a port of `transformer/core/connection.py` ŌĆö designed
from gauge-equivariance + antisymmetry first principles.

- `VFEConfig.use_non_flat_transport: bool = False` (default off).
- Two robustness knobs: `non_flat_max_strength` (global gate amplitude),
  `non_flat_per_edge_delta_max` (per-edge Frobenius clamp on `ŌĆ¢╬┤_ij ┬Ę GŌĆ¢_F`).
- New module: `transformer/vfe/non_flat.py`.
- Connection: `╬┤_ij^a = s ┬Ę (╬╝_i^T B^a ╬╝_j) / d_h` where
  - `B^a = (W^a - W^{a,T}) / 2` ŌĆö antisymmetric part of a per-generator
    bilinear; antisymmetry in `(i,j)` gives `╬┤_ji = -╬┤_ij`.
  - `W^a` is block-masked at construction so the connection respects
    block-diagonal gauge structure (each generator only touches its own
    irrep block).
  - `s = s_max ┬Ę tanh(Žü)`, `Žü` initialized at zero ŌćÆ `s = 0` at init.
- Transport: `╬®_ij = exp(Žå_i ┬Ę G) ┬Ę exp(╬┤_ij ┬Ę G) ┬Ę exp(-Žå_j ┬Ę G)`. Pairwise
  built per block in `compute_pairwise_omega_with_delta`, returning
  `(╬®_ij, ╬®_ij^{-1})` pairs via three matrix exponentials (no explicit
  `torch.linalg.inv` in the hot path).
- KL attention: `compute_kl_attention_pairwise` consumes the pairwise ╬®
  tensor; mirrors the flat kernel's masking, RoPE, and self-attention
  edge-case handling.
- Aggregation: `aggregate_beliefs_pairwise` with optional `tile_size` for
  chunked j-axis aggregation on long sequences.
- Diagnostic: `triangle_holonomy_norm` samples random `(i, j, k)` triples
  and returns the mean Frobenius norm of `╬®_ij ╬®_jk ╬®_ki - I`. Zero at
  init (flat), grows when the connection learns.
- Constraints (enforced in `__post_init__`): `diagonal_covariance=True`,
  `rope_full_gauge='off'`, and `use_autograd_mu_sigma=True` (the analytic
  gradient kernel is Žå-only; the autograd path is required).
- Integration: new `_iter_nonflat` and `_update_phi_nonflat` methods on
  `VFEEStep` that replace the existing flat KL kernel calls with the
  pairwise kernel. The standard flat path is unchanged.

## Q8 ŌĆö Pure-╬® gauge parameterization

- `VFEConfig.gauge_parameterization: str = 'phi'` (default), or `'omega_direct'`.
- Per-block `(╬®_h, ╬®_h^{-1})` pair stored on `BeliefState.omega: Optional[List[Tuple[Tensor, Tensor]]] = None`
  (new optional field; default `None` preserves all existing call sites).
- New module: `transformer/vfe/omega_direct.py`.
- Update rule: right-invariant Riemannian step on `GL_+(d_h)` per block,
  `╬®_new = ╬® ┬Ę exp(-╬Ę ┬Ę X_proj)` and `╬®_new^{-1} = exp(╬Ę ┬Ę X_proj) ┬Ę ╬®^{-1}`
  via two batched `matrix_exp` calls. `X` is the projection of the unconstrained
  `╬®^{-1} ┬Ę ŌłéF/Ōłé╬®` onto `span(G^a)` via the per-block Frobenius Gram-inverse
  (built once at construction in `_build_killing_matrix_per_block`).
- Drift control: `omega_project_slk: bool` plus `omega_normalize_every: int`
  periodically rescale each block to `det = 1` (mirrors the `phi_project_slk`
  control on the algebra side). Necessary because the Killing form on
  `gl(K) = sl(K) ŌŖĢ R┬ĘI` is degenerate on the trace direction.
- Initialization: `PriorBank.encode` runs as usual, producing `(╬╝, ╬Ż, Žå)`;
  `VFEModel.forward` then calls `init_omega_from_phi` once per forward pass
  to populate `beliefs.omega` from the post-`pos_enc` `Žå`.
- Iteration: `VFEEStep.forward` dispatches to `_forward_omega_direct` when
  the flag is on. The Žå-mode loop is untouched.
- Constraints (enforced in `__post_init__`): `diagonal_covariance=True`,
  `rope_full_gauge='off'`, `use_autograd_mu_sigma=True`.

### Known limitation for Q8

In `omega_direct` mode, `pos_enc.pos_phi_free` does not receive a gradient
from `loss.backward()`. Reason: in `omega_direct` the gauge state lives on
the group, and the E-step's iteration boundary is detached at the algebra
side. The Žå-mode autograd chain that carries `pos_phi_free` through the
`_retract_phi` step has no direct analog when the state is `╬®` rather than
`Žå`. This is documented and tested. To recover positional-encoding learning
under `omega_direct`, route an auxiliary `ŌĆ¢log ╬®ŌĆ¢`-style loss to the M-step
or accept positional encoding as a frozen feature in this mode.

## Numerical caveats

The non-flat and omega-direct paths use the per-pair ╬® kernel
(`compute_kl_attention_pairwise`) instead of the flat kernel
(`compute_kl_attention`). Both kernels compute the same mathematical
quantity (sum-of-block diagonal-of-sandwich KL between gauge-transported
Gaussians); on the same inputs they agree to ~1e-6 at fp32.

When wrapped through full-model forward passes, the cumulative drift can
reach ~0.2 in logit space because:

- The pairwise kernel materializes `╬®_ij` and squares it element-wise to
  obtain `Žā_target = ╬Ż_l ╬®_ij[k, l]┬▓ Žā_j[l]`.
- The flat kernel uses the factored form
  `Žā_target = einsum(exp_h, exp_h, einsum(exp_neg_h, exp_neg_h, Žā))`
  which has a different summation order.

The two are bitwise different but mathematically equivalent; no path is
"more correct" than the other. The drift does not affect training stability.

## Legacy mixer defect fixes (2026-05-17 follow-up)

User asked for an ultrathink audit of the legacy `transformer/core/`
`use_equivariant_head_mixer` flag (the analog of the new `/vfe` head mixer).
Three governance defects were found via parallel investigators + verifier,
and the fixes below convert each silent failure mode into a loud, traceable
signal. No mathematical changes; no behavior changes for any currently
correct config.

### Defect 1 ŌĆö dead-code equivariance warning

`attention.py:1672-1690` carries a `RuntimeWarning` meant to fire when
`use_equivariant_head_mixer=True` collapses to a generic W_O under GL(K)
multi-head with independent per-head gauges. The pre-fix predicate
included `irrep_dims_override is None`, but `model.py:338` auto-populates
`ffn_irrep_dims = self._get_effective_irrep_dims(irrep_spec)` whenever
`use_block_diagonal_kl=True` (the default), so the override is never
`None` in production. The warning was dead in every real config ŌĆö verified
empirically by trapping `warnings.warn`.

**Fix**: in `attention.py` replace the dead clause with a label-based check
that fires for both `irrep_dims_override=None` AND the auto-populated
`[d_head]*n_heads` case (both produce `glk_head_*` labels at lines
1466 / 1482), while staying silent for true cross-coupled super-blocks
(`glk_superblock_*` at line 1473).

```python
_independent_glk = (
    gauge_group == 'GLK'
    and getattr(self, 'glk_multihead', False)
    and any(lbl.startswith('glk_head_') for lbl in self.irrep_labels)
    and any(len(h) > 1 for _, _, h in self._mixer_groups)
)
```

Kept as `RuntimeWarning`, matching the codebase's convention for
"mathematically suspect but allowed to run" (e.g. `embeddings.py:999`,
`kl_computation.py:348`, `block_config.py:298`).

### Defect 2 ŌĆö silent mutual exclusion of `use_output_projection` and `use_equivariant_head_mixer`

Both flags True allocated `mixer_params` and `output_proj`; the forward at
`attention.py:2016-2022` dispatches W_O when both exist (mixer
silently dead). Verifier grep confirmed no test or production config
currently hits this combination ŌĆö zero blast radius for a hard error.

**Fix**: `block_config.py::BlockConfig.__post_init__` now raises
`ValueError` when both are True, modeled on the existing
`n_picard_steps`/`closed_form_e_step` precedent (the codebase's "allocated
but silently ignored" fail-fast pattern).

### Defect 3 ŌĆö `skip_attention=True` allocates dead mixer params

`blocks.py:483` constructs `IrrepMultiHeadAttention` (with mixer)
unconditionally; `blocks.py:774` then short-circuits the attention forward
when `skip_attention=True`. `mixer_params` end up in `state_dict`, count
toward trainable param totals, attract weight decay (if any), and never
participate in math or grad.

Option (a) ŌĆö set `self.attention = None` ŌĆö is ruled out: `blocks.py:558`,
`587`, `629`, `669`, `672` read `self.attention.<attr>` unconditionally
in `__init__` (`constant_omega`, `rope_full_gauge_mode`,
`log_kappa_per_head`, `irrep_dims`). Option (e) ŌĆö relocate mixer into the
FFN ŌĆö is also ruled out: the mixer's math is post-head-concatenation;
the VFE FFN operates on `mu_q Ōłł (B, N, K)` without a head split.

**Fix** (option b, the surgical one): `blocks.py:501` now passes
`use_equivariant_head_mixer=(cfg_flag and not self.skip_attention)`, so
`mixer_params` is never allocated when the attention forward will not
run. A `UserWarning` is emitted at `BlockConfig.__post_init__` so the
user's config-level intent is loud.

## Block-level equivariant mixer (`use_block_equivariant_mixer`) ŌĆö 2026-05-17 follow-up

Adds a second insertion point for the Schur-commutant head mixer. The
existing `use_equivariant_head_mixer` flag wires the mixer inside
`IrrepMultiHeadAttention.forward` (post-multi-head-concat); under
`skip_attention=True` that forward is bypassed and the just-merged
fix (576e4e80) force-disables the flag. The new flag wires the same
math at a different location: `GaugeTransformerBlock.forward` after the
FFN's E-step output, before the residual + norm. The mixer is now
available in the pure-VFE (`skip_attention=True`) regime.

### Math

Same as the in-attention path. For `V = ŌŖĢ_t (V_t)^{ŌŖĢ n_t}` with irrep
type `t` of dim `d_t` and multiplicity `n_t`, the gauge-equivariant
linear maps form the Schur commutant `ŌŖĢ_t M_{n_t}(R)`. We parameterize
one `A_t = I + ╬ö_t Ōłł R^{n_t ├Ś n_t}` per type (storing `╬ö_t` separately
so init is bitwise identity), assemble `M = ŌŖĢ_t kron(A_t, I_{d_t})`,
and apply

```
╬╝' = M ╬╝        ╬Ż' = M ╬Ż MßĄĆ        (full-cov: explicit sandwich;
                                    diag-cov: closed form
                                    Žā'[m┬Ęd+k] = ╬Ż_n A[m,n]┬▓ Žā[n┬Ęd+k])
```

The diagonal closed form is mathematically equal to `diag(M╬ŻMßĄĆ)` but
avoids the O(K┬▓) intermediate.

### Mutual exclusions

`BlockConfig.__post_init__` raises `ValueError` for:

1. `use_block_equivariant_mixer + use_equivariant_head_mixer` ŌĆö same
   operation, two locations. Stacking is almost certainly user error.
2. `use_block_equivariant_mixer + use_output_projection` ŌĆö W_O is the
   gauge-breaking KŌåÆK linear that the block mixer is the principled
   replacement for; reproduces the silent-dead-weight defect already
   rejected for the attention-side pair.

### Independent-gauge warning

`BlockEquivariantMixer.__init__` emits a `RuntimeWarning` when
constructed on GLK multi-head with `irrep_labels='glk_head_*'` (the
default config without `cross_couplings`). Under independent per-head
GL(d) gauges the commutant collapses to per-head scalar diagonals and
off-diagonal `A_t` entries break equivariance the same way
`use_output_projection` does. To recover the full Schur commutant,
the user must tie gauges via `cross_couplings`.

### What this is NOT

- Not a remedy for the structural restriction of the commutant under
  independent per-head gauges. Under that gauge layout, the mixer is
  still restricted to per-head scalars ŌĆö by Schur's lemma, no operator
  in the linear-map universe escapes this. The right unlock is
  `cross_couplings` (tied gauges within super-blocks), not a new
  operator.
- Not a gauge-equivariant `W_O`. `use_output_projection` remains the
  gauge-breaking option; this flag is the *replacement*, not a
  compatible variant.
- Not yet ablated empirically. Whether adding the block mixer improves
  perplexity is an open question (mechanism argument suggests yes
  under tied gauges; magnitude unknown).
