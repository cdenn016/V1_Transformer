**Scope.** Brainstorm session with the user ‚Üí design accepted ‚Üí full implementation of an upgraded `red-blue-debate` skill. The original skill was a 1-red, 1-blue, 1-judge protocol with 2 rounds. The upgrade introduces `panel=full` mode (now default), preserves `panel=lite` for backwards compatibility, and addresses three user-identified weaknesses simultaneously: single-perspective blind spots, shallow citations / thin evidence pack, and under-justified verdicts with occasional source-of-truth slippage.

**Design source.** `~/.claude/plans/gleaming-frolicking-quail.md` (the brainstorming plan file, frozen at design-approval time).

### Architecture summary

The new `panel=full` protocol dispatches a coordinator per side that dynamically picks **5 of 10 experts** for each round (philosophy-of-science mandatory). Coordinators may pick different panels ‚Äî adversarial selection is permitted. Each expert performs a dual mandate (`WebSearch`/`WebFetch` for new canon + one-page expert memo). The coordinator synthesizes one opening/rebuttal from the 5 memos and merges newly-discovered canon into `01b_extended_evidence.md` for the judges. A `debate-canon-cop` validator runs after each synthesis: grep pass + LLM pass, with a soft-cap rule (‚â•3 strikes triggers mandatory rewrite; residual strikes weighted by judges). Adjudication uses a 3-judge panel (`canon-strict`, `code-truth`, `scope`) reconciled by a `chief-judge` applying a 5-rule reconciliation algorithm.

Default settings: `panel=full rounds=2 judging=panel`. User can request `rounds=3..5` for sur-rebuttal rounds, `panel=lite` for the original cheap path, `judging=rubric` for a single-judge rubric verdict, or `experts=...` to force a specific 5-expert panel.
