## Bug Fix 1: Silently Dropped Config Keys in `experiment_runner.py`

**File**: `transformer/training/experiment_runner.py`

**Problem**: `run_single_experiment()` constructs a `TrainingConfig` object from the config dict, but several keys were never passed through. They silently fell back to `TrainingConfig` dataclass defaults, ignoring the values set in `EM_CONFIG` / `BASELINE_CONFIG`.

**Root cause of PPL discrepancy (190 ablation vs 265 train_publication)**:
- `lr_decay` was set to `'constant'` in both configs but dropped. `TrainingConfig` default `'linear'` was used instead
- With linear decay and different `max_steps` (5000 in ablation vs 15000 in train_publication), the LR at step 5000 was 10% of peak (ablation, fully annealed) vs 67% of peak (train_publication, still high)
- This explains the entire PPL gap: the ablation run had effectively completed its LR schedule while train_publication was still mid-training

| `kappa_gamma` | `1.0` | `1.0` | Accessed directly as `self.config.kappa_gamma` but was never passed |
| `use_amp` | `False` | `False` | Read by FastTrainer.__init__ but was never passed |

**3. Fixed `irrep_spec` AttributeError** (line ~676):

- Now reads from `self.model.config` (the raw config dict) with fallback to `self.config`

**File**: `transformer/core/variational_ffn.py` (line ~621)

1. `raw_lr` init now uses `mu_lr` parameter instead of hardcoded 0.1
2. `_fixed_sigma_lr = sigma_lr` is now set unconditionally (outside the if/else), so `_get_sigma_trust()` always returns the configured value

2. **sigma_ce_scale partial detach** (`prior_bank.py:590-592`): The pattern `x.detach() + s*(x - x.detach())` correctly passes fraction `s` of backward gradient while preserving the forward value.

8. **Diagonal covariance transport** (`prior_bank.py:391,422`): `einsum('...kl,l->...k', A**2, sigma)` is the exact diagonal of `A @ diag(sigma) @ A^T`.

### New Finding: Cache pollution from diagnostic probe (MEDIUM, latent)

**File:** `model.py:1215-1227`

When `_collect_layer_diagnostics=True` AND `cache_decode_priors=True` AND `gauge_fixed_priors=True` AND the model has only 1 layer: the per-layer CE diagnostic probe calls `prior_bank.decode()` inside `torch.no_grad()`, which can populate `_decode_cache` with gradient-less tensors. ### 1. Multi-Layer Interactions

#### 1a. Hierarchical Priors: `forward_with_attention` vs `GaugeTransformerStack.forward` Detach Inconsistency (MEDIUM)

**Files**: `transformer/core/model.py:1144-1145` vs `transformer/core/blocks.py:840-844`

`GaugeTransformerStack.forward()` (the `model.forward()` inference path) respects the `amortized_inference` flag:
```python
# blocks.py:840-844
if self.hierarchical_priors and not is_final:
    if self.amortized_inference:
        mu_prior = mu_q          # gradient flows through hierarchy
    else:
        mu_prior = mu_q.detach() # classical EM: freeze prior
```

But `model.forward_with_attention()` (the training path) unconditionally detaches:
```python

```

Since `amortized_inference` defaults to `True`, the training path always detaches when the inference path would not. This means during training, cross-layer gradients never flow through the hierarchical prior cascade, even when amortized inference is explicitly requested.

### 2. Closed-Form E-Step

#### 2a. Diagonal Branch Fixed Point (CORRECT)

**File**: `transformer/core/vfe_closed_form.py:128-236`

The diagonal closed-form correctly implements:
- `A = alpha * (1/sigma_p) + lambda * sum_j beta_ij * (1/sigma_j_t)` (prior precision + attention-weighted transported precision)
- `b = alpha * mu_p / sigma_p + lambda * sum_j beta_ij * (Omega_ij @ mu_j) / sigma_j_t` (prior info + transported info)
- `mu* = b / A` (precision-weighted mean)
- `sigma* = (alpha + lambda) / A` (entropy-constrained covariance)

The `(alpha + lambda)` numerator for sigma comes from setting `dF/d(sigma_q) = 0` with `sum_j beta_ij = 1` (softmax normalization). The derivation in the docstring (lines 107-111) is correct.

**Diagonal covariance transport** (`vfe_closed_form.py:181-183`):
```python
sigma_j_t_diag = torch.einsum('bijkl,bijkl,bjl->bijk', Omega_h_cf, Omega_h_cf, sigma_h)
```
This computes `diag(Omega @ diag(sigma) @ Omega^T)_k = sum_l Omega_{kl}^2 * sigma_l`, which is the exact diagonal of the sandwich product. Correct.

#### 2b. Full-Covariance Branch Fixed Point (CORRECT)

**File**: `vfe_closed_form.py:246-335`

The Q_j factorization:
```python
Q_j = exp_phi^T @ Sigma_j_inv @ exp_phi    # per-position j
r_j = exp_neg_phi @ mu_j                    # per-position j
Q_agg = sum_j beta_ij * Q_j                 # aggregated precision factor
Qr_agg = sum_j beta_ij * Q_j @ r_j         # aggregated info factor
A_align = lambda * E_i^T @ Q_agg @ E_i     # projected alignment precision
b_align = lambda * E_i^T @ Qr_agg          # projected alignment info
```

Verified algebraically (detailed derivation in audit notes) that this correctly computes `A_align = lambda * sum_j beta_ij * Sigma_j_transported^{-1}` and `b_align = lambda * sum_j beta_ij * Sigma_j_transported^{-1} @ Omega_ij @ mu_j`. The factorization exploits the product structure `Omega_ij = exp(phi_i) @ exp(-phi_j)` to aggregate `Q_j` before applying the per-query exp(-phi_i) projection, which is an O(N) optimization vs naive O(N^2) matrix inversions.

#### 2c. Softmax Coupling Asymmetry (CONFIRMED, DOCUMENTED)

**File**: `vfe_closed_form.py:29-38`

The diagonal branch includes `S_mu_h`, `c_mu_h`, `S_sigma_h` corrections that account for `d(beta)/d(mu)` -- the softmax coupling between beta weights and the belief means. These terms arise because beta_ij depends on KL(q_i || Omega_ij q_j), which depends on mu_i. The full-covariance branch does NOT include these terms in the initial closed-form solve. Instead, it delegates softmax coupling refinement to the Picard re-solve.

This is explicitly documented at lines 29-38:
> "The diagonal branch additionally absorbs the softmax-coupling correction (S_mu, c_mu, S_sigma terms) into the fixed point directly. The full-covariance branch uses a linear-only closed form and delegates the softmax coupling to the optional Picard re-solve."

**Impact**: When `n_picard_steps=0` (the default), the full-cov branch ignores softmax coupling entirely. For the diagonal branch, the coupling is always included when `lambda_softmax > 0`. This asymmetry means full-cov and diagonal closed-form can give different answers even when the true covariance is diagonal. Not a bug per se, but the full-cov branch is less accurate at the single-shot level.

#### 2d. Full-Covariance Sigma Clamp (MINOR ISSUE)

**File**: `vfe_closed_form.py:327`
```python
Sigma_star_h = Sigma_star_h.clamp(max=ffn.sigma_max)
```

Element-wise clamping of a full covariance matrix at `sigma_max` can destroy positive-definiteness. If off-diagonal elements exceed `sigma_max` but diagonal elements do not (or vice versa), the result may not be SPD. The diagonal branch avoids this because element-wise clamping of a diagonal covariance preserves positivity.

**Severity**: LOW. In practice, `Sigma_star_h = (alpha + lambda) * A_inv_h` where `A_inv_h` is already SPD, so the scaling should produce reasonable off-diagonal magnitudes. But the clamp is not SPD-preserving in general.

#### 2e. Picard Iterations (CORRECT)

**Diagonal branch** (`vfe_closed_form.py:342-458`):
- Beta IS recomputed from the current mu/sigma at each Picard step (line 373-385).
- Convergence criterion: relative change `< 1e-4` with early break (line 456-458). Correct.

## Edit: Add Rényi alpha-divergence support to `kl_computation.py`

**File**: `transformer/core/kl_computation.py`

**Purpose**: Extend the KL computation module to support Rényi alpha-divergences for the diagonal covariance mode, enabling attention weights based on D_alpha(q_i || Omega_ij[q_j]) as an alternative to the standard KL.

### Mathematical basis

For diagonal Gaussians with per-dimension variances s_k (query) and t_k (transported key), the Rényi order-alpha divergence has a closed form. Defining the blended variance:

    sigma_blend_k = (1 - alpha) * s_k + alpha * t_k

the divergence is:

    D_alpha(q || p) = 0.5 * [alpha * sum_k (delta_k^2 / sigma_blend_k)
                              + (1/(alpha-1)) * sum_k ((1-alpha)*log(s_k) + alpha*log(t_k) - log(sigma_blend_k))]

The limit alpha -> 1 recovers the standard KL. Special values: alpha = 0.5 gives the Bhattacharyya distance (symmetric, bounded by log 2 per dimension), alpha in (0, 1) are mode-covering, alpha in (1, 2] are tail-sensitive.

Reference: Rényi (1961), "On measures of entropy and information", Proc. 4th Berkeley Sympos. Math. Statist. Probab.

### Changes made

**1. `_kl_kernel_dense`** -- added `alpha_div: float = 1.0` parameter. When `alpha_div == 1.0`, code path is completely unchanged (guard is a single `if` before the existing `K = mu_q.shape[-1]` line).

**2. `_kl_kernel_diagonal`** -- added `alpha_div: float = 1.0` parameter with full closed-form Rényi implementation:

- When `alpha_div != 1.0`: computes sigma_blend, mahal_term (alpha-scaled), and logdet_term (the log-det of the mixture divided by (alpha-1)), all inside the existing `torch.amp.autocast('cuda', enabled=False)` float32 context
- `sigma_blend` is clamped to `eps` (essential for alpha > 1 where the blend can go negative)
- Output passes through `safe_kl_clamp(kl, kl_max)` just like the existing KL path

**3. `compute_kl_matrix`** -- added `alpha_divergence: float = 1.0` parameter threaded to `_compute_unchunked` and `_compute_chunked` as keyword `alpha_div=alpha_divergence`. ## Feature: Rényi α-Divergence Parameter in Diagonal VFE Gradient Functions

**File**: `transformer/core/vfe_gradients.py`

**Purpose**: Add `alpha_div: float = 1.0` to the diagonal covariance VFE gradient functions so that the self-coupling and belief-alignment terms use D_alpha (Rényi α-divergence) instead of KL. When `alpha_div = 1.0` the execution path is bit-for-bit identical to the original code.

### Mathematical background

For diagonal Gaussians with per-dimension variances s_k = sigma_q_k (query) and t_k = sigma_p_k (prior or transported key), the blended variance under Rényi order alpha_d is:

    sigma_blend_k = (1 - alpha_d) * s_k + alpha_d * t_k

The mu gradient of D_alpha(q||p) per dimension k:

    alpha_d * delta_mu_k / sigma_blend_k

The sigma-q gradient per dimension k:

    -alpha_d * (t_k - s_k) / (2 * s_k * sigma_blend_k)
    - alpha_d * (1 - alpha_d) * delta_mu_k^2 / (2 * sigma_blend_k^2)

    D_alpha = 0.5 * [ alpha_d * sum_k delta_mu_k^2 / sigma_blend_k

where logdet_blend_term = sum_k ((1-alpha_d)*log(s_k) + alpha_d*log(t_k) - log(sigma_blend_k)) / (alpha_d - 1).

At alpha_d -> 1 this recovers KL(q||p) by L'Hopital on the log-det term.

## New Module: `transformer/analysis/fiber_trajectory.py`

**File**: `transformer/analysis/fiber_trajectory.py` (new, 533 lines)

**Purpose**: Fisher-Rao geometry module for offline analysis of VFE E-step iteration trajectories on the diagonal Gaussian belief manifold. Operates entirely on numpy arrays; no torch dependency, no matplotlib.

### Mathematical basis

The fiber at each token position is the space of diagonal Gaussians $\mathcal{G}_K = \{N(\mu, \text{diag}(\sigma^2))\}$ equipped with the Fisher-Rao (information-geometric) metric:

$$ds^2 = \sum_k \left[\frac{d\mu_k^2}{\sigma_k^2} + \frac{(d\sigma_k^2)^2}{2\sigma_k^4}\right]$$

The closed-form 1-D geodesic (Atkinson & Mitchell, 1981) is implemented in `fisher_rao_distance` via the Poincaré half-plane embedding:

$$d_k = \sqrt{2}\,\text{arccosh}\!\left(1 + \frac{(\Delta\mu_k)^2 + (\Delta\sigma_k)^2}{2\,\sigma_{1,k}\,\sigma_{2,k}}\right)$$

Per-dimension distances are combined in quadrature to give the full K-dimensional geodesic.

|---|---|---|
| `_to_std(sigma_diag)` | `(..., K)` | Convert stored variances to std devs |
| `_extract_token(arr, idx)` | `(K,)` | Row accessor, raises `IndexError` cleanly |

| `fisher_rao_distance(mu1, s1, mu2, s2)` | `(N,)` | Closed-form geodesic, `acosh` arg clamped to ≥1 |
| `compute_arc_length(snapshots, token_idx)` | `(T,)` | Cumulative arc, first entry 0.0 |
| `compute_velocity_profile(snapshots, token_idx)` | `(T-1,)` | Per-step speeds; empty for T=1 |
| `compute_convergence_curve(snapshots, token_idx)` | `(T,)` | Distance to final state at each iteration |

`FiberTrajectoryStats` dataclass fields: `token_idx`, `arc_length`, `geodesic_distance`, `deviation_ratio`, `convergence_rate`, `mean_velocity`, `final_velocity`, `mu_displacement`, `sigma_ratio`.

### Edge cases handled

- Single-snapshot trajectories: `compute_arc_length` returns `[0.0]`, `compute_velocity_profile` returns empty array, `compute_convergence_curve` returns `[0.0]`, all higher-level functions degrade gracefully.
- Zero geodesic distance (stationary trajectory): `compute_geodesic_deviation` and `analyze_fiber_trajectory` return `deviation_ratio = 1.0`.
- `acosh` domain: argument clamped to ≥ 1.0 element-wise before `np.arccosh`.
- Zero standard deviation: `denom` in `fisher_rao_distance` guarded at `1e-24`; midpoint std in `fisher_rao_infinitesimal` guarded at `1e-12`.
- `fit_convergence_rate`: positive mask eliminates exact-zero convergence entries before log; returns 0.0 if fewer than 2 positive values remain.
- `sigma_ratio`: dimensions with zero initial std are excluded from the geometric mean; returns 1.0 if all initial stds are zero.

### Design notes

- `sigma_diag` in `IterationSnapshot` stores variances (not std devs), consistent with `TrajectoryRecorder._sigma_to_diag` throughout the codebase. All metric computations convert to std dev internally via `_to_std`.
- `fisher_rao_infinitesimal` and `fisher_rao_distance` are distinct functions with different contracts: the former is a first-order trapezoidal approximation suitable for very small steps; the latter is the exact geodesic and should be preferred for arc-length accumulation and deviation ratios.
- No internal calls to `fisher_rao_infinitesimal` from trajectory-level functions — all trajectory accumulation uses the exact geodesic.

- `_pca_2d(X)` — SVD-based PCA to 2 components with rank-deficiency padding.
- `_compute_velocities(snapshots, token_row)` — midpoint Fisher-Rao metric velocities (T-1,).
- `_compute_convergence_curve(snapshots, token_row)` — closed-form geodesic to final state (T,).
- `_fit_log_linear(y)` — OLS fit of log y ≈ a − λt, returns fitted curve + λ.
- `_safe_legend(ax, ...)` — suppresses matplotlib UserWarning on empty legend.
- `_draw_phase_portrait_on_ax(ax, ...)` — dashboard-internal version of the phase portrait that draws into an existing Axes.

### Design notes

- `MATPLOTLIB_AVAILABLE` guard follows the pattern in `holonomy_plots.py` and `vfe_dynamics_plots.py`; all plot functions return `None` immediately if matplotlib is absent.
- The Fisher-Rao coordinate embedding uses variances (the `sigma_diag` field carries variances throughout the codebase); all internal conversions go through `np.sqrt(np.maximum(sigma_diag, 1e-24))`.
- Velocity computation uses the midpoint-covariance approximation specified in the task (std_mid = (std1 + std2)/2), consistent with `fisher_rao_infinitesimal` in `fiber_trajectory.py`.
- Convergence curves use the exact Atkinson-Mitchell (1981) geodesic rather than the midpoint approximation, consistent with `compute_convergence_curve` in `fiber_trajectory.py`.
- PCA is computed jointly over all requested token trajectories so phase-portrait coordinates are shared — different tokens can be compared in the same 2D embedding.
- `os` is imported but not used in the final implementation (superseded by `pathlib.Path`). It can be removed in a future cleanup pass.

## Feature: `transformer/analysis/gauge_geometry.py` — Gauge Geometry Analysis Module

### Files created

**`transformer/analysis/gauge_geometry.py`** — New module implementing ten functions for
Yang-Mills energy, curvature tensor extraction, and gauge orbit analysis operating on live
PyTorch tensors. All functions run inside `torch.no_grad()` and upcast to float32.

**`transformer/analysis/__init__.py`** — Updated module docstring and `__all__` to export
all ten new public functions.

### Functions implemented

**`extract_curvature_tensor(C, method, series_order)`** — Extracts $F = \log C$ from
holonomy matrices $(B, n\_triples, K, K)$ via two numerical paths. The `'logm'` path
uses `torch.linalg.eig` for complex eigendecomposition, clamps eigenvalue moduli away from
zero, constructs the complex log, then solves $V^T X = (V \operatorname{diag}(\log\lambda))^T$
for numerical safety and returns the real part. The `'series'` path uses the Mercator series
$\log(I+X) = \sum_{n\ge1} (-1)^{n+1} X^n / n$, appropriate only when $\|C - I\|_F \ll 1$.

**`compute_yang_mills_energy(F)`** — $E_{\text{YM}} = \frac{1}{2}\sum_t \|F_t\|_F^2$,
summing Frobenius norm squared over triples, returning a $(B,)$ tensor.

**`compute_yang_mills_energy_from_holonomy(exp_delta, sample_size, method)`** — End-to-end
pipeline calling `compute_holonomy`, `extract_curvature_tensor`, `compute_yang_mills_energy`,
and `decompose_curvature` to produce energy and a diagnostics dict with `mean_F_norm`,
`max_F_norm`, `mean_holonomy_norm`, `abelian_fraction`.

**`compute_gauge_field_energy(phi, beta, generators, metric)`** — Dirichlet energy
$E_D = \sum_{ij} \beta_{ij} \|\phi_i - \phi_j\|^2_g$ on the token graph. For
`'frobenius'` uses $\|\Delta\phi\|^2 = \sum_a (\Delta\phi^a)^2$. For `'killing'` builds
the Killing form gram matrix $G_{ab} = 2K\operatorname{tr}(T_a^T T_b) - 2\operatorname{tr}(T_a)\operatorname{tr}(T_b)$
from generators and contracts via `einsum`.

**`decompose_curvature(F)`** — Splits $F \in \mathfrak{gl}(K)$ into abelian part
$F_{\text{ab}} = (\operatorname{tr}F/K) I$ and traceless non-abelian remainder
$F_{\text{nab}} = F - F_{\text{ab}}$. Returns dict with both components, their
individual Yang-Mills energies, and the abelian fraction clamped against division by zero.

**`gauge_fix_axial(mu, sigma, phi, generators, anchor_idx)`** — Axial gauge: applies
$g = \exp(-\phi_{\text{anchor}} \cdot G)$ to all beliefs, zeroing $\phi$ at the anchor
token. Diagonal covariance transported as $(g \operatorname{diag}(\sigma) g^T)_{kk} =
\sum_l g_{kl}^2 \sigma_l$ via element-wise-squared $g$ einsum.

**`gauge_fix_coulomb(mu, sigma, phi, generators)`** — Coulomb gauge: subtracts
$\bar\phi = \frac{1}{N}\sum_i \phi_i$ so the mean is zero. Uses same matrix-exp
and einsum patterns as axial fix.

**`compute_gauge_invariants(mu, sigma, phi, generators, beta)`** — Returns
`det_omega` via $\det(\Omega_{ij}) = \exp(\operatorname{tr}(\phi_i\cdot G) - \operatorname{tr}(\phi_j\cdot G))$,
`gauge_frame_spectrum` (eigenvalue moduli of $\exp(\phi_i\cdot G)$), and
`gauge_field_energy`.

**`gauge_orbit_sample(mu, sigma, phi, generators, n_samples, perturbation_scale)`** —
Samples $n$ random orbit points by drawing $\epsilon \sim \mathcal{N}(0, \sigma_\epsilon^2 I)$,
building $g = \exp(\epsilon \cdot G)$, and applying to all tokens. Returns list of
`{'mu', 'sigma', 'phi', 'g'}` dicts.

**`compute_gauge_orbit_dimension(phi, generators, eps)`** — Estimates the effective
dimension of the gauge orbit by building the $(N K^2) \times n\_gen$ Jacobian matrix of
the right-invariant gauge action $J_{ia} = \exp(\phi_i \cdot G) \cdot T_a$, then counting
singular values above the threshold. SVD fallback uses $J^T J$ for tall matrices.

### Internal helpers

`_to_float32(t)` — casts any dtype to float32 (passthrough for float64).
`_build_phi_matrix(phi_vec, generators)` — batch einsum `...a,aij->...ij` contracts
coefficients with generators for arbitrary leading dimensions.

### Design notes

The `logm` matrix log uses `torch.linalg.solve` on the complex eigenvector matrix rather
than an explicit inverse, avoiding condition-number amplification from `pinv`. The
`decompose_curvature` energy partition is consistent with `compute_yang_mills_energy`:
$E_{\text{ab}} + E_{\text{nab}} = E_{\text{YM}}$ exactly. Gauge fix functions return
the original input dtype to avoid silent upcast in downstream code. The Killing metric
formula is the standard one for $\mathfrak{gl}(K)$ in the defining representation; for
traceless generators ($\operatorname{tr}(T_a) = 0$) it reduces to $2K\operatorname{tr}(T_a^T T_b)$,
recovering the standard $\mathfrak{sl}(K)$ Killing form up to a factor.
