## Session 8: Full Covariance GL(K) + RoPE Full Gauge Implementation

### Full covariance support (diagonal_covariance=False)

Removed NotImplementedError guard. VFEPriorBank now produces exact full covariance:

- Diagonal path (unchanged): `diag(A diag(s) A^T) = sum_j A_ij^2 * s_j`
- Full-cov path (new): `Sigma_h = A_h @ diag(s_h) @ A_h^T` (exact sandwich product per block), assembled into block-diagonal `(K, K)` matrix

Decode always uses diagonal KL for O(V*K) efficiency — extracting diagonal from full-cov at decode boundary only.

### RoPE full gauge (rope_full_gauge=True)

Framework-consistent RoPE: R(θ) acts on BOTH μ→Rμ AND Σ→RΣR^T. Requires `diagonal_covariance=False`.

Dispatches to per-head autograd path `_compute_rope_full_gauge_gradient_per_head()`. Falls back during inference mode (generation).

### Three covariance transport modes

| Mode | Config | Exactness |
|------|--------|-----------|
| Diagonal approx | `diagonal_covariance=True` | Approximate for GL(K) |
| Exact diagonal lift | `diagonal_covariance=True, exact_diagonal_transport=True` | Exact transport, diagonal storage |
| Full covariance | `diagonal_covariance=False` | Exact |

## Session 11: Fix enforce_orthogonal silently ignored for d < 16 blocks

### Problem

Newton-Schulz orthogonalization was gated on `d >= 16` in `fused_block_matrix_exp_pairs` and `K >= 16` in three other files. ### Changes

Newton-Schulz works for any d >= 2 (d=1 is scalar, no rotation). The threshold of 16 was arbitrary.

- `transformer/core/gauge_utils.py:203` — docstring `d >= 16` → `d >= 2`
