## Principled EFE integration into natural-gradient VFE geometry (Stage A)

**Problem**: The EFE (Expected Free Energy) and bootstrap distillation terms were applied as heuristic Euclidean μ-only corrections outside the natural-gradient chain. Three issues: (1) wrong geometry — EFE used Euclidean gradients on a Riemannian belief manifold, (2) incomplete derivatives — only ∂G/∂μ was computed, missing ∂G/∂Σ and ∂G/∂φ, (3) internal inconsistency — observation CE gradient was already in the natural gradient path while structurally identical EFE was not.

The code comment "EFE terms do not have a Fisher-metric justification w.r.t. mu" was incorrect. The Fisher metric F(θ) is a property of the belief parameterization q_θ = N(μ, Σ), not the loss function (Amari 1998). The natural gradient ∇̃f = F⁻¹·∇f applies to ANY scalar f(θ).

### Changes

**`transformer/core/block_config.py`**: Added `active_inference_unified_natgrad: bool = True` config field. When True (default), EFE/distillation gradients are folded into VFE grad_mu/grad_sigma before Fisher projection. When False, legacy Euclidean path is preserved.

**`transformer/core/active_inference.py`**:
- `_compute_active_inference_gradient` now returns `(grad_mu, grad_sigma)` tuple. Σ is tracked through autograd in both pragmatic and epistemic terms. Pragmatic: sigma_var tracked through readout (PriorBank gets non-zero ∂H/∂Σ, linear W_out gets zero). Epistemic: sigma tracked through reparameterization trick `mu_s = mu + sqrt(sigma_var) * noise` (diagonal case) and through readout (both cases). This captures the explore-exploit gradient ∂MI/∂Σ.
- `_compute_distillation_gradient` now returns `(grad_mu, grad_sigma)` tuple. Sigma tracked through the local readout.
- `compute_ai_gradients` returns 4-tuple: `(grad_efe_mu, grad_efe_sigma, grad_distill_mu, grad_distill_sigma)`.
- `configure_ffn_active_inference` propagates `_ai_unified_natgrad` to FFN instance.
- `apply_ai_mu_updates` docstring corrected: marked as LEGACY path, corrected the incorrect Fisher metric claim.

**`transformer/core/variational_ffn.py`**:
- `_vfe_iteration`: When `_ai_unified_natgrad=True`, folds EFE and distillation Euclidean gradients into `grad_mu` and `grad_sigma` BEFORE the Fisher natural-gradient projection (line ~2095). The existing `apply_ai_mu_updates` call receives (None, None) and is a no-op.
- `_compute_phi_grad`: When unified natgrad + distillation active, adds ∂L_distill/∂φ to alignment_loss. Uses reversed direction: CE(readout(μ̃(φ)), sg[readout(μ)]) — transport adapts so transported prediction matches local truth. Stop-gradients on β and μ preserved (BYOL collapse prevention). Only Ω_ij(φ) tracks phi_for_grad. Function signature extended with `beta_current` and `beta_heads` parameters; both call sites updated.

### Theoretical status

This is Stage A: geometric consistency for the one-step EFE approximation. Without a generative model p(s_{t+1}|s_t, π) and multi-step rollout, this remains a one-step EFE approximation, not full active inference. Stage B (EFE on transported beliefs with explicit policy selection) and Stage C (horizon-H rollout) are documented in the plan but not yet implemented.

## Canonical active inference: expected free energy module

**New file**: `transformer/core/expected_free_energy.py`

Implements the canonical active inference formulation from ACTIVE_IDEA.md Section 11. One-step expected free energy G_t(a) = Risk + Ambiguity - Epistemic over candidate token actions, with policy posterior q(a) ∝ exp(-γ G(a)).

### Phase 1A: Generation-time active inference

- `rollout_candidates()`: Batched one-step belief rollouts — appends each of K candidate tokens to context, runs model.forward in a single (K, T+1) batch, extracts last-position beliefs
- `compute_risk()`: Cross-entropy with preference distribution. Three modes: `current_belief` (model's own prediction as preference), `target` (delta preference, risk = -log q(y|a)), `uniform` (ablation)
- `compute_ambiguity()`: Entropy of readout at rolled-out beliefs
- `compute_epistemic_value()`: BALD MI via MC sampling — optional, off by default (structurally weaker for self-generated text)
- `compute_efe()`: Orchestrates rollout + all EFE terms + policy posterior
- `generate_active_inference()`: Full autoregressive generation loop using EFE-based token selection

Uses `model._compute_logits()` for readout (respects PriorBank tau, reflection-aware linear), NOT a custom closure. Uses `torch.no_grad()` (not `@torch.inference_mode()`) to preserve future autograd extensibility.

### Phase 1B: Teacher-forced EFE proxy loss

- `efe_teacher_forced_proxy_loss()`: Surrogate training loss using adjacent-position beliefs as rollouts. Position t+1's beliefs approximate the rollout for action y_{t+1}. Risk = -log q(y_{t+2} | z_{t+1}). Labeled explicitly as proxy, not genuine active inference. Consumes mu_q/sigma_q from `forward_with_attention()`, compatible with implicit_em regime.

### model.py change

Added `generate_active_inference()` delegator method to `GaugeTransformerLM` (~line 1519). Lazy import pattern matching existing delegators (hebbian.py, etc.).

### Key design decisions

1. Risk formula with delta preferences: special-cased as -log q(y|a) to avoid -log(0) divergence (ACTIVE_IDEA.md formula direction is inconsistent with its own textual description)
2. Epistemic term off by default: `include_epistemic=False` — BALD MI is structurally weaker without exogenous observations
3. Compute bottleneck accepted: O(K·T²·L·I) per generated token from prefix recomputation, ~50x standard generation cost
4. Detached agents from return_agents=True: correct for Phase 1A scoring, noted as constraint for future differentiable planner

## Decompose _vfe_iteration into 6 private helper methods

**File**: `transformer/core/variational_ffn.py`

**Problem**: `_vfe_iteration` on `VariationalFFNDynamic` was 1053 lines — a single method containing debug printing, observation gradients, SPD retraction, diagnostics collection, multi-head VFE gradient computation, and single-beta VFE gradient computation all interleaved. Any change to one concern required reading the entire method.

**Changes**: Six private methods extracted onto `VariationalFFNDynamic`, placed immediately before `_vfe_iteration` in the file. All mathematical logic, tensor operations, detach patterns, clamp values, and trust regions are exactly preserved.

1. **`_print_vfe_grad_debug`**: The 142-line debug printing block (Euclidean/natural gradient breakdown, per-head multihead table, metrics aggregation). Reads `_vfe_utils_mod._VFE_GRAD_DEBUG`; resets it to `None` on exit. Side-effect only: no return value.

2. **`_compute_observation_gradients`**: The 68-line CE + Stein's lemma block. Computes `obs_grad_mu` (from `(softmax(W_out μ) - one_hot) @ W_out`) and optionally `obs_grad_sigma` (Hessian diagonal of E_q[CE] via Var_softmax[W_k]). Returns `(obs_grad_mu, obs_grad_sigma)` where `obs_grad_sigma` is `None` when `self.obs_sigma_gradient` is False.

3. **`_retract_sigma`**: The 96-line SPD retraction + condition clamping + isotropic enforcement block. Wraps `retract_spd_diagonal_torch` / `retract_spd_torch`, then clamps eigenvalue ratio to ≤10 (geometric-mean clamping for diagonal, ridge regularization for full), then collapses to σ²I when `self.isotropic_covariance`. Returns updated `sigma_current`.

4. **`_record_iteration_diagnostics`**: The 77-line diagnostics + fiber snapshot block. Appends to `self._iteration_diagnostics` (when `self._collect_iteration_diagnostics`) and `self._fiber_snapshots` (when `self._record_fiber_trajectory`). The `'beta_current' in locals()` pattern was replaced with `beta_current is not None` (equivalent logic since `beta_current` is now an explicit parameter). Side-effect only: no return value.

5. **`_compute_multihead_vfe_gradients`**: The 190-line per-head β_h loop. Handles all three per-head paths (rope_full_gauge, fused diagonal block-diagonal, fallback separate attention+gradient), accumulates per-head debug prefixes in `_VFE_GRAD_DEBUG`. Returns `(grad_mu, grad_sigma, beta_heads, beta_current, beta_history_entry, _mh_cached_bep)`. The `_mh_cached_bep` is returned so the phi-gradient and active-inference helpers downstream can reuse it.

6. **`_compute_single_beta_vfe_gradients`**: The 83-line single-β path. Handles fused and fallback paths for the non-multihead case. Returns `(grad_mu, grad_sigma, beta_current, beta_history_entry, _cached_bep)`.

**Result**: `_vfe_iteration` reduced from 1053 lines to ~490 lines of orchestration. All six helper methods are documented with LaTeX docstrings matching the codebase convention.

## VFE hot path performance fixes (4 changes)

**File**: `transformer/core/variational_ffn.py`

### Fix 1: Lazy imports moved to module level

Four sets of lazy imports inside hot-path methods were moved to module level, eliminating repeated `importlib` lookups on every forward pass.

- `apply_cartan_preconditioning`, `apply_killing_form_natural_gradient`, `apply_pullback_natural_gradient` from `transformer.core.gauge_preconditioner` — were re-imported on every call to `_precondition_phi_grad` for non-clip modes. Added to a new module-level `from transformer.core.gauge_preconditioner import (...)` block after the vfe_gradients import. The three `from ... import` lines inside the `if/elif/elif` branches of `_precondition_phi_grad` were removed.

- `compute_attention_weights` from `transformer.core.attention` — was redundantly re-imported inside `_compute_omega_grad_direct` (line ~897) despite being imported at module level (line 97). The redundant lazy import was deleted.

- `compute_transport_operators_direct` from `transformer.core.transport_ops` — was lazily imported inside `_vfe_iteration` on every iteration. Added to the existing line 98 module-level import alongside `compute_transport_operators`. The lazy import inside `_vfe_iteration` was removed.

- `_compute_rope_full_gauge_gradient_per_head` from `transformer.core.vfe_gradients` — was lazily imported inside the `if _use_rope_full:` branch of `_compute_multihead_vfe_gradients` on every multihead iteration. Added to the existing module-level `from transformer.core.vfe_gradients import (...)` block. The `from ... import (...)` block inside the `if` guard was removed (the guard itself remains).

### Fix 2: Remove unnecessary `.clone()` in early-exit convergence check

`_mu_prev = mu_current.detach().clone()` changed to `_mu_prev = mu_current.detach()`. The clone is unnecessary because `mu_current` is a fresh tensor produced by the update step each iteration (never mutated in-place), so `detach()` alone safely breaks the graph without allocating a copy.

### Fix 3: Remove unnecessary `.clone()` on beta history entries

Two sites:
- `beta_history_entry = beta_stacked.detach().clone()` in `_compute_multihead_vfe_gradients` changed to `.detach()`.
- `beta_history_entry = beta_current.detach().clone()` in `_compute_single_beta_vfe_gradients` changed to `.detach()`.

`beta_stacked` and `beta_current` are fresh tensors from `torch.stack` / `softmax` respectively. `detach()` breaks the graph; `.clone()` adds a redundant allocation that is never needed since these tensors are only read (stored in history, never mutated).

### Fix 4: Pass precomputed BEP to single-beta path

The most impactful fix. Previously `_hoisted_bep` (block-exp-pairs, the expensive `matrix_exp` computation) was precomputed in `_forward_vfe_body` only for `self.multihead_vfe=True`. The single-beta path recomputed it fresh on every iteration via `_build_block_exp_pairs` inside `_compute_single_beta_vfe_gradients`, wasting `n_iters - 1` redundant `matrix_exp` calls per forward pass.

Three coordinated changes:
1. `_compute_single_beta_vfe_gradients` signature gained `_precomputed_block_exp_pairs=None`. Inside the method, `_cached_bep = self._build_block_exp_pairs(...)` is now guarded: if `_precomputed_block_exp_pairs is not None`, use it directly; otherwise call `_build_block_exp_pairs`.
2. The `_vfe_iteration` call to `_compute_single_beta_vfe_gradients` now passes `_precomputed_block_exp_pairs=_precomputed_block_exp_pairs`, threading the hoisted value through.
3. The `_hoisted_bep` precomputation condition in `_forward_vfe_body` was changed from `... and self.multihead_vfe` to just `... and not self.update_phi_per_iteration` (the multihead guard removed), so the single-beta path also benefits from the hoist when `n_iters > 1` and phi is frozen across iterations.
