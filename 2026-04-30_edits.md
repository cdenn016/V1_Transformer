# 2026-04-30 Edits

## CJK font rendering: persistent per-glyph fallback for wiki-ja attention plots

**File:** `transformer/training/experiment_runner.py` (`_setup_cjk_fonts`, ~line 545)

### Bug

Training on `wiki-ja` produced `Glyph NNNNN ... missing from font(s) DejaVu Serif`
warnings starting around step 6000. Steps 2000 and 4000 rendered correctly. The
old `_setup_cjk_fonts` set `font.family = 'sans-serif'` and prepended a CJK font
to `font.sans-serif`, then cached `_cjk_fonts_configured = True` to bail on
subsequent calls.

The reset path: `transformer/visualization/pub_style.py::set_pub_style` writes
`rcParams.update({'font.family': 'serif'})`. `analysis/semantics.py::run_semantic_analysis`
calls `set_pub_style()`, and the user's config in
`transformer/config_values_for_settings.py:158` runs that semantic pass every
4000 steps. Order of operations within a step: validation -> attention viz ->
semantic analysis. So the step-4000 attention viz ran *before* the step-4000
semantic analysis (still on `'sans-serif'` from `_setup_cjk_fonts`), then
semantic analysis flipped `font.family` back to `'serif'`, and the cached flag
prevented re-applying the CJK setup at step 6000. matplotlib resolved
`font.family='serif'` -> `font.serif[0] = 'DejaVu Serif'`, which has no CJK
glyphs, hence the warnings.

### Fix

Use matplotlib 3.6+ per-glyph fallback: set `font.family` to the list
`['serif', cjk_font_name]`. ASCII renders via DejaVu Serif (publication
quality preserved), CJK glyphs fall back per-glyph to the first available CJK
font (`MS Gothic` etc.). Re-apply on every `_setup_cjk_fonts` call (no cache)
so subsequent `set_pub_style()` resets self-heal at the next attention viz tick.

The "no CJK font available" branch still suppresses warnings, but uses a
one-shot guard (`_cjk_warnings_silenced`) instead of the old configured-flag,
so the per-glyph fallback path keeps re-applying.

### Verification

- Entry path traced: `save_attention_visualization` (line 573) reads
  `self.train_loader.dataset.dataset_name` (set in
  `transformer/data/datasets.py:903`); user passes `dataset='wiki-ja'` via
  `transformer/train_publication.py` config dict, which flows through
  `transformer/training/experiment_runner.py:1726` into the dataset constructor.
  When `dataset_name == 'wiki-ja'`, `_setup_cjk_fonts(plt)` is called at
  line 595 before figure construction at line 660.
- `font.family` list semantics: matplotlib's `Text` artist captures
  `FontProperties.family` at construction time; per-glyph fallback iterates
  the family list at draw/savefig time. Setting `rcParams['font.family']`
  before `ax.set_title(...)` is sufficient — confirmed by the matplotlib 3.6
  release notes and `matplotlib.text.Text._get_layout`.
- `font.serif` list is *not* touched by `set_pub_style`, so the CJK font in
  the family list survives subsequent `rcParams.update`s that only assign
  `font.family`. The next attention viz call re-applies anyway as a defense
  in depth.

### Constraints respected

- Surgical change scoped to one helper method.
- No new dependencies, no changes to figure layout or non-CJK styling.
- Publication quality preserved: ASCII still uses DejaVu Serif via the
  generic `'serif'` family alias.
