The manuscript notes W_O as post-aggregation cross-head mixing (Section on Off-Diagonal Gauge Mixing, line ~1687), consistent with the code.

**VERDICT: MATCH** ‚Äî Value aggregation uses `Œ£_j Œ≤_ij Œ©_ij Œº_j` with transport, W_O applied after aggregation, residual adds to original mu.

### 3. State-Dependent Precision (Bayesian Alpha)

**Manuscript**: Eq. `\ref{eq:state_dependent_alpha}` (line ~916-921 of `GL(K)_attention.tex`):

```
Œ±_i* = c‚ÇÄ / (b‚ÇÄ + D_KL(q_i || p_i))
```

With log-barrier regularizer `R(Œ±_i) = b‚ÇÄ¬∑Œ±_i ‚àí c‚ÇÄ¬∑log(Œ±_i)`.

**Code**: `get_bayesian_alpha()` in `variational_ffn.py` (line 763-821).

- Formula (line 819): `alpha = c0 / (b0 + kl_k)` ‚Äî **structurally matches**, but with a per-dimension generalization.
- The manuscript defines `Œ±_i` as scalar (one per agent). The code computes `alpha: (B, N, K)` ‚Äî per-dimension precision. This is a generalization: `kl_k` is the per-dimension KL contribution (line 816): `kl_k = 0.5 * (trace_term + mahal_term - 1 + logdet_term)`.
- `c0 = F.softplus(self.raw_c0)` with shape `(K,)` and `b0 = F.softplus(self.raw_b0)` with shape `(K,)` (line 788-789). Each belief dimension gets its own c‚ÇÄ, b‚ÇÄ.
- For diagonal covariance (line 795-801), the per-dimension KL decomposition is exact: `kl_k = œÉ_q_k/œÉ_p_k + Œ¥Œº_k¬≤/œÉ_p_k + log(œÉ_p_k/œÉ_q_k) ‚àí 1`, all divided by 2.

**DISCREPANCY (benign extension)**: The manuscript defines `Œ±_i` as scalar; the code generalizes to per-dimension `Œ±_k`. When summed over dimensions, the scalar case is recovered as a special case where `c‚ÇÄ_k = c‚ÇÄ/K` and `b‚ÇÄ_k = b‚ÇÄ`. The per-dimension version is strictly more expressive but reduces to the scalar form under appropriate initialization. The manuscript mentions "Both hyper-parameters may be learned via empirical Bayes" (line 925), which the code implements via learnable `raw_c0`, `raw_b0`.

**VERDICT: CONSISTENT (generalization)** ‚Äî `Œ±_k = c‚ÇÄ_k / (b‚ÇÄ_k + kl_k)` is a per-dimension generalization of the manuscript's scalar `Œ±_i* = c‚ÇÄ / (b‚ÇÄ + D_KL)`.

### 7. Weight Decay as Implicit Hyper-Prior

**Manuscript**: Eq. `\ref{eq:deterministic_free_energy_standard}` (line ~1320-1325 of `GL(K)_attention.tex`):

In the deterministic limit with constant Œ±:
```

```
where `Œª_p = Œ± / œÉ_p¬≤`.

"This identifies the standard weight decay coefficient Œª_p as the product of two quantities that the general theory treats independently: the prior precision Œ± and the inverse prior variance 1/œÉ_p¬≤."

The correspondence table (line ~1578) lists: `Œ± ¬∑ D_KL(q_i || p_i)` with "Œ± = const, deterministic" maps to "L2 weight decay" (status: D = derived).

**Code**: `embed_weight_decay` in `train_publication.py` (line 241):
```python
'embed_weight_decay': 0.002,  # L2 hyper-prior on embeddings (Œº_p, œÉ_p, œÜ) via AdamW
```

The comment explicitly calls it "L2 hyper-prior on embeddings." AdamW applies decoupled weight decay as `Œ∏ ‚Üê Œ∏ ‚àí wd ¬∑ Œ∏`, which is equivalent to L2 regularization toward zero with coefficient `wd`. This differs from the VFE prior term `Œ± ¬∑ D_KL(q_i || p_i)` which regularizes toward the prior mean `Œº_prior`, not toward zero.

**Are they independent?** Yes: the VFE E-step implements `Œ± ¬∑ D_KL(q_i || p_i)` internally via the `get_bayesian_alpha()` mechanism (self-coupling term in the VFE gradient). The AdamW weight decay acts on the M-step (embedding parameters), adding an additional L2 penalty on the parameters themselves. These operate on different timescales (E-step vs M-step) and on different variables (beliefs vs embedding parameters).

The manuscript's identification is that in the deterministic constant-gauge limit, the VFE's prior term `Œ± ¬∑ D_KL(q_i || p_i)` collapses to L2 weight decay. The code implements both: the VFE's prior coupling (via `alpha * KL(q||p)` in the E-step) and the optimizer's weight decay (via AdamW). The AdamW weight decay is the hyper-prior (h in the hierarchy `h ‚Üí s ‚Üí p ‚Üí q`), while the VFE's Œ±-term is the prior coupling (p level).

**VERDICT: CONSISTENT** ‚Äî The manuscript's identification `weight_decay ‚Üî Œ± ¬∑ D_KL(q||p)` is valid in the deterministic limit. The code implements both the VFE prior coupling (E-step) and optimizer weight decay (M-step) as independent mechanisms operating at different hierarchical levels.

## Issues Found

**No correctness bugs identified.** All eight focus areas show either exact match or consistent generalization between manuscript and code.

**One notable extension**: The code's per-dimension Bayesian alpha (item 3) is more expressive than the manuscript's scalar alpha. The manuscript defines Œ±_i as a scalar per agent; the code implements Œ±_k as a vector per agent with per-dimension c‚ÇÄ, b‚ÇÄ. **Observation on residual form**: The default residual type ('additive': `mu_q + mu_attn`) does not perfectly match the manuscript's gradient-flow derivation (Eq. residual_gradient_flow: `Œº^(‚Ñì+1) = Œº^(‚Ñì) ‚àí Œ∑ÃÉ ¬∑ Œ£ ¬∑ ‚àÇF/‚àÇŒº`), which would correspond to `mu_q + delta` where delta is the negative gradient step. The `mu_attn` from `aggregate_messages()` is `Œ£_j Œ≤_ij Œ©_ij Œº_j` ‚Äî the weighted transported mean, not a gradient correction. However, the natural gradient of the alignment term simplifies to `Œ£_i (Œ©_ij Œ£_j Œ©_ij^T)^{-1}(Œº_i ‚àí Œ©_ij Œº_j)`, and under the covariance alignment condition `Œ£_i ‚âà Œ©_ij Œ£_j Œ©_ij^T`, this becomes `Œ≤_ij(Œº_i ‚àí Œ©_ij Œº_j)`. The natural gradient step then gives `Œº_i ‚àí Œ∑ ¬∑ Œ£_j Œ≤_ij(Œº_i ‚àí Œ©_ij Œº_j) = (1 ‚àí Œ∑) Œº_i + Œ∑ ¬∑ Œ£_j Œ≤_ij Œ©_ij Œº_j`. The additive residual `mu_q + mu_attn` implicitly absorbs the step size into the learned parameters (consistent with the manuscript's discussion at line ~1377 about the natural gradient recovering the standard transformer update). This is internally consistent but the correspondence is approximate rather than exact.

## Phase 6 Deep Audit (2026-04-12)

Continuation of Phases 1-5 (2026-04-11). Three exploration agents identified 4 code bugs, 3 FD test gaps, 1 gauge equivariance gap, and 5 new manuscript issues.

### Code Bugs Fixed

**BUG A (HIGH) ‚Äî Gradient clipping mismatch with natural gradients**
- **File**: `transformer/training/experiment_runner.py` lines 1248-1285
- **Issue**: When `use_killing_form=True` and `optimizer_type='adamw'` (not `'riemannian_adam'`), phi gradients were preconditioned by the Cartan projector (Killing metric inverse), then clipped using Euclidean `clip_grad_norm_`. This is geometrically wrong: clipping in the Euclidean norm after Riemannian preconditioning inflates the effective trust radius on non-compact symmetric directions by a factor of ~2K.
- **Fix**: Store the Killing metric tensor at init alongside the Cartan projector. In the per-group clipping loop, phi parameter groups now use Riemannian norm clipping: `||Œæ||_K = sqrt(Œ£_v Œæ_v^T K Œæ_v)`, mirroring `RiemannianAdamW._clip_riemannian()` (optimizer.py:212-224). Non-phi groups retain Euclidean clipping.
- **Impact**: Prevents large steps in non-compact (symmetric) gauge directions when using AdamW + Killing form. Only affects configs with `use_killing_form=True` and `optimizer_type='adamw'`.

**BUG B (MEDIUM) ‚Äî Eigenvalue floor too loose in gauge_geometry.py**
- **File**: `transformer/analysis/gauge_geometry.py` line 125
- **Issue**: Eigenvalue modulus clamped to `1e-7`, but float32 machine epsilon is ~`1.2e-7`. Values in `[1e-7, 1.2e-7]` survive the clamp but are numerically indistinguishable from zero; `log(1e-7) ‚âà -16.1` introduces noise in the holonomy logm reconstruction.
- **Fix**: Changed `eps = 1e-7` to `eps = 1e-6`, providing a full order of magnitude above float32 machine epsilon.

**BUG C (MEDIUM) ‚Äî Exponential fitting hides oscillatory convergence**
- **File**: `transformer/analysis/fiber_trajectory.py` lines 348-402
- **Issue**: `fit_convergence_rate()` used log-space OLS to fit `d(t) = A¬∑exp(-Œªt)`. For oscillatory convergence curves (e.g., damped-then-overshooting E-step dynamics), a negative OLS slope was silently clamped to `0.0`, hiding the oscillation.
- **Fix**: Before the OLS fit, check monotonicity of `log_d` first differences. If sign changes exceed 50% of intervals (highly oscillatory), return `float('nan')` so callers can detect the poor model fit. Updated docstring to document the NaN return case.
- **Caller impact**: Only caller is `analyze_fiber_trajectory()` (line 484), which stores the rate in `FiberTrajectoryStats.convergence_rate` (a float field). NaN propagates correctly.

**BUG D (LOW) ‚Äî Prior bank silent SO(3) default**
